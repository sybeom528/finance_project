# 3.2 LSTM 의 게이트 메커니즘 — 덧셈 경로로 gradient 를 살리다

**프로젝트 연결점:** 본 프로젝트에서 최종적으로 사용할 시퀀스 모델(GRU)의 원형(prototype)

## 0. 이 문서에서 답할 4가지 질문

1. **LSTM 은 왜 만들어졌는가?** → 3.1 의 vanishing/exploding gradient 문제를 **구조적으로** 해결하기 위해 (Hochreiter & Schmidhuber, 1997).
2. **"cell state" 는 정확히 무엇인가?** → 시간을 따라 **덧셈으로 이어지는** 정보 파이프라인. 수학적으로 "gradient highway".
3. **세 게이트는 각각 무엇을 제어하는가?** → forget (무엇을 잊을지), input (무엇을 새로 넣을지), output (얼마만큼 내보낼지).
4. **덧셈 구조가 왜 소실을 막는가?** → `∂c_t/∂c_{t-1} = f_t` 이므로 `f_t≈1` 이면 gradient 가 거의 손실 없이 과거까지 도달.

---

## 1. 문제 다시 보기 — 3.1 에서 합의한 것

Vanilla RNN 의 상태 전이:

```
h_t = tanh( W_x · x_t + W_h · h_{t-1} + b )
```

Gradient 역전파 시:

```
∂h_T/∂h_0 = Π_{t=1}^{T} diag(tanh'(z_t)) · W_h
```

**두 개의 독립적인 감쇠 원천:**
- `W_h` 의 spectral radius σ: σ<1 → 소실, σ>1 → 폭발
- `tanh'` 의 평균값 ≈ 0.61: σ=1 이어도 매 시점 0.61 씩 감쇠

→ **T=60 에서 (0.61)^60 ≈ 8.8×10⁻¹⁴**. 근본적으로 **곱셈 연쇄**가 원인.

**따라서 해결의 실마리는:**

> "어떻게 하면 매 시점 **곱해지지 않는** 정보 경로를 만들 수 있을까?"

답: **덧셈으로 이어지는 별도의 통로를 만든다.** 이것이 LSTM 의 핵심 발명품인 **cell state `c_t`** 입니다.

---

## 2. LSTM 의 핵심 아이디어 — "두 개의 파이프"

### 2.1 직관: 회사의 회의 노트와 장부

비유: 매 분기마다 회의가 열리는 회사를 상상해봅시다.
- **Vanilla RNN (`h_t`)**: 회의실 화이트보드 하나. 매 회의마다 **전부 지우고 다시 쓴다**. 1년 전 논의한 내용은 완전히 날아감.
- **LSTM (`c_t` + `h_t`)**: 장부(`c_t`) 와 화이트보드(`h_t`) 분리.
  - **장부 `c_t`**: "잊지 말아야 할 것" 만 **골라서** 계속 누적. 필요 없는 건 **명시적으로** 지움.
  - **화이트보드 `h_t`**: 현재 회의에서 보여줄 요약본. 장부를 기반으로 매번 새로 작성.

→ 장부는 **기본적으로 유지(덧셈)**, 화이트보드는 **매번 갱신(곱셈)**. 곱셈으로 인한 소실은 화이트보드에서만 발생 → 장부에는 과거 정보가 **유실 없이** 보존됨.

### 2.2 수학적으로: Cell State `c_t`

LSTM 의 cell state 갱신 규칙 (단순화):

```
c_t = f_t ⊙ c_{t-1} + i_t ⊙ g_t
      └───┬────┘    └───┬────┘
  과거 기억 중 남길 부분   새로 추가할 정보
```

- `⊙` = Hadamard product (원소별 곱셈). 따라서 `f_t`, `i_t`, `g_t` 는 `c_t` 와 같은 차원의 벡터.
- `f_t ∈ (0, 1)^H`: **forget gate** — 각 차원별로 얼마나 "기억"할지 결정 (1 = 그대로, 0 = 삭제).
- `i_t ∈ (0, 1)^H`: **input gate** — 각 차원별로 얼마나 "받아들일지" 결정.
- `g_t ∈ (-1, 1)^H`: **candidate** — 새로 추가할 후보 정보.

**핵심 관찰**:
- `c_t` 가 `c_{t-1}` 에 **직접 더해지는** 구조.
- `f_t ≈ 1` 이면 `c_t ≈ c_{t-1} + (새 정보)` → 정보가 시간을 따라 **무손실로 흐름**.

이게 **"constant error carousel"** 이라 불리는 LSTM 의 핵심 구조입니다.

## 3. Hidden State `h_t` 와 Cell State `c_t` 의 관계

LSTM 은 두 개의 상태를 **동시에** 관리합니다.

```
c_t : 내부 장기 기억 (long-term memory) — 시간을 따라 거의 변하지 않음
h_t : 외부 출력 + 단기 기억 (short-term memory / output) — 매 시점 갱신
```

`h_t` 는 `c_t` 로부터 파생됩니다:

```
h_t = o_t ⊙ tanh(c_t)
```

- `o_t ∈ (0,1)^H`: **output gate** — 장기 기억 중 얼마만큼을 현재 출력으로 드러낼지 결정.
- `tanh(c_t)`: 값의 범위를 `(-1, 1)` 로 제한 (cell state 는 이론상 무한히 누적될 수 있으므로).

**정리**: LSTM 셀은 "장부(c_t)는 안에 숨기고 요약본(h_t)만 외부에 공개" 하는 구조.

---

## 4. 세 게이트의 수식

**게이트 = 시그모이드로 0~1 사이 값을 만드는 작은 신경망**.

각 게이트는 "현재 입력 `x_t`" 와 "이전 hidden `h_{t-1}`" 을 보고 결정합니다.

### 4.1 Forget Gate `f_t`

```
f_t = σ( W_f · [h_{t-1}, x_t] + b_f )
```

- `[h_{t-1}, x_t]`: 두 벡터를 **concat** (shape: H + F)
- `W_f`: (H, H+F) 가중치
- `σ`: 시그모이드 함수, `1/(1+e^{-z})` ∈ (0, 1)
- 출력 `f_t ∈ (0,1)^H`: 원소별로 "이 차원의 과거 기억을 얼마나 유지할까?"

### 4.2 Input Gate `i_t` 와 Candidate `g_t`

```
i_t = σ( W_i · [h_{t-1}, x_t] + b_i )
g_t = tanh( W_g · [h_{t-1}, x_t] + b_g )
```

- `i_t`: 새 정보를 **얼마나** 받을지 (0~1)
- `g_t`: **무엇을** 받을지 (-1~1, tanh 이므로)
- 두 개를 곱한 `i_t ⊙ g_t` 가 cell state 에 더해질 실질적 새 정보.

### 4.3 Output Gate `o_t`

```
o_t = σ( W_o · [h_{t-1}, x_t] + b_o )
```

- `o_t`: cell state 중 얼마나 외부로 드러낼지 (0~1)

### 4.4 전체 LSTM 셀의 수식 요약

매 시점 `t` 에 대해:

```
f_t = σ(W_f · [h_{t-1}, x_t] + b_f)     # forget gate
i_t = σ(W_i · [h_{t-1}, x_t] + b_i)     # input gate
g_t = tanh(W_g · [h_{t-1}, x_t] + b_g)  # candidate
c_t = f_t ⊙ c_{t-1} + i_t ⊙ g_t         # cell state update (덧셈!)
o_t = σ(W_o · [h_{t-1}, x_t] + b_o)     # output gate
h_t = o_t ⊙ tanh(c_t)                   # hidden state update
```

초기값: `h_0 = 0`, `c_0 = 0`.

### 4.5 파라미터 수 계산

Vanilla RNN 대비 얼마나 증가하는지 보면:

| 모델 | 학습 파라미터 | 근거 |
|---|---|---|
| Vanilla RNN | `H² + HF + H` | W_h + W_x + b |
| **LSTM** | **`4·(H² + HF + H)`** | 위를 4벌 (f, i, g, o 각각) |
| GRU (3.3) | `3·(H² + HF + H)` | 게이트 2개 + candidate 1개 |

**실무 함의**: LSTM 은 RNN 대비 **4배의 파라미터**가 필요 → 학습 데이터가 충분해야 일반화됨. 시계열 금융 데이터처럼 데이터가 제한적일 때는 **GRU (3배)** 가 과적합 덜 됨.

> 우리 프로젝트는 MSFT 단일 종목 약 10년치 (≈2500일) → 과적합 위험 있어 **GRU 로 출발**, 필요 시 LSTM 으로 확장.

## 5. Gradient 분석 — 왜 덧셈이 소실을 막는가

### 5.1 Cell State 의 시간 미분

LSTM 의 핵심 주장은 **"cell state 는 gradient 가 거의 손실 없이 흐른다"** 입니다. 수학적으로 증명해봅시다.

`c_t = f_t ⊙ c_{t-1} + i_t ⊙ g_t` 에서 `c_{t-1}` 에 대한 편미분:

```
∂c_t/∂c_{t-1} = diag(f_t)
```

(입력/후보/게이트들은 `c_{t-1}` 에 직접 의존하지 않는다고 1차 근사)

### 5.2 BPTT 에서의 효과

Vanilla RNN:
```
∂h_T/∂h_0 = Π_{t=1}^{T} diag(tanh'(z_t)) · W_h
→ |∂h_T/∂h_0| ≤ (max_t tanh'(z_t))^T · σ(W_h)^T
→ σ<1 이면 지수 소실
```

LSTM (cell state 경로):
```
∂c_T/∂c_0 = Π_{t=1}^{T} diag(f_t)
→ |∂c_T/∂c_0| ≤ (max_t f_t)^T
```

**핵심**: `f_t` 는 **학습 가능한** 값이라, 네트워크가 "과거 기억이 중요하다" 고 판단하면 **스스로 `f_t ≈ 1` 을 학습**. 그럼 `1^T = 1` 로 gradient 가 **거의 유실 없이** 과거까지 도달.

### 5.3 구체적 비교 (T=60)

| 시나리오 | Vanilla RNN | LSTM (학습된 f_t≈0.95) | LSTM (학습된 f_t≈0.99) |
|---|---|---|---|
| T=60 감쇠 | σ=0.9, tanh'=0.6 → (0.54)^60 ≈ 10⁻¹⁷ | 0.95^60 ≈ 0.046 | 0.99^60 ≈ 0.547 |

**결론**: LSTM 은 **포기하지 않아야 할 정보에 대해 자동으로 `f_t → 1`** 을 학습함으로써 gradient 가 60일 전까지 의미 있게 흐르게 합니다. Vanilla RNN 은 이 선택지가 없음.

### 5.4 Exploding 은 어떻게?

Cell state 는 덧셈으로 무제한 누적 가능 → **값 자체가 폭발**할 수 있음.
- 완화 1: `h_t = o_t ⊙ tanh(c_t)` 에서 **tanh 으로 출력 범위를 (-1,1) 로 제한**
- 완화 2: gradient clipping (여전히 필요)
- 완화 3: forget gate bias 를 1 로 초기화 (`b_f = 1`) — 학습 초기에 "일단 기억하자" 상태로 시작

## 6. Forget Gate Bias 초기화 — 실무 트릭

`b_f = 0` 으로 시작하면 `f_t = σ(0) = 0.5` → 매 시점 **절반씩 정보가 지워짐**. 학습 초기부터 vanishing 발생.

**권장**: `b_f = 1` 로 초기화 → `f_t = σ(1) ≈ 0.73` → 초기에도 대부분의 정보 유지.

**이론적 근거** (Jozefowicz et al., 2015): 수백 개의 LSTM 변형을 실험한 결과, `b_f = 1` 초기화가 가장 안정적.

**PyTorch 구현**:

```python
lstm = nn.LSTM(input_size=F, hidden_size=H, batch_first=True)
# nn.LSTM 의 bias 는 [b_i, b_f, b_g, b_o] 를 concat 한 구조 (4H,)
for name, param in lstm.named_parameters():
    if 'bias' in name:
        # forget gate bias 구간만 1 로
        n = param.size(0)
        param.data[n//4 : n//2].fill_(1.0)
```

**우리 프로젝트 적용**: 금융 시계열은 장기 의존성이 중요 → `b_f=1` 필수.

## 7. LSTM vs Vanilla RNN 비교표

| 항목 | Vanilla RNN | LSTM |
|---|---|---|
| 상태 | `h_t` 한 개 | `c_t`, `h_t` 두 개 |
| 상태 갱신 | 완전 교체 (곱셈 경로만) | 선택적 유지 + 추가 (덧셈 경로) |
| 파라미터 | `H²+HF+H` | `4·(H²+HF+H)` |
| 장기 기억 | 이론상 불가 (T>20 무리) | 수백 스텝까지 가능 |
| 폭발 | `W_h` 에서 발생 | cell state 값에서 발생 (완화 필요) |
| 소실 | 구조적으로 발생 | `f_t` 학습으로 회피 가능 |
| 학습 안정성 | 매우 나쁨 | 비교적 안정 |
| 실무 사용 | 거의 없음 | 표준 (+ GRU 가 더 흔함) |

## 8. 우리 프로젝트와의 연결

### 8.1 현재 상태 (XGBoost)

```
X_is.shape = (2310, 17)        ← 날짜별 독립된 17개 피처
모델: XGBoost                   ← 행 단위 학습 (시간 의존성 무시)
출력: proba[5]                  ← 5분위 확률
```

### 8.2 목표 상태 (GRU)

```
X_is_seq.shape = (N, 60, 17)   ← 60일 lookback window
모델: GRU (LSTM 의 경량판)
출력: proba[5]                  ← 동일 인터페이스
```

### 8.3 왜 LSTM 이 아니라 GRU 인가?

- **데이터 양**: 단일 종목 2500일 → LSTM 의 4배 파라미터 감당 어려움
- **실험 결과 (논문)**: GRU 와 LSTM 성능 비슷, GRU 가 빠름
- **해석 용이성**: 게이트 2개가 3개보다 분석 쉬움

→ **본 프로젝트는 GRU 로 진행**, LSTM 은 **비교 baseline** 으로만 학습.

### 8.4 B-L 파이프라인 연결

LSTM/GRU 의 출력 `proba[5]` 가 XGBoost 의 `proba[5]` 와 **동일한 shape** 이므로, 후속 Black-Litterman 코드는 **전혀 수정하지 않아도** 됩니다. 이 인터페이스 보존이 3.1~4.4 전체 학습의 설계 원칙.

## 10. 다음 토픽 (3.3 GRU) 으로 가는 다리

GRU 는 LSTM 의 **단순화**:
- **cell state 와 hidden state 를 `h_t` 하나로 통합**
- 게이트 3개 → 2개 (reset, update)
- 파라미터 25% 감소, 성능은 비슷

수식만 미리 보면:

```
z_t = σ(W_z · [h_{t-1}, x_t])           # update gate (LSTM 의 f+i 통합)
r_t = σ(W_r · [h_{t-1}, x_t])           # reset gate
h̃_t = tanh(W · [r_t ⊙ h_{t-1}, x_t])    # candidate
h_t = (1 - z_t) ⊙ h_{t-1} + z_t ⊙ h̃_t   # 덧셈 구조 유지
```

**핵심 아이디어 유지**: 마지막 줄이 `h_t = (1-z_t) h_{t-1} + z_t h̃_t` 형태의 **덧셈 업데이트** — LSTM 의 gradient highway 원리를 그대로 계승.

3.3 에서 이 구조를 자세히 보고, 3.4 PyTorch 학습 루프, 3.5 분류 출력, 3.6 학습 안정화까지 이어집니다.

## 부록 A — "왜 4개의 게이트인가?" 설계 철학

LSTM 은 인위적으로 복잡해 보이지만, 각 게이트는 **독립적인 질문에 답합니다**:

| 게이트 | 답하는 질문 | 비유 |
|---|---|---|
| `f_t` | "과거의 어떤 부분을 **잊을까**?" | 장부에서 **줄 긋기** |
| `i_t` | "새 정보를 **받아들일 가치**가 있나?" | 장부에 **기록할지** 결정 |
| `g_t` | "**무슨 내용**을 기록할까?" | 장부에 **쓸 문구** 작성 |
| `o_t` | "이걸 **외부에 보여줄까**?" | 장부를 **공개할지** |

**분리의 이유**: 각 결정이 **독립적으로 학습**돼야 유연함. 예: "과거는 유지하되(f=1), 새 정보는 안 받음(i=0), 내부적으로만 보관(o=0)" 같은 선택도 가능.

**cf. GRU 는 `f` 와 `i` 를 하나의 `z` 로 통합** (`z = 1 - (1-z)`, 즉 "유지할 비율 = 1 - 새로 받을 비율"). 단순화 대가로 유연성 약간 희생.

## 부록 B — LSTM 의 변형들 (참고용)

실무에서 쓰이는 대표 변형:

1. **Peephole LSTM** (Gers & Schmidhuber, 2000): 게이트들이 `c_{t-1}` 도 볼 수 있게 확장
2. **Coupled LSTM**: `f_t + i_t = 1` 로 묶음 (GRU 와 비슷)
3. **Bidirectional LSTM**: 시퀀스를 **앞→뒤, 뒤→앞 양방향** 으로 처리. 분류 문제에선 잘 작동하지만 **시계열 예측에는 미래 정보 유출 위험** 있어 주의.

**우리 프로젝트 주의점**: Bidirectional 금지 (look-ahead bias, 2.3 참고).
