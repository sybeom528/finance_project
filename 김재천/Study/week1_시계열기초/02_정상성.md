## 1. 정상성이란 무엇인가

앞 챕터에서 "주가는 i.i.d.가 아니라서 문제다"라고 말했습니다. 하지만 시계열 분석에서는 i.i.d.보다 *느슨한* 조건으로도 충분한 경우가 많습니다. 그 느슨한 조건이 바로 **정상성**입니다.

정상성에는 두 가지가 있습니다.

### 엄밀 정상성 (Strict Stationarity)

시계열의 **결합 분포(joint distribution)** 자체가 시간 이동에 대해 불변.

$$ F(X_{t_1}, X_{t_2}, \ldots, X_{t_k}) = F(X_{t_1+h}, X_{t_2+h}, \ldots, X_{t_k+h}) \quad \forall h $$

실무에서는 거의 검증 불가능(전체 분포를 알아야 함). 이론용.

### 약정상성 (Weak Stationarity) — 실무에서 다루는 것

평균, 분산, 자기공분산만 시간에 대해 일정하면 됨. 실무 시계열 분석의 표준.

**세 가지 조건**

| 조건 | 수식 | 의미 |
|---|---|---|
| (1) 평균 일정 | $E[X_t] = \mu$ (상수) | 시간이 흘러도 평균값이 바뀌지 않음 |
| (2) 분산 일정 | $\text{Var}(X_t) = \sigma^2$ (상수) | 시간이 흘러도 분산이 바뀌지 않음 |
| (3) 자기공분산이 시차에만 의존 | $\text{Cov}(X_t, X_{t+h}) = \gamma(h)$ | $t$ 시점이 언제든 $h$만큼 떨어진 값들과의 공분산은 같음 |

> 세 번째 조건이 처음 보면 어렵습니다. 쉽게 말하면:
> *"지금 값과 10일 전 값의 관계"가 2020년에도, 2023년에도, 2026년에도 똑같아야 한다.*

이 세 조건을 만족하는 시계열을 **약정상(weakly stationary)** 또는 **공분산 정상(covariance stationary)** 이라고 합니다. 이후로는 그냥 "정상"이라고 하면 이걸 의미합니다.

### 시각적으로 보는 정상 vs 비정상

```
비정상 시계열 (주가)              정상 시계열 (수익률)
   │                                │
   │         ╱╲                     │  ╲    ╱╲
값 │       ╱╲╱  ╲                값 │   ╲  ╱  ╲   ╱
   │     ╱╲╱        ╲               │────╲╱────╲─╱────  ← 평균 0
   │   ╱╲╱            ╲              │        ╲╱
   │ ╱╲╱                ╲            │
   │╱                                │
   └─────────────── 시간              └─────────────── 시간

평균↑, 분산↑ (시간에 따라 변함)      평균·분산 일정
```

## 2. 왜 정상성이 중요한가

### (1) 통계적 추론의 전제

평균이나 분산을 추정하려면 "값이 뽑힌 분포가 안정적"이어야 합니다. 비정상이면 관측값마다 다른 분포에서 온 셈이라 평균 추정 자체가 무의미합니다.

### (2) 고전 시계열 모델의 전제

ARIMA, VAR, 공적분(Cointegration) 등 거의 모든 고전 시계열 모델은 정상성을 **전제**로 설계되어 있습니다. 비정상 시계열에 그대로 적용하면 결과가 왜곡됩니다.

### (3) 딥러닝 시퀀스 모델에도 유리

LSTM/GRU/Transformer 같은 시퀀스 모델은 엄밀히 말해 정상성을 *요구*하지는 않습니다. 하지만:

- **학습 안정성**: 비정상 입력은 경사가 폭발/소실되기 쉬움
- **일반화**: 학습 분포와 추론 분포가 다르면(예: 학습 시 주가 $200 범위, 실전 $400 범위) 모델이 본 적 없는 영역에서 예측
- **실전 규칙**: "딥러닝에 넣기 전에 정상화해라"가 업계 표준

> **현재 프로젝트에서 `log_return`을 입력으로 쓰는 선택이 이미 정상성을 확보하는 전처리**였습니다. 다음 챕터(1.3 자기상관)에서 이 선택이 얼마나 중요한지 한 번 더 느끼게 될 것입니다.

## 3. 정상성 진단하기

정상성을 확인하는 방법은 두 단계로 합니다.

```
[Step A] 시각적 진단 (빠른 스크리닝)
            ↓
[Step B] 통계 검정 (ADF + KPSS 이중 확인)
```

### Step A — 시각적 진단

#### (1) 시계열 자체 플롯

뚜렷한 추세(상향/하향)나 평균 변화가 보이면 비정상 의심.

#### (2) Rolling 통계 플롯

**rolling mean / rolling std** 를 시계열 위에 겹쳐 그립니다.

```python
rolling_mean = series.rolling(window=252).mean()   # 1년 이동평균
rolling_std  = series.rolling(window=252).std()    # 1년 이동표준편차
```

- rolling mean이 평평하게 유지 → 평균 정상성 (조건 1)
- rolling std가 평평하게 유지 → 분산 정상성 (조건 2)

> 시각적 진단은 "의심 가는지" 확인하는 초기 스크리닝입니다. 최종 판단은 통계 검정으로.

### Step B — 통계 검정

정상성 검정에는 두 개의 대표 검정이 있습니다.

#### ADF 검정 (Augmented Dickey-Fuller Test)

**단위근(Unit Root) 검정**이라고도 부릅니다. AR(1) 모형 $X_t = \rho X_{t-1} + \epsilon_t$ 에서 $\rho = 1$ 이면 (= 단위근 존재) 비정상이 됩니다.

| 항목 | 내용 |
|---|---|
| **귀무가설 $H_0$** | 단위근 존재 → **비정상** |
| **대립가설 $H_1$** | 단위근 없음 → **정상** |
| **결론 기준** | p-value < 0.05 → $H_0$ 기각 → **정상** |

```python
from statsmodels.tsa.stattools import adfuller

result = adfuller(series, autolag='AIC')
stat, p_value = result[0], result[1]
print(f'ADF statistic: {stat:.4f}, p-value: {p_value:.4f}')
```

**해석 포인트**
- p-value가 0.05보다 **작으면** → 정상 (귀무가설 기각)
- p-value가 0.05보다 **크면** → 비정상 (기각 실패, 단 "정상이다"를 증명한 것은 아님)

#### KPSS 검정 (Kwiatkowski-Phillips-Schmidt-Shin Test)

ADF와 **정반대 가설**을 쓰는 검정입니다.

| 항목 | 내용 |
|---|---|
| **귀무가설 $H_0$** | **정상** (level 또는 trend stationary) |
| **대립가설 $H_1$** | **비정상** |
| **결론 기준** | p-value < 0.05 → $H_0$ 기각 → **비정상** |

```python
from statsmodels.tsa.stattools import kpss

result = kpss(series, regression='c')   # 'c'=level stationary, 'ct'=trend stationary
stat, p_value = result[0], result[1]
print(f'KPSS statistic: {stat:.4f}, p-value: {p_value:.4f}')
```

**주의:** ADF와 해석이 반대입니다. p-value가 **크면** 정상이고, **작으면** 비정상입니다.

#### 왜 두 검정을 함께 써야 하는가

각 검정 단독으로는 한계가 있습니다.

- ADF: **검정력(power)이 낮다** — 실제로 정상인데도 비정상으로 잘못 판단(Type II 오류)하기 쉬움
- KPSS: 작은 샘플에서 불안정할 수 있음
- 두 검정의 가설이 **반대**라 서로 보완이 됨

**결합 판단 표 (중요)**

| ADF 결과 | KPSS 결과 | 결론 |
|---|---|---|
| 정상 (p<0.05) | 정상 (p>0.05) | **명확히 정상** |
| 비정상 (p>0.05) | 비정상 (p<0.05) | **명확히 비정상** |
| 정상 (p<0.05) | 비정상 (p<0.05) | 차이 정상(difference stationary) 의심 — 차분 필요 |
| 비정상 (p>0.05) | 정상 (p>0.05) | 추세 정상(trend stationary) 의심 — 추세 제거 필요 |

> 현재 프로젝트 데이터에 대해 이 표를 직접 채워봅니다. 실습 노트북에서!

## 4. 정상성 만들기 (Transformation)

진단 결과 비정상이면 변환을 적용합니다. 우선순위는 다음과 같습니다.

### (1) 차분 (Differencing) — 1순위 선택지

**1차 차분:**
$$ \Delta X_t = X_t - X_{t-1} $$

**2차 차분 (거의 안 쓰임):**
$$ \Delta^2 X_t = \Delta X_t - \Delta X_{t-1} = X_t - 2X_{t-1} + X_{t-2} $$

```python
series_diff = series.diff().dropna()    # 1차 차분
```

**왜 1차 차분이 "거의 항상" 1순위인가**

1. 거의 모든 경제·금융 시계열은 1차 차분만으로 정상이 됨
2. 계산이 단순하고 해석이 직관적 ("어제 대비 변화량")
3. **주가의 1차 차분 ≒ 수익률** — 이미 우리가 하고 있던 일!

> **핵심 통찰**: `log_return = log(P_t) - log(P_{t-1})` 은 **로그 가격의 1차 차분**입니다. 우리는 이미 정상성을 확보하는 표준 처리를 하고 있었습니다.

**과차분 (Over-differencing) 주의**

차분을 너무 많이 하면 오히려 노이즈가 증폭됩니다. 1차면 충분한데 2차·3차까지 하면 MA(q) 항이 인위적으로 늘어나서 모델 복잡도만 올라갑니다. **우선 1차 차분 → 정상이면 멈춤**이 원칙.

### (2) 로그 변환 (Log Transform)

분산이 수준(level)에 비례해 커지는 경우 (예: 매출이 커질수록 변동폭도 커짐) 쓴 이유는 분산 안정화:

$$ Y_t = \ln(X_t) $$

로그 변환 후에도 추세가 남아있으면 추가로 차분을 적용합니다. → **"로그 후 차분"이 곧 `log_return`** 입니다.

### (3) 추세 제거 (Detrending)

추세 정상(trend stationary) 시계열(= 추세만 빼면 정상)인 경우 쓰는 방법.

```python
# 선형 추세 제거
from scipy.signal import detrend
series_detrended = detrend(series)
```

### (4) 계절 차분 (Seasonal Differencing)

주간/연간 주기가 있는 경우. 예를 들어 매월 반복되는 패턴이면:

$$ Y_t = X_t - X_{t-12} $$

> 일별 주가에서는 뚜렷한 계절성이 드물어서 현재 프로젝트에선 거의 쓰지 않습니다. 참고만.

### 변환 우선순위 정리

```
1. 1차 차분 시도  → 정상이면 STOP
2. 로그 + 1차 차분 시도 (= log_return)
3. 추세 제거 시도
4. 계절 차분 시도 (필요한 경우만)
```

대부분의 금융 시계열은 **1번 또는 2번**에서 끝납니다.

## 부록 A — 시계열 정규화(Normalization) 기법 정리

> **왜 별도 부록인가**: 시계열에서는 일반 ML에서 쓰는 `StandardScaler`/`MinMaxScaler` 조합만으로 부족합니다. 시계열 전용 기법을 이해해야 합니다.

### A-1. 시계열 정규화의 3대 원칙

1. **Look-ahead bias 금지** — 미래 정보를 과거 정규화에 쓰면 안 됨
2. **Train fold 통계만 사용** — test/val fold는 학습된 스케일러로 transform만
3. **분포 변화에 강건(robust)** — 코로나·리먼 같은 극단 사건에도 안정

### A-2. 방법 목록 (실무 우선순위)

#### (1) Rolling / Expanding Z-Score — **시계열의 Gold Standard**

$$ z_t = \frac{x_t - \mu_{[t-w,\ t-1]}}{\sigma_{[t-w,\ t-1]}} $$

```python
window = 252  # 약 1년 거래일
rolling_mean = series.rolling(window).mean().shift(1)   # shift(1) 필수
rolling_std  = series.rolling(window).std().shift(1)
z_score = (series - rolling_mean) / rolling_std
```

**핵심:**
- **과거 데이터만** 사용 → look-ahead bias 원천 차단
- `.shift(1)`로 "오늘 값으로 오늘 평균 만드는" 누수 방지
- 금융 시계열의 **표준 선택**

#### (2) StandardScaler (전역 Z-Score)

$$ z = \frac{x - \mu_{\text{전체}}}{\sigma_{\text{전체}}} $$

- 전체 평균/분산 사용 → **시계열에서 위험**
- 안전하게 쓰려면: **train fold 안에서만 `fit()`**, test에는 `transform()`만

#### (3) MinMaxScaler

$$ z = \frac{x - x_{\min}}{x_{\max} - x_{\min}} $$

- 범위 [0, 1]
- **극단치에 매우 민감** → 블랙스완 한 번이면 나머지 다 작아짐
- **시계열에서는 비추천**

#### (4) RobustScaler — 이상치 강건

$$ z = \frac{x - \text{median}}{\text{IQR}} $$

- 중앙값과 사분위범위 사용
- **fat-tail 피처에 적합** (변동성, 거래량, PER 등)
- 코로나 폭락 같은 극단 사건에 흔들리지 않음

#### (5) Log 변환

$$ y = \log(x) \quad \text{또는} \quad y = \log(1 + x) \ (\text{음수 가능})$$

- 오른쪽으로 긴 꼬리(거래량, 시가총액) → 정규에 가깝게
- 1.1 챕터에서 다룬 변환

#### (6) 차분 (Differencing)

$$ \Delta x_t = x_t - x_{t-1} $$

- 1.2의 정상화 수단. 정규화와 동시에 정상성 확보
- `log_return = log(close)` 의 1차 차분 = 이미 한 작업

#### (7) Quantile Transformation — 비선형 강제 매핑

```python
from sklearn.preprocessing import QuantileTransformer
qt = QuantileTransformer(output_distribution='normal')
```

- 분위수를 사용해서 정규 or 균등분포로 강제
- 이상치 영향 거의 0
- **단점**: 비선형이라 해석이 어려움

#### (8) Power Transformation (Box-Cox / Yeo-Johnson)

```python
from sklearn.preprocessing import PowerTransformer
pt = PowerTransformer(method='yeo-johnson')
```

- 분산 안정화 + 정규화 동시
- Box-Cox: 양수만 / Yeo-Johnson: 음수 가능
- 변동성 시계열에 효과적

#### (9) GARCH 잔차 표준화 — 금융 전용 고급

```python
y_t_normalized = y_t / σ_t_GARCH
```

- 변동성 클러스터링 자체를 모델로 추정 후 제거
- 구현 복잡 → 1차로는 RobustScaler로 충분

### A-3. 프로젝트 피처별 추천 매핑

| 피처 | 1차 추천 | 대안 | 이유 |
|---|---|---|---|
| `log_return_1d` | **정규화 불필요** | (필요 시) RobustScaler | 이미 정상 + 작은 수치 범위 |
| `vol_20d_ann` | **Rolling Z-Score** | RobustScaler | 클러스터링 + fat-tail |
| `mom_60d` | **Rolling Z-Score** | StandardScaler (train만) | 추세 잔존 가능 |
| `volume` | **Log + Rolling Z-Score** | Log + RobustScaler | 매우 왜곡된 분포 |
| `pe_ratio` | **RobustScaler + Winsorization** | Quantile Transform | 음수/극단치 존재 |

### A-4. 잘못된 정규화의 대표 사례

```python
# ❌ 잘못된 예 — look-ahead bias 발생
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
X_all_normalized = scaler.fit_transform(X_all)  # 전체 데이터로 fit
X_train, X_test = split(X_all_normalized)
# → test 데이터의 통계가 train 정규화에 섞여들어감 → 누수

# ✅ 올바른 예
X_train, X_test = split(X_all)
scaler = StandardScaler()
X_train_norm = scaler.fit_transform(X_train)   # train으로만 fit
X_test_norm = scaler.transform(X_test)          # test는 transform만

# ✅✅ 더 좋은 예 (시계열 표준) — Rolling Z-Score
rolling_mean = X.rolling(252).mean().shift(1)
rolling_std  = X.rolling(252).std().shift(1)
X_norm = (X - rolling_mean) / rolling_std
```

### A-5. 한 줄 요약

> **시계열 정규화의 기본은 "과거 데이터만으로 계산"이다. Rolling Z-Score가 그것을 가장 깔끔히 구현한다.**

## 부록 B — ADF vs KPSS 심화

> **핵심 질문**: 두 검정은 해석만 반대인가, 아니면 본질적으로 다른가?
> **답**: 본질적으로 다름. 두 검정은 *다른 수학적 모델* 을 검증한다.

### B-1. 서로 다른 수학적 모델

**ADF는 단위근(Unit Root) 검정**

AR(1) 모형을 가정하고 한 가지만 묻는다:

$$ X_t = c + \rho X_{t-1} + \epsilon_t, \quad \text{검정: } \rho = 1 \text{인가?} $$

- $\rho = 1$ → 랜덤워크 → **비정상**
- $|\rho| < 1$ → 평균 회귀 → **정상**

즉 ADF는 **"이 시계열이 랜덤워크인가?"** 를 묻는다.

**KPSS는 정상성 분해 검정**

시계열을 세 성분으로 분해:

$$ X_t = r_t + \beta t + \epsilon_t, \quad r_t = r_{t-1} + u_t $$

그리고 묻는다: **"랜덤워크 성분 분산 $\sigma^2_u = 0$ 인가?"**

- $\sigma^2_u = 0$ → 랜덤워크 성분 없음 → **정상**
- $\sigma^2_u > 0$ → 랜덤워크 성분 존재 → **비정상**

### B-2. 검정력과 오류 방향 차이

| 검정 | 자주 틀리는 방향 | 약점 영역 |
|---|---|---|
| ADF | 정상→비정상 오판 (Type II) | $\rho$가 1 근처일 때 |
| KPSS | 비정상→정상 오판 (Type I) | 장기 메모리, 조건부 이분산성 |

**→ 두 검정의 오류 방향이 반대라서 함께 쓰면 상호 보완**

### B-3. 실습 결과 재해석 — `vol_20d_ann` 케이스

| 시계열 | ADF p | KPSS p | 해석 |
|---|---|---|---|
| `close` | 0.9957 | 0.0100 | 명확히 비정상 |
| `log_return_1d` | 0.0000 | 0.1000 | 명확히 정상 |
| **`vol_20d_ann`** | **0.0000** | **0.0565** | **갈등** |

**`vol_20d_ann` 해석:**
- ADF 관점: AR(1)에서 $\rho < 1$ (평균 회귀) → 정상
- KPSS 관점: 장기 의존성·클러스터링 감지 → 약한 비정상 시사
- **결론**: "AR 관점에선 정상이지만, 분산 구조 자체가 시변(time-varying)한다" → **조건부 이분산성**의 흔적. GARCH 영역.

### B-4. 한 문장

> **ADF와 KPSS는 같은 사물의 "앞뒤"가 아니라, 서로 다른 각도에서 본 "사진 두 장"이다.**
