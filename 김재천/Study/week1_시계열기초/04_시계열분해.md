## 1. 분해란 무엇인가 — 기본 아이디어

### 1.1 시계열 = 추세 + 계절성 + 잔차

전통적 시계열 모형은 관측값 $X_t$ 를 세 성분으로 나눕니다.

$$ X_t = T_t + S_t + R_t \quad \text{(가법 모형, Additive)} $$

- $T_t$ (Trend, 추세): 장기적 방향성. 천천히 변하는 성분.
- $S_t$ (Seasonal, 계절성): 일정 주기 $m$ 마다 반복되는 패턴.
- $R_t$ (Residual / Remainder, 잔차): 위 두 성분으로 설명되지 않는 나머지.

### 1.2 왜 분해하는가 — 3가지 목적

**(1) 해석력 확보** — 전체 움직임을 "계절성 효과"와 "추세 효과"로 나눠 설명할 수 있게 됩니다. 예: 월별 매출 증가가 성장 추세 때문인지, 12월 연말 특수 때문인지 구분.

**(2) 이상치 탐지** — 잔차 $R_t$ 를 보면 "평소 패턴에서 벗어난 이상 관측"을 찾기 쉽습니다. 잔차의 z-score 로 이상 일자를 식별.

**(3) 예측 용이성** — 각 성분을 따로 예측한 뒤 다시 합치면 예측 정확도가 올라가는 경우가 많습니다. 특히 계절성이 강한 데이터에서 효과적.

### 1.3 금융 시계열에서는?

**가격(close)** 에는 추세가 강하지만 계절성은 약합니다. **수익률(log_return)** 은 추세·계절성 모두 약하고 잔차가 지배적입니다. 그래서 분해 결과 해석에 주의가 필요합니다 — 이것이 이번 챕터의 핵심 교훈.

## 2. 가법(Additive) vs 승법(Multiplicative)

### 2.1 두 모형의 수식

| 모형 | 식 | 언제 사용하나 |
|---|---|---|
| **가법** | $X_t = T_t + S_t + R_t$ | 계절 변동 크기가 **시간에 무관하게 일정** |
| **승법** | $X_t = T_t \times S_t \times R_t$ | 계절 변동 크기가 **추세에 비례해서 커짐** |

### 2.2 시각적 구분

```
 가법 모형                     승법 모형
 계절 폭 일정                   계절 폭이 점점 커짐
 
 │   ∿∿∿∿                     │      ∿∿∿∿
 │ ∿∿∿∿∿∿∿                     │   ∿∿∿∿∿∿
 │∿∿∿∿∿                       │ ∿∿∿
 └───────                      └───────
```

### 2.3 선택 기준

실무에서 구분은 **시계열을 눈으로 보고** 결정합니다. 또는:

- **로그 변환으로 승법 → 가법 전환**: $\log(X_t) = \log T_t + \log S_t + \log R_t$
- 금융에서 이미 `log_return` 을 쓰는 이유 중 하나가 승법 구조를 가법화하기 위함

> **우리 프로젝트 함의**: `log_return_1d` 는 이미 로그 변환이 들어간 지표이므로 **가법 분해(`model='additive'`)** 를 쓰는 것이 일반적으로 맞습니다.

## 3. 고전적 분해 (Classical Decomposition)

### 3.1 알고리즘

$m$ 을 계절 주기(예: 월별 → 12, 일별 금융 → 5(주) 또는 252(연))라고 할 때:

**Step 1 — 추세 추정**: 중심이동평균(Centered Moving Average)으로 평활.
$$ \hat{T}_t = \frac{1}{m} \sum_{k=-m/2}^{m/2} X_{t+k} $$

**Step 2 — 추세 제거**: $X_t - \hat{T}_t$ (가법) 또는 $X_t / \hat{T}_t$ (승법).

**Step 3 — 계절성 추정**: 같은 계절(예: 같은 월, 같은 요일)의 평균을 뽑아 $\hat{S}_t$ 계산.

**Step 4 — 잔차**: $\hat{R}_t = X_t - \hat{T}_t - \hat{S}_t$ (가법 기준).

### 3.2 장점과 한계

| 장점 | 한계 |
|---|---|
| 구현이 단순, 해석 직관적 | 처음·끝 $m/2$ 개 관측 추세값 결측 |
| 추세/계절을 명시적으로 분리 | 계절성이 **시간에 따라 변하면** 포착 불가 |
| — | 이상치에 취약 (이동평균이 왜곡됨) |

### 3.3 `statsmodels` 의 `seasonal_decompose`

```python
from statsmodels.tsa.seasonal import seasonal_decompose

result = seasonal_decompose(series, model='additive', period=5)
result.trend        # T_t
result.seasonal     # S_t
result.resid        # R_t
result.observed     # X_t (원본)
```

핵심 파라미터:

- `model`: `'additive'` / `'multiplicative'`
- `period`: 계절 주기 $m$. **필수**. 일별 데이터면 5(주간), 21(월간 영업일), 252(연간).
- `extrapolate_trend`: 추세 결측을 선형 외삽으로 채우기. `'freq'` 로 설정하면 대부분 채워줌.

## 4. STL — Seasonal-Trend decomposition using LOESS

### 4.1 왜 STL 이 더 선호되는가

`seasonal_decompose` 는 "고정된 계절성"을 가정합니다. 하지만 현실 데이터는 계절 패턴 자체가 시간에 따라 변하기도 합니다. STL은 이를 **LOESS(Locally Estimated Scatterplot Smoothing)** 로 유연하게 추정합니다.

| 특징 | `seasonal_decompose` | `STL` |
|---|---|---|
| 계절성 | 고정 | **시간가변(time-varying)** |
| 이상치 | 취약 | **robust 옵션** 제공 |
| 추세 평활도 | 이동평균 고정 | LOESS 파라미터로 조절 |
| 결측 처리 | 제한적 | NaN 지원 |
| 계산 속도 | 빠름 | 조금 느림 |

### 4.2 STL 사용법

```python
from statsmodels.tsa.seasonal import STL

stl = STL(series, period=5, robust=True)
result = stl.fit()
result.trend / result.seasonal / result.resid
```

핵심 파라미터:

- `period`: 계절 주기 (필수).
- `seasonal`: LOESS 윈도우 크기. 홀수, 클수록 매끄러운 계절 패턴. 기본 7.
- `trend`: 추세 LOESS 윈도우. 계절보다 크게 잡음.
- `robust=True`: 이상치에 둔감 — 금융 데이터에서 **권장 설정**.

### 4.3 STL 결과 해석의 원칙

1. **계절성 진폭** 이 잔차 표준편차보다 작으면 계절성이 "약하다"고 판단
2. **잔차 ACF** 를 확인 — 잔차에 자기상관이 남아있으면 계절 설명이 부족한 것 (1.3 Ljung-Box 로 검정)
3. **추세가 wiggly(흔들리면)** `trend` 윈도우를 키워야 함 — 노이즈를 추세로 잘못 흡수

## 5. 금융 시계열에서 분해가 "기대만큼 안 되는" 이유

### 5.1 계절성이 약한 이유

금융 자산 가격·수익률에는 전통적 의미의 계절성(예: 월별 반복)이 **거의 없습니다**. 이유:

- **효율적 시장 가설의 약형**: 공개된 계절 패턴은 차익거래로 사라짐
- **거래일 캘린더의 비규칙성**: 주말·공휴일로 인해 주기가 고르지 않음
- **뉴스 기반 움직임**: 거시 이벤트(금리, 실적)가 계절보다 훨씬 큰 영향

따라서 `seasonal_decompose` 의 $S_t$ 는 대부분 **노이즈 수준의 작은 값**으로 나옵니다.

### 5.2 존재하는 "약한" 캘린더 효과

단, 미약하지만 통계적으로 관찰되는 효과는 있습니다.

| 효과 | 내용 | 크기 |
|---|---|---|
| **월요일 효과(Monday Effect)** | 월요일 수익률이 평균적으로 낮음 | 최근 수십 년간 약화 |
| **1월 효과(January Effect)** | 1월 수익률이 타월 대비 높음 (소형주 중심) | 역사적으로 관찰, 최근 약함 |
| **Turn-of-Month** | 월말~월초 수익률 플러스 경향 | 유의하나 작음 |
| **FOMC Drift** | FOMC 이벤트 전후 수익률 패턴 | 존재 |

이들은 계절성이라기보다 "행동 편향" 또는 "제도적 요인"에 가깝습니다.

### 5.3 추세 정의의 모호성

**"주가의 추세"가 무엇인가?** 해석이 관측 윈도우에 따라 완전히 달라집니다.

- 60일 이동평균 → 중기 추세
- 252일 이동평균 → 장기 추세
- HP 필터 λ = 1600 → 경기 주기 개념의 추세

분해 결과 중 **무엇을 "추세"라 부를지는 분석 목적이 정합니다.** 이 모호성이 금융에서 분해를 기계적으로 돌리면 안 되는 이유입니다.

### 5.4 잔차의 이분산(Heteroskedasticity)

1.3 에서 봤듯 수익률 제곱의 ACF 가 강합니다. 즉 잔차 $R_t$ 의 **분산 자체가 시간에 따라 변합니다.** 전통 분해는 잔차를 white noise 로 가정하므로, 금융에서는 잔차의 추가 모형화(GARCH 등)가 필요합니다.

## 6. 실무 대안: HP 필터·Wavelet·Differencing

### 6.1 Hodrick-Prescott (HP) 필터

거시경제에서 추세/경기 분리에 널리 쓰이는 방법. 목적함수:

$$ \min_{\tau_t} \sum (x_t - \tau_t)^2 + \lambda \sum ((\tau_{t+1} - \tau_t) - (\tau_t - \tau_{t-1}))^2 $$

- 첫째 항: 원데이터와 추세의 차이 최소화
- 둘째 항: 추세의 이차차분(굴곡) 페널티 — $\lambda$ 가 클수록 매끄러운 추세

**일별 데이터 권장 $\lambda$**: 통상 $10^4$ ~ $10^6$ 범위에서 실험.

```python
from statsmodels.tsa.filters.hp_filter import hpfilter
cycle, trend = hpfilter(series, lamb=1600)  # 분기 데이터 기준, 일별은 더 큰 값
```

**주의**: HP 필터는 최근 구간에서 추세 추정이 불안정합니다(end-point bias). 실시간 분석에는 주의.

### 6.2 Wavelet decomposition

주파수별로 신호를 분해 → 단기 노이즈와 장기 추세를 동시에 분리 가능. 금융 시계열의 multi-scale 구조 분석에 적합하나 진입 장벽이 높아 이번 챕터에서는 언급만.

### 6.3 Differencing 기반 "분해"

1.2 에서 본 1차차분(= 수익률) 자체가 "추세 제거" 역할을 합니다. 금융 실무에서는 공식적 분해보다 이 접근이 더 일반적입니다:

- 원본 가격: 추세 + 노이즈
- 1차차분(수익률): 주로 노이즈 (+ 약한 자기상관)
- 수익률 제곱: 변동성 구조

이것이 "실질적으로는 이게 분해"인 셈.

## 7. 요일효과 (Day-of-Week Effect) — 가장 자주 시험되는 계절성

### 7.1 검정 방법

1. 각 일자의 요일 부여 (월=0, ..., 금=4)
2. 요일별 수익률 평균 계산
3. **일원 ANOVA** 또는 요일 간 **t-test** 로 평균 차이 검정
   - H₀: 모든 요일 평균이 같다
   - p-value < 0.05 → 요일 효과 존재

```python
df['weekday'] = df.index.dayofweek
groups = [df[df['weekday']==d]['log_return_1d'].dropna() for d in range(5)]
from scipy import stats
f, p = stats.f_oneway(*groups)
```

### 7.2 추가 검증 — 회귀로 요일 더미

더 엄밀한 접근은 요일 더미변수 회귀:

$$ r_t = \alpha + \sum_{d=1}^{4} \beta_d \cdot D_{d,t} + \epsilon_t $$

($D_{d,t}$ 는 "수요일이면 1" 같은 요일 더미) 각 $\beta_d$ 의 유의성을 t-test 로 확인.

### 7.3 주의 — 다중비교

요일 5개를 쌍별 비교하면 **다중비교 문제**가 생깁니다 (10번 검정 시 우연 유의 확률 증가). Bonferroni 나 FDR 보정 필요.

> 1.3 에서 "Ljung-Box 로 여러 lag 을 한꺼번에 omnibus 검정" 했던 논리와 같은 맥락입니다.

## 8. 우리 프로젝트와의 연결

### 8.1 현재 파이프라인 관점

- `close`: 강한 상향 추세 존재 → 분해하면 $T_t$ 가 우세
- `log_return_1d`: 추세·계절 거의 없음 → 잔차가 지배적
- `mom_60d` (현재 피처): 사실상 "이동평균 기반 추세 추출"의 수공 버전
- `vol_20d_ann` (현재 피처): "잔차의 이분산 구조"를 수동으로 포착한 것

### 8.2 시퀀스 모델(GRU) 관점

GRU는 내부 은닉 상태를 통해 다음을 **자동 학습할 수 있습니다**:

1. 저주파 성분(추세) — 긴 범위 의존을 hidden state 에 누적
2. 변동성 구조(이분산) — gate 가 large/small regime 을 구분
3. 일시적 충격(잔차의 큰 값) — reset gate 로 과거 리셋

즉 "분해 → 각 성분 모형화 → 합치기"의 전체 파이프라인을 GRU 하나로 대체할 수 있다는 것이 시퀀스 모델의 매력입니다.

### 8.3 그래도 수공 피처를 섞는 이유 (Hybrid)

완전히 raw 만 넣으면 학습 데이터가 부족할 때 시그널을 놓치기 쉽습니다. 그래서 **도메인 지식으로 만든 피처 + raw 시퀀스** 를 함께 넣는 hybrid 가 실무 표준입니다. 우리 프로젝트에서도 이 접근을 쓸 것.

## 9. 이론 vs 실측 — 이번 챕터에서 확인할 핵심 대조

| 주제 | 교과서 서술 | 금융 일별 데이터 실측 | 함의 |
|---|---|---|---|
| 계절성 | 명확한 주기 반복 | 거의 없음, 진폭 무시 수준 | 분해 해석 제한적 |
