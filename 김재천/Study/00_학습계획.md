## 학습의 큰 그림

### 기존 baseline (XGBoost 버전) — 비교 대상으로 유지

```
입력: X_is.shape = (2310, 17)     ← 행마다 독립된 17개 피처
       ↓
모델: XGBoost                     ← 행 단위 분류 (날짜 순서 무시)
       ↓
출력: 5분위 확률 proba[5]          ← 분류 출력
       ↓
변환: proba[5] → Q, Ω 휴리스틱     ← 연결부
       ↓
B-L: 최적 포트폴리오
```

⚠️ **우리 구현인 COL-BL(=논문 CGL-BL의 Optuna 변형)은 이 구조를 통째로 교체**합니다. 분류 → 회귀, `proba[5]` → 예측 수익률 ŷ, 휴리스틱 변환 → 이론적 View 구성으로 모두 변경.

### 최종 목표 아키텍처 — **COL-BL** (v2.4 확정, 논문 CGL-BL의 Optuna 기반 동등 변형)

**COL-BL** 은 하나의 통합된 end-to-end 시스템입니다. **COL** (CEEMDAN-OLSTM-LSTM)이 각 자산의 **미래 수익률을 회귀 예측**하고, 그 예측값이 **BL View 행렬 (P, Q)** 로 직접 변환되어 포트폴리오 최적화로 이어지는 단일 파이프라인입니다. 논문의 GLSTM(GA로 IMF별 hp 탐색)을 **OLSTM(Optuna/TPE로 동일 목적 수행)** 로 교체했을 뿐, 나머지 구조와 이론적 주장은 CGL-BL과 동일합니다.

```
입력: 개별 자산 i의 일별 수익률 r_i,t   (자산 N개 × 시점 T개)
       │    자산별로 독립 COL 파이프라인 실행
       ↓
╔══════════════════════════════════════════════════════════════╗
║               COL (자산 i별 수익률 예측 파이프라인)             ║
║                                                              ║
║   ① C: CEEMDAN 신호 분해                                      ║
║      r_i,t → IMF_1, IMF_2, …, IMF_n, Residue                 ║
║      (고주파 노이즈 ~ 저주파 추세)                             ║
║                       ↓                                      ║
║   ② O: IMF별 OLSTM (= Optuna-tuned LSTM)                      ║
║      OLSTM_1 ← IMF_1    ┐                                    ║
║      OLSTM_2 ← IMF_2    │ Optuna(TPE)로 IMF마다 hp 자동 탐색  ║
║      ⋮                   │ (논문 기여 ①의 동등 구현)            ║
║      OLSTM_n ← IMF_n    ┘ suggest_int / suggest_float +       ║
║                           MedianPruner로 비용 절감            ║
║                       ↓                                      ║
║      OLSTM 출력: ŷ_1, ŷ_2, …, ŷ_n  (IMF별 수익률 성분 예측)   ║
║                       ↓                                      ║
║   ③ L: LSTM 비선형 앙상블 (meta-learner, 회귀)                 ║
║      ŷ_1,…,ŷ_n → LSTM 결합기                                 ║
║      (단순 합산 대신 비선형 통합 — 논문 기여 ②)                 ║
║                       ↓                                      ║
║   COL 출력: r̂_i,t+1  (자산 i의 미래 수익률 예측, 회귀 값)      ║
╚══════════════════════════════════════════════════════════════╝
       │
       │  모든 자산 i = 1,…,N에 대해 반복 → 예측 벡터 r̂ ∈ ℝ^N
       ↓
┌──────────────────────────────────────────────────────────────┐
║             View 구성기 (논문 기여 ③)                         │
│                                                              │
│   P: asset selection matrix  (어떤 자산에 대한 view인지)     │
│       → 절대적 view면 P = I_N (N×N 단위행렬)                 │
│   Q: expected return vector  = r̂                            │
│   Ω: view uncertainty matrix (예측 분산 / historical resid)  │
│                                                              │
│   → BL View = (P, Q, Ω)                                      │
└──────────────────────────────────────────────────────────────┘
       ↓
┌──────────────────────────────────────────────────────────────┐
│  BL: Black-Litterman                                         │
│      (P, Q, Ω) + 시장 균형 π + 공분산 Σ                       │
│      → 사후분포 μ̂_BL, Σ̂_BL                                   │
│      → 평균-분산 최적화 → 최적 포트폴리오 w*                   │
└──────────────────────────────────────────────────────────────┘
       ↓
   포트폴리오 성과 평가
   (초과수익률 · Sharpe · MDD — 논문과 동일)
```

**핵심 설계 원칙:**
- **회귀 모델**: COL은 **수익률 회귀** (분류 아님). Loss는 MSE 또는 Huber.
- **종속변수 명시**: 각 자산 i에 대해 t+1 수익률 r_{i,t+1}가 타겟.
- **View 자동 생성**: 논문의 핵심 기여 — COL 예측 → P, Q 행렬 구성 → BL 입력. 주관적 투자자 view를 **객관적 모델 예측으로 대체**.
- **3단 모듈화(COL 내부)**: C(분해) / O(per-IMF Optuna-LSTM) / L(비선형 결합) 각각 독립 개발.
- **비교 벤치마크(논문 기준)**: 지수 자체(SSE 50/DJIA) / 시총 가중 / CEEMDAN-LSTM / CEEMDAN-GRU / 단일 LSTM + 우리 프로젝트 기존 XGBoost baseline. (논문의 CGL-BL 수치와도 정성적 비교)
- **검증 프로토콜**: Walk-Forward (논문: SSE 50 9개월, DJIA 1년). 한국 시장 적용 시 별도 기간 설정.

**✅ OLSTM = Optuna-tuned LSTM (v2.4 구현 선택)**: Optuna(TPE sampler)로 LSTM의 hyperparameter(hidden_dim, num_layers, dropout, window_size, lr, batch_size)를 IMF별로 자동 탐색. CEEMDAN이 만든 6~10개 IMF 각각이 주파수 대역이 다르므로 단일 hyperparam으로는 최적화 불가 — 이것이 **논문 기여 ①의 내용**이고, 논문은 GA로 풀었지만 우리는 Optuna로 푼다는 차이만 있음. **기법 동등성**: GA와 TPE 모두 black-box optimization으로 같은 문제를 풀며, 최근 벤치마크에서 TPE가 일반적으로 더 빠르고 코드가 간결함(Bergstra et al. 2011; Akiba et al. 2019).

**📖 입력 데이터 명시 (v2.3 유지)**: COL-BL의 입력은 **각 자산의 일별 수익률 시계열**. 논문은 포트폴리오 최적화이므로 **N개 자산**을 동시에 다루고, **자산 i별로 독립적인 COL 파이프라인**을 실행해 `r̂_i` 를 얻습니다. 이 N개의 예측이 Q 벡터를 이룹니다. 공분산 행렬 Σ는 BL 단계에서 별도로 historical returns로부터 계산.

**⚠️ v2.3 → v2.4 주요 변경 (2026-04-24)**: GLSTM(GA-LSTM) → **OLSTM(Optuna-LSTM)** 로 구현 교체. 이유:
1. GA 수동 구현은 population/selection/crossover/mutation/elitism 코드량이 크고 디버깅 난도가 높음
2. Optuna는 `study.optimize(objective, n_trials=N)` + pruner로 같은 문제를 20~50줄로 해결
3. 2026년 현재 업계 표준이며 learning curve 대비 실무 가치가 큼
4. 논문의 **기여 ① (per-IMF 최적화)** 의 과학적 주장은 보존됨 — 구현 수단만 변경
5. Week 4.3 실습에서 Optuna vs GA vs Random Search 비용·성능 비교를 통해 선택 정당화

    ```python
    def objective(trial):

        return train_one_individual(params, ...)  # best_val_loss 반환
    ```
    → 3.4 §16에서 설계한 경량 학습 패턴이 바로 Optuna의 objective가 됨 (설계 의도 실현)

    - IMF당 `n_trials` 수: 논문 GA 대비 1/3~1/5 (예: 30~50 trials)

### 6주차 — 비선형 앙상블 + View 생성 + Black-Litterman 통합 ⭐ (v2.3 전면 수정)

> 목표: IMF별 **OLSTM(Optuna-tuned LSTM)** 예측 → LSTM 비선형 앙상블 → 자산별 수익률 예측 → **BL View 행렬 (P, Q, Ω) 직접 구성** → Black-Litterman 최적화 → 논문 벤치마크 대비 Walk-Forward 성능 실측

- [ ] **6.1 앙상블 기초 — 선형 vs 비선형**
  - 단순 평균 / 가중 평균 (선형) — IMF별 예측의 직관적 재조립(reconstruction)
  - 스태킹 (meta-learner) — IMF 예측을 피처로 취급
  - Attention-weighted 결합 (학습형 가중)
  - **왜 비선형이 논문의 기여 ②인가** — IMF마다 noise/trend 특성이 다르고 cross-IMF interaction이 존재 → 단순 합산이 놓치는 구조적 신호를 meta-learner가 학습
- [ ] **6.2 LSTM 비선형 앙상블 구현 (회귀 meta-learner)** (v2.3 수정: 분류→회귀)
  - **입력 구성**: 시점 t에서 IMF별 OLSTM 예측 (ŷ_1, ŷ_2, …, ŷ_n, r̂esidue) 를 **시퀀스 또는 벡터**로 결합
    - 옵션 A: 시퀀스 형태 (T, n+1) → 결합기 LSTM → r̂
    - 옵션 B: 벡터 형태 (n+1,) → MLP 결합기 → r̂
  - **출력**: 자산별 수익률 예측 r̂_{i, t+1} (회귀 스칼라)
  - **Loss**: MSE 또는 Huber (4.5 설계 재사용)
  - 앙상블 학습 시 과적합 위험과 대응 (Dropout, weight decay, validation monitoring)
  - **실측 검증**: 단순 합산 (Σ ŷ_k) vs 비선형 앙상블 성능 비교 — 논문 기여 ② 재현
- [ ] **6.3 Walk-Forward + CEEMDAN 통합**
  - **누수 방지 핵심**: 각 fold마다 CEEMDAN 재계산 (미래 정보로 과거 IMF를 재구성하지 않기)
  - **Optuna 튜닝 시점** (v2.4 신규 고려): 초기 training 구간에서 1회만 hp 탐색 → best_params 고정 후 모든 fold에서 재사용 (매 fold마다 재튜닝은 look-ahead 금지)
  - IMF 수가 fold마다 다를 때 처리 (논문 권장: 고정 n 또는 zero-padding)
  - 자산 N개 × fold마다 COL 파이프라인 실행 → 계산 비용 추정
  - 논문 기준 Walk-Forward 윈도우: **SSE 50 = 9개월 / DJIA = 1년** (한국시장 적용 시 별도 결정)
- [ ] **6.4 View 행렬 생성 (COL → BL 연결)** ⭐ (v2.3 신규 — 논문 기여 ③ 재현)
  - **Q 구성**: 자산 i의 COL 예측 r̂_i → Q = (r̂_1, r̂_2, …, r̂_N)^T ∈ ℝ^N
  - **P 구성 (asset selection matrix)**:
    - 절대적 view: P = I_N (N×N 단위행렬) — 각 view가 개별 자산에 대응
    - 상대적 view: P_kj - P_kl = 1 (k번째 view가 자산 j와 l의 수익률 차이) — 선택적 확장
  - **Ω 구성 (view uncertainty)** — 논문 핵심 설계 포인트:
    - 옵션 ①: 예측 모델의 **validation residual 분산 σ̂²_i** 를 대각 원소로 사용 → Ω = diag(σ̂²_1, …, σ̂²_N)
    - 옵션 ②: Meucci-style — Ω = τ · P Σ P^T / confidence (τ: scaling, Σ: historical covariance)
    - 논문이 쓰는 방식 그대로 재현 (예측 모델 자체의 불확실성 활용)
  - **구현**: `build_view_matrices(col_preds, residual_vars, N) -> (P, Q, Omega)`
  - **검증**: 입력 차원 일치 체크 (P: k×N, Q: k×1, Ω: k×k)
- [ ] **6.5 Black-Litterman 사후분포 + 포트폴리오 최적화** (v2.3: 기존 BL 코드 재사용)
  - BL 사후평균: μ̂_BL = [(τΣ)^{-1} + P^T Ω^{-1} P]^{-1} [(τΣ)^{-1} π + P^T Ω^{-1} Q]
  - BL 사후공분산: Σ̂_BL
  - 평균-분산 최적화 (Markowitz) → 최적 가중치 w*
  - 제약 조건: long-only / weight bound / turnover 제약 (논문 설정 따라)
