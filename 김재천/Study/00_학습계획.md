## 학습의 큰 그림

### 기존 baseline (XGBoost 버전) — 비교 대상으로 유지

```
입력: X_is.shape = (2310, 17)     ← 행마다 독립된 17개 피처
       ↓
모델: XGBoost                     ← 행 단위 분류 (날짜 순서 무시)
       ↓
출력: 5분위 확률 proba[5]          ← 분류 출력
       ↓
변환: proba[5] → Q, Ω 휴리스틱     ← 연결부
       ↓
B-L: 최적 포트폴리오
```

⚠️ **논문의 CGL-BL은 이 구조를 통째로 교체**합니다. 분류 → 회귀, `proba[5]` → 예측 수익률 ŷ, 휴리스틱 변환 → 이론적 View 구성으로 모두 변경.

### 최종 목표 아키텍처 — **CGL-BL** (v2.3 확정, 논문 기준)

**CGL-BL** 은 하나의 통합된 end-to-end 시스템입니다. **CGL** (CEEMDAN-GLSTM-LSTM)이 각 자산의 **미래 수익률을 회귀 예측**하고, 그 예측값이 **BL View 행렬 (P, Q)** 로 직접 변환되어 포트폴리오 최적화로 이어지는 단일 파이프라인입니다.

```
입력: 개별 자산 i의 일별 수익률 r_i,t   (자산 N개 × 시점 T개)
       │    자산별로 독립 CGL 파이프라인 실행
       ↓
╔══════════════════════════════════════════════════════════════╗
║               CGL (자산 i별 수익률 예측 파이프라인)             ║
║                                                              ║
║   ① C: CEEMDAN 신호 분해                                      ║
║      r_i,t → IMF_1, IMF_2, …, IMF_n, Residue                 ║
║      (고주파 노이즈 ~ 저주파 추세)                             ║
║                       ↓                                      ║
║   ② G: IMF별 GLSTM (= GA + LSTM)                              ║
║      GLSTM_1 ← IMF_1    ┐                                    ║
║      GLSTM_2 ← IMF_2    │ GA로 IMF마다 hyperparam 자동 탐색   ║
║      ⋮                   │ (논문 기여 ①)                       ║
║      GLSTM_n ← IMF_n    ┘                                    ║
║                       ↓                                      ║
║      GLSTM 출력: ŷ_1, ŷ_2, …, ŷ_n  (IMF별 수익률 성분 예측)   ║
║                       ↓                                      ║
║   ③ L: LSTM 비선형 앙상블                                     ║
║      ŷ_1,…,ŷ_n → LSTM 결합기                                 ║
║      (단순 합산 대신 비선형 통합 — 논문 기여 ②)                 ║
║                       ↓                                      ║
║   CGL 출력: r̂_i,t+1  (자산 i의 미래 수익률 예측, 회귀 값)      ║
╚══════════════════════════════════════════════════════════════╝
       │
       │  모든 자산 i = 1,…,N에 대해 반복 → 예측 벡터 r̂ ∈ ℝ^N
       ↓
┌──────────────────────────────────────────────────────────────┐
║             View 구성기 (논문 기여 ③)                         │
│                                                              │
│   P: asset selection matrix  (어떤 자산에 대한 view인지)     │
│       → 절대적 view면 P = I_N (N×N 단위행렬)                 │
│   Q: expected return vector  = r̂                            │
│   Ω: view uncertainty matrix (예측 분산 / historical resid)  │
│                                                              │
│   → BL View = (P, Q, Ω)                                      │
└──────────────────────────────────────────────────────────────┘
       ↓
┌──────────────────────────────────────────────────────────────┐
│  BL: Black-Litterman                                         │
│      (P, Q, Ω) + 시장 균형 π + 공분산 Σ                       │
│      → 사후분포 μ̂_BL, Σ̂_BL                                   │
│      → 평균-분산 최적화 → 최적 포트폴리오 w*                   │
└──────────────────────────────────────────────────────────────┘
       ↓
   포트폴리오 성과 평가
   (초과수익률 · Sharpe · MDD — 논문과 동일)
```

**핵심 설계 원칙:**
- **회귀 모델**: CGL은 **수익률 회귀** (분류 아님). Loss는 MSE 또는 Huber.
- **종속변수 명시**: 각 자산 i에 대해 t+1 수익률 r_{i,t+1}가 타겟.
- **View 자동 생성**: 논문의 핵심 기여 — CGL 예측 → P, Q 행렬 구성 → BL 입력. 주관적 투자자 view를 **객관적 모델 예측으로 대체**.
- **3단 모듈화(CGL 내부)**: C(분해) / G(per-IMF 예측) / L(비선형 결합) 각각 독립 개발.
- **비교 벤치마크(논문 기준)**: 지수 자체(SSE 50/DJIA) / 시총 가중 / CEEMDAN-LSTM / CEEMDAN-GRU / 단일 LSTM + 우리 프로젝트 기존 XGBoost baseline.
- **검증 프로토콜**: Walk-Forward (논문: SSE 50 9개월, DJIA 1년). 한국 시장 적용 시 별도 기간 설정.

**✅ GLSTM = Genetic Algorithm + LSTM (논문 표기 그대로)**: 유전 알고리즘으로 LSTM의 hyperparameter(hidden_dim, num_layers, dropout, window_size, lr, batch_size)를 IMF별로 자동 탐색. CEEMDAN이 만든 6~10개 IMF 각각이 주파수 대역이 다르므로 단일 hyperparam으로는 최적화 불가 — 이것이 논문 기여 ①의 핵심.

**📖 입력 데이터 명시 (v2.3)**: CGL-BL의 입력은 **각 자산의 일별 수익률 시계열**. 논문은 포트폴리오 최적화이므로 **N개 자산**을 동시에 다루고, **자산 i별로 독립적인 CGL 파이프라인**을 실행해 `r̂_i` 를 얻습니다. 이 N개의 예측이 Q 벡터를 이룹니다. 공분산 행렬 Σ는 BL 단계에서 별도로 historical returns로부터 계산.

**⚠️ v2.2 → v2.3 주요 변경 (2026-04-24)**: 제가 v2.2까지 가정했던 "CGL 출력 = proba[5]" 는 **잘못된 가정**이었고, 논문 실제 구조는 **CGL 출력 = 수익률 회귀값 → BL View 직접 생성**. 이에 따라 기존 "XGBoost와 동일 인터페이스 유지" 원칙은 **폐기**되고, XGBoost 버전은 **비교 벤치마크 baseline**으로 강등됩니다.

  - **실습**: 동일 과제에서 단방향 vs BiLSTM 비교
- [ ] **4.3 LSTM 파생 III — GLSTM = Genetic Algorithm + LSTM** ⭐
  - **핵심 아이디어**: GA로 LSTM의 hyperparameter를 자동 탐색 (IMF별 개별 최적화가 필요한 CEEMDAN 구조와 필연적으로 맞물림)
  - **탐색 대상 (염색체 구성)**:
    - `hidden_dim` ∈ {32, 64, 128, 256}
    - `num_layers` ∈ {1, 2, 3}
    - `dropout` ∈ [0.0, 0.5]
    - `window_size` ∈ {20, 40, 60, 90}
    - `learning_rate` ∈ [1e-4, 5e-3] (log-scale)
    - `batch_size` ∈ {32, 64, 128}
  - **GA 구성 요소 학습**:
    - 초기 population (random sampling)
    - Fitness function (validation loss 또는 금융용 Sharpe ratio)
    - Selection (tournament / roulette)
    - Crossover (single-point / uniform)
    - Mutation (gaussian / uniform)
    - Elitism (상위 개체 보존)
  - **실전 고려**:
    - 계산 비용 `population × generations × epochs` 폭증 → **각 학습은 조기 종료 (5~10 epoch)**
    - GA는 "탐색용" — 찾은 hyperparam은 Walk-Forward 전체에서 고정 재사용
    - **대안 벤치마크**: Optuna(TPE) / CMA-ES / Random Search 와 성능·비용 비교 필수
  - **라이브러리 선택**: DEAP vs PyGAD vs 직접 구현
  - **실습**:
    1. PyGAD로 간단한 GA 개념 실습 (1차원 fitness)
    2. LSTM hyperparameter tuning을 GA로 수행 (MSFT 단일 IMF 기준)
    3. GA vs Random Search 비용·성능 비교
    4. Optuna(TPE)와의 동일 예산 하 성능 대결
  - **최종 아키텍처 연결**: Week 5의 CEEMDAN IMF 각각에 이 GA-LSTM 탐색을 적용해 IMF별 전용 모델 6~10개 생성 → Week 6 비선형 앙상블의 base learner로 투입
- [ ] **4.4 Attention-LSTM (선택)**
  - 시점별 중요도 학습
  - 금융 시계열에서 해석 가능성 제공
  - 최종 앙상블의 결합기로 활용 가능
- [ ] **4.5 회귀 헤드 — (B, T, H) → (B, 1) 수익률 예측** ⭐ (v2.3 수정: 분류→회귀)
  - **CGL-BL은 회귀 모델**: 타겟은 t+1 수익률 r_{t+1} ∈ ℝ (1차원 스칼라)
  - 출력 헤드 설계:
    - 마지막 시점 hidden h_T → `Linear(H, 1)` → ŷ
    - 평균 풀링 (T 전체를 평균) vs attention pooling 비교
  - **Loss 선택**:
    - `nn.MSELoss` — 기본 회귀 손실, 큰 오차에 큰 penalty (outlier 민감)
    - `nn.HuberLoss(delta=…)` — |ε|>δ 구간에서 L1로 전환, outlier robust (금융 데이터 권장)
    - `nn.L1Loss` — MAE, robust하지만 미분 특이점
  - **평가 지표 (논문 기준)**: RMSE, MAE — 회귀 품질 / 방향성 정확도(hit ratio) — 경제적 유용성
  - **스케일 주의**: 수익률은 보통 |r| < 0.1 → standardize 후 학습하고 예측 시 역변환
  - **View 생성 준비**: 출력 ŷ와 validation residual 분산 σ̂² 를 기록 — 6주차에서 Ω 행렬 구성에 사용
  - ⚠️ **v2.2 폐기**: 기존 계획의 "5분위 확률 proba[5] + CrossEntropyLoss + calibration"은 XGBoost 호환용 가정이었으므로 CGL-BL에서는 **사용하지 않음**. proba[5]는 XGBoost baseline 재현 시에만 유지.

### 5주차 — CEEMDAN 신호 분해 ⭐ (신규 추가)

> 목표: EMD → EEMD → CEEMDAN 계열을 이해하고, MSFT 수익률을 IMF로 분해해 각 대역별로 학습시키는 파이프라인 구축

- [ ] **5.1 EMD (Empirical Mode Decomposition) 기초**
  - Hilbert-Huang Transform의 출발점
  - Sifting process — 극값·포락선·IMF 추출 알고리즘
  - IMF(Intrinsic Mode Function)의 정의와 성질
  - **한계**: mode mixing 문제
- [ ] **5.2 EEMD (Ensemble EMD) — 노이즈로 mode mixing 해결**
  - 화이트 노이즈 여러 번 주입 후 평균
  - 장점 vs 단점 (residual noise 문제)
- [ ] **5.3 CEEMDAN — Complete Ensemble EMD with Adaptive Noise**
  - EEMD의 단점 해결: 각 단계마다 adaptive noise 추가
  - IMF 완전 복원성 (reconstruction error ≈ 0)
  - **핵심 실험**: MSFT 수익률을 IMF로 분해 후 원본과 합산 일치 검증
  - Python 라이브러리: `PyEMD`
- [ ] **5.4 IMF 해석 — 금융 시계열 맥락**
  - 고주파 IMF = 노이즈 / 고빈도 변동
  - 저주파 IMF = 중장기 추세
  - Residue = 장기 추세
  - **Fourier 스펙트럼 vs IMF 스펙트럼** 비교
- [ ] **5.5 분해-학습-재조립 파이프라인**
  - Walk-Forward에서 **CEEMDAN을 언제 수행하는가** (look-ahead 방지 중요)
  - 각 IMF의 정상성 검정 (1.2 복습)
  - IMF별 LSTM 학습 전략 (hyperparam 차별화)

### 6주차 — 비선형 앙상블 + View 생성 + Black-Litterman 통합 ⭐ (v2.3 전면 수정)

> 목표: IMF별 GLSTM 예측 → LSTM 비선형 앙상블 → 자산별 수익률 예측 → **BL View 행렬 (P, Q, Ω) 직접 구성** → Black-Litterman 최적화 → 논문 벤치마크 대비 Walk-Forward 성능 실측

- [ ] **6.1 앙상블 기초 — 선형 vs 비선형**
  - 단순 평균 / 가중 평균 (선형) — IMF별 예측의 직관적 재조립(reconstruction)
  - 스태킹 (meta-learner) — IMF 예측을 피처로 취급
  - Attention-weighted 결합 (학습형 가중)
  - **왜 비선형이 논문의 기여 ②인가** — IMF마다 noise/trend 특성이 다르고 cross-IMF interaction이 존재 → 단순 합산이 놓치는 구조적 신호를 meta-learner가 학습
- [ ] **6.2 LSTM 비선형 앙상블 구현 (회귀 meta-learner)** (v2.3 수정: 분류→회귀)
  - **입력 구성**: 시점 t에서 IMF별 GLSTM 예측 (ŷ_1, ŷ_2, …, ŷ_n, r̂esidue) 를 **시퀀스 또는 벡터**로 결합
    - 옵션 A: 시퀀스 형태 (T, n+1) → 결합기 LSTM → r̂
    - 옵션 B: 벡터 형태 (n+1,) → MLP 결합기 → r̂
  - **출력**: 자산별 수익률 예측 r̂_{i, t+1} (회귀 스칼라)
  - **Loss**: MSE 또는 Huber (4.5 설계 재사용)
  - 앙상블 학습 시 과적합 위험과 대응 (Dropout, weight decay, validation monitoring)
  - **실측 검증**: 단순 합산 (Σ ŷ_k) vs 비선형 앙상블 성능 비교 — 논문 기여 ② 재현
- [ ] **6.3 Walk-Forward + CEEMDAN 통합**
  - **누수 방지 핵심**: 각 fold마다 CEEMDAN 재계산 (미래 정보로 과거 IMF를 재구성하지 않기)
  - IMF 수가 fold마다 다를 때 처리 (논문 권장: 고정 n 또는 zero-padding)
  - 자산 N개 × fold마다 CGL 파이프라인 실행 → 계산 비용 추정
  - 논문 기준 Walk-Forward 윈도우: **SSE 50 = 9개월 / DJIA = 1년** (한국시장 적용 시 별도 결정)
- [ ] **6.4 View 행렬 생성 (CGL → BL 연결)** ⭐ (v2.3 신규 — 논문 기여 ③ 재현)
  - **Q 구성**: 자산 i의 CGL 예측 r̂_i → Q = (r̂_1, r̂_2, …, r̂_N)^T ∈ ℝ^N
  - **P 구성 (asset selection matrix)**:
    - 절대적 view: P = I_N (N×N 단위행렬) — 각 view가 개별 자산에 대응
    - 상대적 view: P_kj - P_kl = 1 (k번째 view가 자산 j와 l의 수익률 차이) — 선택적 확장
  - **Ω 구성 (view uncertainty)** — 논문 핵심 설계 포인트:
    - 옵션 ①: 예측 모델의 **validation residual 분산 σ̂²_i** 를 대각 원소로 사용 → Ω = diag(σ̂²_1, …, σ̂²_N)
    - 옵션 ②: Meucci-style — Ω = τ · P Σ P^T / confidence (τ: scaling, Σ: historical covariance)
    - 논문이 쓰는 방식 그대로 재현 (예측 모델 자체의 불확실성 활용)
  - **구현**: `build_view_matrices(cgl_preds, residual_vars, N) -> (P, Q, Omega)`
  - **검증**: 입력 차원 일치 체크 (P: k×N, Q: k×1, Ω: k×k)
- [ ] **6.5 Black-Litterman 사후분포 + 포트폴리오 최적화** (v2.3: 기존 BL 코드 재사용)
  - BL 사후평균: μ̂_BL = [(τΣ)^{-1} + P^T Ω^{-1} P]^{-1} [(τΣ)^{-1} π + P^T Ω^{-1} Q]
  - BL 사후공분산: Σ̂_BL
  - 평균-분산 최적화 (Markowitz) → 최적 가중치 w*
  - 제약 조건: long-only / weight bound / turnover 제약 (논문 설정 따라)
- [ ] **6.6 성능 비교 — 논문 벤치마크 재현** ⭐ (v2.3 재정렬)
  - **논문 기준 벤치마크**:
    1. 시장 지수 자체 (SSE 50 / DJIA)
    2. 시가총액 가중 포트폴리오 (equal-weight or cap-weight)
    3. CEEMDAN-LSTM + BL (GA 없음)
    4. CEEMDAN-GRU + BL
    5. 단일 LSTM + BL (CEEMDAN 없음)
    6. **제안 모델 CGL-BL**
  - **우리 프로젝트 추가 벤치마크**:
    7. 기존 XGBoost + BL (원래 baseline)
  - **평가 지표 (논문 기준)**:
    - Excess return (초과수익률, 지수 대비)
    - Sharpe ratio
    - Maximum Drawdown (MDD)
    - (보조) RMSE / MAE / Hit ratio — CGL 예측 품질 자체
  - **정량적 개선 폭** 표로 보고 (bootstrap CI 포함 권장)
- [ ] **6.7 리스크와 한계**
  - CEEMDAN의 계산 복잡도 (Walk-Forward × 자산 N × GA 탐색 → 연산량 폭증)
  - 앙상블의 해석 어려움 (회귀 메타러너의 gradient-based attribution)
  - Overfitting 진단 체크리스트 (train/val/test 곡선, residual 분포)
  - View 과신 문제: Ω가 너무 작으면 BL이 CGL 예측에 과도하게 끌려감 → τ 튜닝 필요
  - 실전 투입 시 주의점: 거래비용, 슬리피지, rebalancing 주기
