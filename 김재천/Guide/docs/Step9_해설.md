## 1. 배경과 목적

### 🎯 문제 설정

**기존 연구 (v3)의 한계**:
- Step 7: "경보 기반 동적 리밸런싱"을 시연했으나 Equal Weight(1/30) 기준
- 분명한 효과가 있었으나 **MV 최적화와 결합 시 효과가 유지되는가?** 미검증

**Step 9의 목적**:
> **"MV 최적화라는 정식 baseline 위에서 대안데이터 2경로(경보, 레짐 Σ)가 실제로 가치를 더하는지 엄밀히 검증"**

### 🔬 실험 설계의 핵심: 4개 모드

**연구 방법론 비유 - 약효 실험**:
- **M0 (플라세보)**: 약 없이 기본만 (순수 MV)
- **M1 (약 A만)**: 경로 1만 추가
- **M2 (약 B만)**: 경로 2만 추가
- **M3 (약 A+B)**: 두 약 조합

→ 각 경로의 **순수 기여도**와 **상호작용**을 분리 측정 가능

## 3. 4개 모드 의미

### 🧭 각 모드의 동작 방식

| 모드 | Σ 선택 | 일별 경보 대응 | 의미 |
|------|-------|-----------|------|
| **M0** | Σ_all | ❌ 안 함 | 순수 MV (Step 4 재현) |
| **M1** | Σ_all | ✅ 주식 축소 | **경로 1만** |
| **M2** | Σ_stable/Σ_crisis (월별) | ❌ 안 함 | **경로 2만** |
| **M3** | Σ_stable/Σ_crisis (월별) | ✅ 주식 축소 | **통합** |

### 🚗 자동차 안전장치 비유

- **M0**: 기본 차량 (안전장치 없음)
- **M1**: 에어백 (사고 발생 시 즉각 작동) — 경로 1
- **M2**: 자동 감속 시스템 (빗길 등 위험 환경 감지) — 경로 2
- **M3**: 에어백 + 감속 시스템

### 📊 구체적 비교 예시

**2020년 3월 (COVID 급락) 대응**:

| 모드 | 2월 말 ~ 3월 중순 |
|------|----------------|
| M0 | 1월 세팅한 비중 그대로 유지 → 주식 급락 모두 맞음 |
| M1 | 3월 초 L2 경보 발동 → **즉시 주식 35% 축소**, 채권·금으로 이동 |
| M2 | 3월 시작 시 경보 확인 → Σ_crisis 사용하여 재최적화, 1일 후 비중 적용 |
| M3 | M1 + M2 동시 (3월 초 전환 + 매일 경보) |

## 4. 시뮬레이션 구조

### 🔧 Walk-Forward 프레임워크

```
[전체 구조]
  - 31개 윈도우 (Step 8 산출물 활용)
  - 각 윈도우: IS 24개월 (학습) + OOS 3개월 (운용)
  - OOS 3개월 동안 일별 수익률 계산

[각 윈도우에서]
  1. IS 데이터로 μ (기대수익률), Σ 추정
  2. MV 최적화 → 30개 자산 비중 산출
  3. OOS 3개월 보유 + 일별 동작:
     - M0, M2: 고정 유지 (경보 무반응)
     - M1, M3: 매일 경보 체크, 필요 시 주식 축소
  4. 거래비용 차감 (편도 15bps, 턴오버 기반)
```

### 🎛️ 거래비용 처리 방식

**turnover × 15bps** 공식:
- turnover = Σ|Δw_i| (비중 변화의 절댓값 합)
- `|Δw|`는 매수+매도를 모두 포함 → **사실상 왕복 30bps 등가**
- 이 정도가 실전 소매 투자자의 현실적 수준

### 📐 시뮬레이션 수

```
기본: 4 모드 × 4 성향 × 4 Config = 64
효율화 적용:
  - M0는 경보 무관 → 4 성향만 실제 실행, 4 Config에 복제
  - 결과: 4(M0) + 48(M1/M2/M3) = 52 실제 실행
+ 벤치마크: EW, SPY, 60/40 = 3
= 총 67 전략 (저장 키 기준)
```

### ⚙️ v4.1 주요 변경

**경로 2 Σ 선택 방식**:
- **v4.0 초안**: OOS 시작일 1회 확인 → 3개월 동안 고정
- **v4.1 재설계**: OOS를 월별로 분할 → 매월 재확인 → **동적 Σ 전환**

→ 이 재설계가 피드백 반영의 핵심. 실패 결과는 아래 참조.

## 5. 판단 과정

### 🤔 주요 결정 사항

#### 결정 1: Baseline을 EW → MV로 변경

**v3 (Step 7)**: Equal Weight 1/30이 baseline
**v4 (Step 9)**: MV 최적화 baseline

**이유**:
- MV는 실전 기관 투자자의 표준
- Step 4의 엄격한 WF MV를 계승하여 일관성 확보
- EW로는 "대안데이터의 순수 효과"만 측정되나, MV는 "실전 가능한 전략의 추가 가치" 측정

#### 결정 2: 경로 2 재설계 (v4.1 핵심)

**발견 과정**:
1. v4.0 초안: OOS 시작일 1회 Σ 선택
2. 피드백: "3개월 고정 Σ는 intra-quarter 변화 미포착"
3. v4.1: **월 단위 Σ 재전환**으로 재설계

**구현**:
```python
for month in oos_months:  # OOS 3개월을 월별 분할
    alert_at_month = alert_series[month_start]
    Σ_used = Σ_crisis if alert_at_month >= 2 else Σ_stable
    weights = MV_optimize(μ, Σ_used)
```

**결과** (충격적):
- M2 Sharpe: 0.810 → **0.749** (악화)
- M3 Sharpe: 0.993 → **0.920** (악화)
- M3 - M1: +0.025 → **-0.040** (역효과 확인)

**발견**: M0는 경보 무관이므로 4 Config가 동일 결과
**수정**: 성향별 1번 실행 후 4 Config에 복제
**절감**: 전체 실행의 19% 감소

#### 결정 4: 복원 로직 문서 vs 코드 동기화

**초기 문서**: "3~5일 점진적 복원"

## 8. 경로 2 실패 사후분석

### 🚨 왜 경로 2는 실패했는가?

#### 원인 1: Σ 차이가 MV 최적화에서 충분히 반영 안 됨

**기술적 이유**:
- Ledoit-Wolf 수축이 Σ_stable과 Σ_crisis를 유사하게 보정
- 30개 자산 분산투자 효과로 공분산 원소 차이가 희석
- 결과적으로 최적화된 비중 차이가 작음

#### 원인 2: 재최적화 비용이 이론적 이득 초과

**일상 비유 - 택시 환승**:
- 목적지에 좀 더 빨리 가려고 중간에 계속 택시 갈아탐
- 매번 정거장에서 환승 비용 발생
- **결국 한 택시로 끝까지 가는 게 더 빠름**

**우리 상황**:
- 월마다 Σ 전환 → MV 재최적화 → 비중 변동 증가
- 매번 거래비용 발생
- **누적 비용이 레짐 포착 이득 초과**

#### 원인 3: 경로 1과 정보 중복

**비유 - 두 명의 운전자가 같은 신호 보고 각자 반응**:
- 경로 1: 경보 레벨 보고 즉시 주식 축소
- 경로 2: 같은 경보 레벨 보고 Σ 전환해서 MV가 비슷한 결론 도출
- **동일한 정보를 두 번 반영 → 과도한 조정 → 혼란**

### 📜 학술적 교훈 3가지

1. **이론적 정교함 ≠ 실증적 효과**
   - Σ_stable/Σ_crisis 분리 추정은 이론적으로 합리적
   - 그러나 실제 데이터에서 효과 입증 실패

2. **단순성의 가치 (Occam's Razor)**
   - 경로 1의 즉시 반응(일별 주식 축소)이 복잡한 공분산 전환 능가
   - "더 똑똑한 모델 = 더 좋은 결과"라는 가정 기각

3. **비용 현실성의 중요성**
   - 재최적화 빈도 증가 → 거래비용 폭증 → 이득 상쇄
   - 이론 분석 시 거래비용 가정이 결론 좌우

### 🚀 v5 대안 제안

- **레짐 조건부 자산 배분 제약**: Crisis 시 `max_equity`를 동적 축소
- **레짐 조건부 Risk Budgeting**: 자산군별 VaR 한도 조정
- **Σ 전환이 아닌 μ 전환**: 기대수익률만 레짐별 다르게 설정

## 9. FAQ

### ❓ Q1. M1이 최우수인데 왜 경로 2를 구현했나요?

**A**: 구현 전에는 경로 2가 효과 있을지 모르는 상태였습니다. 과학적 방법론은 "이론 → 가설 → 실증 검증"이며, 실증 결과가 음성이어도 **가치 있는 발견**입니다. 이를 통해 v5 설계를 개선할 수 있습니다.

### ❓ Q2. Sharpe 1.064는 좋은 수치인가요?

**A**: 맥락 중요:
- 헤지펀드 평균 Sharpe: 0.5~0.8 정도
- S&P 500 장기 Sharpe: 약 0.5
- **Sharpe 1.0 이상은 상당히 우수한 수준**
- 다만 **거래비용 30bps 가정 시** 이 수치가 떨어질 수 있음 (민감도 분석 필요)

### ❓ Q3. MDD -15.53%는 실전에서 감내 가능한가요?

**A**: 상대 비교:
- SPY 단독: MDD -33.7% (2배 더 나쁨)
- 60/40: MDD -21.7% (1.4배 나쁨)
- **M1_보수형_ALERT_B가 하방 통제 최강**
- 다만 개인 성향에 따라 다를 수 있음 (중립형 이상은 MDD -20% 이상)

### ❓ Q4. 경보 Config B와 C 중 어떤 게 나은가요?

**A**: **미묘한 차이**:
- Config B (VIX + Contango): Sharpe 1.064, MDD -15.53%
- Config C (7지표 복합): Sharpe 1.072, MDD -16.97%
- **Sharpe는 C가 약간 높음, MDD는 B가 낮음**
- Multi-criteria 종합 시 B가 근소 우세 (하방 통제 중시)

### ❓ Q5. 왜 공격형보다 보수형이 좋게 나오나요?

**A**: Sharpe (위험 조정 수익률) 기준이기 때문:
- 공격형 M1: Total +428% (높음), 하지만 Sharpe 0.97 (낮음)
- 보수형 M1: Total +151% (낮음), Sharpe 1.06 (높음)
- → **같은 위험당 얻는 수익률**로는 보수형이 효율적
- 총수익률 극대화가 목표면 공격형 선택 가능

### ❓ Q6. 실전에서 이 결과를 믿고 운용해도 되나요?

**A**: **여러 한계 인식 필요**:
- 2018~2025 특이 기간 (COVID, 2022 긴축 포함)
- 미국 시장 단독
- 거래비용 15bps 가정 (실전 환경 따라 다름)
- Bonferroni 보정 후 FDR 유의율 경계선
- → **참고로 활용하되, 소액 먼저 시도 후 확대** 권장
