## 1. 배경과 목적

### 🎯 Step 3의 한계

Step 3은 10년 전체 데이터로 **"이론적 최적 비중"**을 계산했으나:
- 2025년 시점 투자자가 2016년에 이 비중을 알 수 없음 (미래 정보)

## 2. 사전 지식

### 📚 용어 사전

| 용어 | 쉬운 설명 |
|------|---------|
| **Walk-Forward** | "과거로 학습, 미래로 테스트, 슬라이드" 반복 방식 |
| **IS (In-Sample)** | 학습용 과거 데이터 (24개월) |
| **OOS (Out-of-Sample)** | 검증용 "미래" 데이터 (3개월) |
| **Look-ahead bias** | 미래 정보를 사용하여 성과를 부풀리는 오류 |
| **분기 리밸런싱** | 3개월마다 비중 재조정 |
| **턴오버** | 리밸런싱 시 바뀌는 비중 총합 |
| **Sharpe Ratio** | 연율수익률 / 연율변동성 |
| **MDD** | Maximum Drawdown, 최대 낙폭 |

### 🎓 Walk-Forward의 원리

**학생 비유**:
- **부정행위 (look-ahead)**: 2025 수능에 2024 학습 + **2025 수능 답지 미리** 봐서 풀기
- **진짜 실력 (Walk-Forward)**: 2024까지만 공부 → 2025 수능 풀이

**우리 설정**:
```
IS: 2016-01 ~ 2017-12 (24개월 학습)  →  OOS: 2018-01 ~ 2018-03 (3개월 운용)
    ↓ 슬라이드
IS: 2016-04 ~ 2018-03 (24개월)       →  OOS: 2018-04 ~ 2018-06
    ↓ 슬라이드
  ... (31회 반복) ...

IS: 2023-10 ~ 2025-09                →  OOS: 2025-10 ~ 2025-12 (마지막)
```

## 3. 진행 과정

### 🔧 WF 주기별 동작

```
[매 윈도우에서]
1. IS 24개월 데이터 추출
2. 기대수익률 μ = mean(IS 일별수익률) × 252
3. 공분산 Σ = Ledoit-Wolf.fit(IS 일별수익률) × 252
4. MV 최적화 (성향 제약 반영) → 30자산 비중
5. OOS 3개월 동안 비중 유지
6. 일별 수익률 계산 + 거래비용 차감
7. 다음 윈도우로 슬라이드
```

### 📊 최종 수집 데이터

- 각 성향별 약 2,000 영업일 OOS 수익률
- 2018-01 ~ 2025-10 커버

## 4. 주요 개념 설명

### 🎓 개념 1: Ledoit-Wolf 수축이 왜 중요한가

**문제**: 24개월 IS = 약 500일
- 30×30 공분산 = 900개 원소 추정 필요
- 500일로 900개를 추정하면 **불안정**

**해결**:
```
Σ_estimated = (1-α) × Σ_sample + α × Σ_shrink
```
- Σ_sample: 실제 데이터 공분산 (불안정)
- Σ_shrink: 구조화된 공분산 (예: 대각 행렬) (안정)
- α: 수축 강도 (데이터가 부족하면 α ↑)

**결과**: 극단적 추정 방지 → 최적화 안정

### 🎓 개념 2: 거래비용 처리

**현재 설정**:
```python
COST_BPS = 10  # 매매 수수료
SLIP_BPS = 5   # 슬리피지
TOTAL_COST = 15bps / 편도

일별 수익률 차감:
  if 분기 첫날:
      turnover = |새 비중 - 이전 비중|.sum()
      cost = turnover × 15bps
      일별 수익률 -= cost
```

**왜 편도 15bps**:
- 기관 투자자 평균 수준
- 소매 투자자는 30bps 이상일 수 있음 (거래비용 감응도는 Step 10에서 검토)

### 🎓 개념 3: 벤치마크의 의미

**벤치마크 3종**:
- **EW (1/30)**: "무지성 균등 배분" — 이길 수 있는가?
- **SPY 100%**: "단순 S&P 500 ETF 보유" — 시장 대비 우위?
- **60/40**: "주식 60% + 채권 40%" — 전통적 안전 포트폴리오

→ MV가 이 벤치마크들을 이겨야 "가치 창출" 입증

## 5. 판단 과정

### 🤔 주요 결정 사항

#### 결정 1: IS 24개월, OOS 3개월

**WHY**:
- IS 24개월: 공분산 안정 추정 최소 기간
- OOS 3개월: 분기 리밸런싱 실무 표준
- 슬라이드 3개월: OOS 중복 없이 전체 커버

**대안 고려**:
- IS 12개월: 너무 짧아 공분산 추정 불안정
- OOS 1개월: 리밸런싱 빈번 → 거래비용 폭증

#### 결정 2: 벤치마크 3개 병행

**이유**: 단일 벤치마크로는 우위가 우연일 수 있음. 3개 모두 초과해야 진정한 개선.

#### 결정 3: MV 단독 구조

**이유**: Step 3의 "개념 시연"을 실전 환경에서 재현
- 대안데이터 주입(경로 1/2) 없이 순수 MV만으로 성과 측정
- 이후 Step 7~10에서 개선 효과 측정의 baseline

## 8. FAQ

### ❓ Q1. Sharpe 0.81이 나쁜가요?

**A**: 절대 기준으로는 좋은 편 (헤지펀드 평균 0.5~0.8). 그러나 **벤치마크 대비 추가 가치 = 거의 없음**. 그래서 다음 단계로 진행.

### ❓ Q2. 왜 31개 윈도우인가요? 32개 아닌가요?

**A**: 2016-01 ~ 2025-12 기간에서 실제 거래일 기준 생성 시 31개가 맞음. Step 8에서 확인됨.

### ❓ Q3. 공격형이 Sharpe 0.81로 보수형과 같은 이유?

**A**: 공격형은 수익률 ↑ 변동성 ↑ (둘 다 증가). Sharpe는 둘의 비율이라 비슷한 수준. 다만 **MDD는 공격형이 훨씬 나쁨** → 실전에서는 성향 선택 중요.

### ❓ Q4. 거래비용 없다면 얼마나 좋아지나요?

**A**: 분기 리밸런싱 기준 연 0.3~0.5% 개선 가능. Sharpe +0.02~0.03 수준. 큰 영향 아님.

### ❓ Q5. 왜 RP, HRP는 WF에서 안 하나요?

**A**: v3 decision_log가 MV 주력 결정. RP/HRP는 Step 3 개념 시연용. v4.2 후속 과제로 WF 적용 고려 가능.
