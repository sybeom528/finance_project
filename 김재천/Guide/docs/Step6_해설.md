- 맑은 날 옷과 비 오는 날 옷이 다르듯, 시장 상태에 따라 투자 전략 달라져야 함

3. **경로 1 (경보→주식 축소)의 입력 생성**

## 2. 사전 지식

### 📚 용어 사전

| 용어 | 쉬운 설명 |
|------|---------|
| **HMM (Hidden Markov Model)** | "숨은 시장 상태를 관측 가능한 지표(VIX 등)로 추론하는 AI" |
| **레짐 (Regime)** | 시장의 체제 (저변동/고변동 등) |
| **BIC (Bayesian Information Criterion)** | 모델 복잡도와 적합도 균형 기준 (작을수록 좋음) |
| **경보 레벨** | L0(정상) / L1(주의) / L2(경계) / L3(위기) |
| **VIX Contango** | VIX3M - VIX (양수=정상, 음수=공포) |
| **Sahm Rule** | 실업률 급등 → 침체 신호 (경제학자 Sahm 2019) |
| **디바운스 (Debounce)** | 짧은 변동 필터링 (5일 중 3일 같은 레벨이면 확정) |

## 3. HMM 레짐 분류

### 🎓 HMM의 원리

**숨은 상태 추론**:
```
관측: VIX, HY, yield_curve, Cu/Au 등
  ↓ (HMM이 역으로 추론)
숨은 상태: "저변동" / "일반" / "고변동" / "위기"
```

**확률적 전이**:
```
오늘 "일반" → 내일:
  - "일반" 유지: 95%
  - "고변동" 전환: 4%
  - "위기" 전환: 1%
```

### 📊 BIC 기준 최적 n 선택

**n=2, 3, 4, 5 비교 결과**:

| n | log-L | 파라미터 k | BIC |
|---|-------|-----------|-----|
| 2 | -4,839 | 23 | 9,856 |
| 3 | -4,854 | 38 | 10,002 |
| **4** | **-2,406** | 55 | **5,239** ⭐ |
| 5 | -3,040 | 74 | 6,653 |

**결론**: n=4가 **BIC 최소** → 데이터가 4개 레짐 구조를 지지 (Kass & Raftery 기준 "매우 강한 증거")

### 📊 4개 레짐의 특성

| 레짐 | VIX 평균 | HY 평균 | 영업일 | 비중 | 해석 |
|------|---------|--------|------|------|------|
| 0 | 12.33 | 3.63% | 470 | 20.2% | **저변동** (조용한 시장) |
| 1 | 19.19 | 3.17% | 702 | 30.2% | **일반** (건전) |
| 2 | 19.93 | 4.34% | 729 | 31.3% | **일반+신용긴장** |
| 3 | 22.26 | 4.48% | 427 | 18.3% | **고변동** (스트레스) |

### 💡 레짐 분류의 활용

**Step 7에서**: 경보 레벨 판단의 참조
**Step 8에서**: Σ_stable (레짐 0+1) / Σ_crisis (레짐 2+3) 그룹화

## 4. 4가지 경보 Config

### 🎓 Config 설계 원칙

**단순 → 복잡** 순차 확장:
- A: VIX만 (가장 단순)
- B: A + Contango (2지표)
- C: 7지표 복합 (가장 정교)
- D: C + 디바운스 (false alarm 필터)

### 🎯 Config A: VIX 단독

```python
if VIX >= 35: return L3
if VIX >= 28: return L2
if VIX >= 20: return L1
return L0
```

**근거**: VIX 20/28/35는 역사적 중요 임계값

### 🎯 Config B: A + Contango

```python
level = alert_A(row)
if VIX_contango < 0:  # 백워데이션 (공포)
    level = min(level + 1, 3)
return level
```

**추가 의미**: Contango 음수 = 단기 공포 > 장기 공포 = **급성 위기**

### 🎯 Config C: 7지표 복합 스코어

**7개 지표 가중 평균**:

| 지표 | 가중 | 방향 |
|------|-----|------|
| VIX_level | 0.25 | (+) |
| HY_spread | 0.20 | (+) |
| VIX_contango | 0.15 | (-) |
| yield_curve | 0.10 | (-) |
| SKEW_level | 0.10 | (+) |
| claims_zscore | 0.10 | (+) |
| Cu_Au_ratio_chg | 0.10 | (-) |

**임계값**: 롤링 500일 p75/p90/p97 기반 (동적 조정)

**+ 4개 전문가 트리거**:
- HY_spread 5일 변화 > 0.5%p
- 수익률 곡선 역전 (yield_curve < 0)
- Sahm Rule 지시자 ≥ 0.50
- VIX_contango < -0.05 (깊은 백워데이션)

### 🎯 Config D: C + 디바운스

```python
if 최근 5일 중 3일 이상 current_level 이상이면 유지
else: current_level - 1 (완화)
```

**효과**: 1~2일 스파이크는 무시 → false alarm 감소

## 5. 판단 과정

### 🤔 주요 결정 사항

#### 결정 1: HMM 레짐 수 n=4

**대안**:
- n=2: 단순 이분법 (Stable/Crisis)
- **n=4**: BIC 최적, 세분화
- n=5~: 과적합 위험

**선택**: **n=4** (데이터의 목소리 존중)

#### 결정 2: HMM 학습 전체 데이터 vs WF 재학습

**v3 Step 6**: 전체 10년 데이터 학습 (고정 라벨)
**v4.1 Step 8**: 이를 "고정 렌즈"로 사용 (Deployment Simulation)

**이유**: WF 24개월 IS는 Crisis 관측 너무 적어 HMM 학습 불안정

#### 결정 3: 4개 Config 병행

**이유**: 단순성 vs 정교성 trade-off 실증:
- A/B: 2 변수로 충분한가?
- C/D: 7 변수 복합이 더 좋은가?

→ Step 7 Ablation에서 답 도출

### 📝 경보 설계의 2가지 철학

**"경보 발동 시점 우선"** (민감): Config A
**"경보 정확성 우선"** (정밀): Config D (디바운스)

실전에서는 **정밀 > 민감** 선호 (false alarm → 거래비용)

## 8. FAQ

### ❓ Q1. HMM이 VIX 외 다른 변수도 보나요?

**A**: 네. 5개 입력: VIX_level, VIX_contango, HY_spread, yield_curve, Cu_Au_ratio_chg

### ❓ Q2. 왜 BIC n=4가 최적인데 Σ는 2그룹인가요?

**A**: 별도 문제:
- HMM 4레짐: 시장 상태의 정교한 분류
- Σ 2그룹: 공분산 추정에 필요한 충분한 관측수 확보
- 4개 Σ 별도 추정도 Step 8에서 병행 (v4.2 비교용)

### ❓ Q3. Config D 디바운스 5일 3회는 왜?

**A**: 경험적 결정:
- 3일: 단기 노이즈 필터 적정
- 5일: 주 1회 수준 리밸런싱 주기와 일치

### ❓ Q4. 경보 레벨이 기존 v3의 20/28/35 VIX 임계값과 연결?

**A**: Config A는 정확히 이 값 사용. Config B/C/D는 이를 기반으로 확장.

### ❓ Q5. Sahm Rule이 정말 경제 예측에 유용한가요?

**A**: 2021년 논문 이후 광범위 활용. 실업률 급등은 경제 침체 강력 선행지표.
