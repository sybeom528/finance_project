## 3. HMM 레짐 분류

### 🎓 HMM의 원리

**숨은 상태 추론**:
```
관측: VIX, HY, yield_curve, Cu/Au 등
  ↓ (HMM이 역으로 추론)
숨은 상태: "저변동" / "일반" / "고변동" / "위기"
```

**확률적 전이**:
```
오늘 "일반" → 내일:
  - "일반" 유지: 95%
  - "고변동" 전환: 4%
  - "위기" 전환: 1%
```

### 📊 최적 n 선택 — 왜 BIC만으로는 부족한가 (v4.2)

#### ⚠️ BIC의 함정: n이 커질수록 계속 감소

**n=2~8 확장 탐색 결과 (수정된 BIC 공식 적용)**:

| n | log-L | 파라미터 k | BIC | 감소폭 | 최소 레짐 일수 |
|---|-------|-----------|-----|--------|--------------|
| 2 | -14,025 | 43 | 28,387 | — | 842 |
| 3 | -12,211 | 68 | 24,954 | -3,433 | 588 |
| **4** | **-10,558** | **95** | **21,859** ⭐ (채택) | -3,095 | **424** |
| 5 | -9,736 | 124 | 20,442 | -1,417 | 407 |
| 6 | -8,902 | 155 | 19,016 | -1,426 | 167 |
| 7 | -8,712 | 188 | 18,895 | -121 (가짜 엘보우) | **70** ⚠ |
| 8 | -7,920 | 223 | **17,584** (BIC 최소) | -1,311 | **59** ⚠ |

**관찰**:
- BIC는 n=2→8까지 **단조 감소** → 단일 기준으로 사용 불가
- n=6→7에서 감소폭이 줄어 "엘보우처럼" 보이나, n=7→8에서 다시 급감
- **HMM은 regularity conditions를 완전히 만족하지 않아 BIC는 근사 지표**
- n=7, 8은 최소 레짐 < 100일 → 공분산 추정 불안정

#### 🎯 다기준 Scorecard로 n 선택

BIC 단독 한계를 극복하기 위해 **8개 기준** 종합 평가:

| 기준 | 측정 | 목표 |
|------|-----|------|
| ① BIC | 낮을수록 좋음 | 기계적 최소 |
| ② ICL | HMM 전용 지표 | 낮을수록 좋음 |
| ③ **해석성** | 경제적 명명 가능성 | 직관적 |
| ④ 수렴 | 10 seed 중 수렴 개수 | 10/10 |
| ⑤ **Rare state** | 최소 레짐 관측 일수 | ≥ 100일 |
| ⑥ 전이 적정성 | 연간 전이 5~20회 | 적정 범위 |
| ⑦ T/k | 파라미터 대비 관측치 | ≥ 10 |
| ⑧ 지속성 | 평균 duration 10~100일 | 실무적 |

**종합 점수 (80점 만점)**:

| n | BIC | ICL | 해석성 | 수렴 | rare | 전이 | T/k | 지속 | **합계** |
|---|-----|-----|--------|------|------|------|------|------|---------|
| 2 | 0.0 | 0.0 | 7.0 | 10.0 | 10.0 | 5.0 | 10.0 | 5.8 | 47.8 |
| 3 | 3.2 | 3.1 | 9.0 | 10.0 | 10.0 | 5.0 | 10.0 | 10.0 | 60.3 |
| **4** | 6.0 | 6.1 | **10.0** | 10.0 | **10.0** | 5.0 | 8.7 | 7.8 | **63.7** ⭐ |
| 5 | 7.4 | 7.4 | 6.0 | 10.0 | 10.0 | 5.0 | 6.7 | 7.5 | 60.0 |
| 6 | 8.7 | 8.7 | 4.0 | 10.0 | 5.6 | 5.0 | 5.4 | 7.8 | 55.1 |
| 7 | 8.8 | 8.8 | 2.0 | 10.0 | 2.3 | 5.0 | 4.4 | 10.0 | 51.3 |
| 8 | **10.0** | **10.0** | 1.0 | 10.0 | 2.0 | **10.0** | 3.7 | 10.0 | 56.8 |

**결론**: **n=4가 63.7점으로 1위** — BIC 최소인 n=8(56.8점)보다 6.9점 우위

| 레짐 | VIX 평균 | HY 평균 (%) | Yield Curve | 영업일 | 비중 | 해석 |

### 📊 설계 결정: Full-sample 학습 + Forward 추론

**핵심 분리**: 레짐 정의(파라미터)는 고정, 시점별 판단은 인과적.

#### (a) 학습 단계 — Full-sample 1회

본 HMM은 **의도적으로** 전체 시계열(2016~2025)로 1회 학습합니다. 짧은 IS 윈도우(예: 24개월)로 매 walk-forward마다 재학습하지 **않는** 이유:

| 문제 | 설명 |
|------|------|
| (1) 추정 불안정 | 위기 레짐 관측이 거의 없는 윈도우(예: 2017년 calm 구간)에서 Σ_crisis 추정이 비정상적으로 큼 |
| (2) 비교 불가 | 윈도우마다 레짐 정의가 달라져 "calm 레짐의 비중 변화" 같은 해석이 무의미 |
| (3) 운영 복잡 | 매 윈도우마다 label switching 보정 + 다기준 n 재선택 필요 |

→ "레짐 **정의**는 고정, 시점별 **판단**은 인과적"으로 책임 분리.

#### (b) 추론 단계 — Forward 알고리즘 (시점별)

학습 완료된 모델로 각 시점 t의 레짐 분류 시 **Forward 알고리즘**을 사용. t까지의 관측만으로 P(state_t | obs_1..t) 산출.

- **Viterbi**: 전체 관측치로 최적 경로 탐색 → **미래 정보 사용** (backtest 편향)
- **Forward**: 각 시점까지의 관측치로만 확률 계산 → **인과적**

**실측 차이**: Forward vs Viterbi 레짐 불일치 39일 (1.57%) — 2026-04-19에 Viterbi → Forward로 교체.

→ 결과: **Strict OOS와 호환** (학습 파라미터는 deployment-sim 성격이지만, 시점별 라벨 부여는 미래 정보 미사용).

### 💡 레짐 분류의 활용

**Step 7에서**: 경보 레벨 판단의 참조
**Step 8에서**: Σ_stable (레짐 0+1) / Σ_crisis (레짐 2+3) 그룹화

## 4. 4가지 경보 Config

### 🎓 Config 설계 원칙

**단순 → 복잡** 순차 확장:
- A: VIX만 (가장 단순)
- B: A + Contango (2지표)
- C: 7지표 복합 (가장 정교)
- D: C + 디바운스 (false alarm 필터)

### 🎯 Config A: VIX 단독

```python
if VIX >= 35: return L3
if VIX >= 28: return L2
if VIX >= 20: return L1
return L0
```

**근거**: VIX 20/28/35는 역사적 중요 임계값

### 🎯 Config B: A + Contango

```python
level = alert_A(row)
if VIX_contango < 0:  # 백워데이션 (공포)
    level = min(level + 1, 3)
return level
```

**추가 의미**: Contango 음수 = 단기 공포 > 장기 공포 = **급성 위기**

### 🎯 Config C: 7지표 복합 스코어

**7개 지표 가중 평균**:

| 지표 | 가중 | 방향 |
|------|-----|------|
| VIX_level | 0.25 | (+) |
| HY_spread | 0.20 | (+) |
| VIX_contango | 0.15 | (-) |
| yield_curve | 0.10 | (-) |
| SKEW_level | 0.10 | (+) |
| claims_zscore | 0.10 | (+) |
| Cu_Au_ratio_chg | 0.10 | (-) |

**임계값**: 롤링 500일 p75/p90/p97 기반 (동적 조정)

**+ 4개 전문가 트리거**:
- HY_spread 5일 변화 > 0.5%p
- 수익률 곡선 역전 (yield_curve < 0)
- VIX_contango < -0.05 (깊은 백워데이션)
- (실물경기 트리거는 v5 과제로 보류 — 데이터 PIT 제한 해소 후 도입)

### 🎯 Config D: C + 디바운스

```python
if 최근 5일 중 3일 이상 current_level 이상이면 유지
else: current_level - 1 (완화)
```

**효과**: 1~2일 스파이크는 무시 → false alarm 감소

## 8. FAQ

### ❓ Q1. HMM이 VIX 외 다른 변수도 보나요?

**A**: 네. 5개 입력: VIX_level, VIX_contango, HY_spread, yield_curve, Cu_Au_ratio_chg

### ❓ Q2. 왜 BIC 최소가 n=8인데 n=4를 선택했나요? (v4.2 업데이트)

**A**: BIC만으로는 n을 결정할 수 없습니다. 이유:

1. **BIC 단조 감소**: n=2→8까지 BIC가 계속 하락 → "멈출 지점" 없음 (만약 n=9, 10까지 확장해도 계속 하락 가능성 높음)
2. **HMM의 regularity 위반**: identifiability (label switching), boundary parameters 문제로 BIC의 penalty가 HMM에 부적합
3. **ICL, AIC도 동일 문제**: 모든 정보 기준이 n=8 선호 → 정보 기준 자체의 한계

**다기준 평가 결과**:
- n=8 (BIC 최소): 최소 레짐 59일(rare), 해석 불가 (VIX 13-16 구간 4분할 무의미)
- n=4: 4개 레짐 모두 428일 이상 관측, {calm/normal/stress} 명확 해석

**학술적 입장**: Celeux & Durand (2008)의 "선택 기준의 병행" 원칙 준용.

### ❓ Q2-보조. HMM 4레짐인데 왜 Σ는 2그룹인가요?

**A**: 별도 문제:
- HMM 4레짐: 시장 상태의 정교한 분류
- Σ 2그룹: 공분산 추정에 필요한 충분한 관측수 확보 (5×5 정규 공분산에 최소 100일)
- 4개 Σ 별도 추정도 Step 8에서 병행 (v4.1 비교용)

- 3일: 단기 노이즈 필터 적정
- 5일: 주 1회 수준 리밸런싱 주기와 일치
