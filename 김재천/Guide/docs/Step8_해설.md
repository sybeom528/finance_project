## 1. 배경

### 🤔 왜 평상시와 위기를 나누어 봐야 하나요?

**일상 비유 - 교통 흐름**:
- **평상시 도로**: 자동차들이 대체로 비슷한 속도로 움직임, 서로 간격 유지
- **사고 난 도로**: 자동차들이 한꺼번에 멈추거나, 다른 차선으로 몰림 — **움직임 패턴이 완전히 다름**

**금융 시장도 마찬가지입니다**:
- **평상시**: 주식이 오르면 채권은 내림 (상쇄 효과로 포트폴리오 안정)
- **위기 시 (예: 2020 COVID)**: 모든 자산이 동시에 하락 → 기존의 "상쇄 효과"가 사라짐

### 🎯 Step 8의 목적

**"평상시"와 "위기" 두 상태의 자산 움직임 패턴을 각각 수치화**하여, Step 9의 포트폴리오 최적화에 입력으로 제공합니다.

## 2. 사전 지식

### 📚 용어 사전

| 용어 | 쉬운 설명 |
|------|----------|
| **공분산 (Σ)** | "자산들이 함께 움직이는 패턴을 숫자로 정리한 표". 30개 자산이라면 30×30 표 |
| **HMM (Hidden Markov Model)** | "시장 상태를 자동으로 분류하는 AI 모델". 4가지 상태(저변동/일반/경계/위기)로 분류 |
| **레짐 (Regime)** | "시장의 체제/상태". 평상시 vs 위기 같은 구분 |
| **Walk-Forward** | "과거 2년 데이터로 학습하고 다음 3개월은 예측하는 방식". 이를 계속 슬라이드 |
| **IS (In-Sample)** | 학습용 과거 데이터 (여기서는 24개월) |
| **OOS (Out-of-Sample)** | 검증용 미래 데이터 (여기서는 3개월) |
| **Ledoit-Wolf 수축** | "공분산 추정을 안정화하는 수학 기법". 부족한 데이터로도 믿을 만한 결과 만듦 |
| **Fallback** | "Plan A가 안 될 때 쓰는 Plan B, C". 여기서는 데이터 부족 시 대체 방안 |

## 4. 주요 개념 설명

### 🎓 개념 1: HMM 레짐의 의미

Step 6에서 HMM이 10년 데이터를 학습하여 매일을 4개 상태 중 하나로 분류했습니다.

| 레짐 | VIX 평균 | 해석 |
|------|---------|------|
| 0 | 12.3 | **저변동** (매우 조용한 시장) |
| 1 | 19.2 | **일반** (건전한 신용) |
| 2 | 19.9 | **일반+긴장** (신용 스프레드 확대) |
| 3 | 22.3 | **고변동** (스트레스) |

**VIX + HY_spread 이중 검증**:
- VIX 평균: 0(12.3) → 1(19.2) → 2(19.9) → 3(22.3), 단조 증가
- HY_spread: 1(3.17%) vs 2(4.34%), 명확한 차이

→ "평상시 = 레짐 0+1, 위기 = 레짐 2+3" 그룹화가 타당함을 양면으로 검증

### 🎓 개념 2: Walk-Forward의 의미

**일상 비유 - 요리 실습**:
- **나쁜 방식 (학업 평가)**: 1~10회 수업 모두 듣고 시험 보기 → 시험에서 이전 수업 정답 기억해냄 (look-ahead)
- **좋은 방식 (Walk-Forward)**: 1~8회 수업만 듣고 9회 시험 → 실제 실력 평가

**포트폴리오 백테스트도 같은 원리**:
- 2018년 1~3월 성과를 평가하려면 **2016~2017년 데이터만** 사용
- 2025년 데이터로 2018년을 판단하면 부정행위 (look-ahead bias)

**우리 설정**: 24개월 학습 → 3개월 검증, 총 **31회** 반복

### 🎓 개념 3: Ledoit-Wolf 수축이 필요한 이유

**일상 비유 - 설문조사**:
- 100명 조사 결과: "과자 1개 평균 가격은 1,500원 ± 500원" (신뢰도 높음)
- 3명 조사 결과: "과자 평균 1,000원 ± 2,000원" (변동 너무 큼, 믿을 만한 수치 아님)

**공분산 추정도 비슷**:
- 자산 30개의 관계 추정 = 900개 숫자 계산 (30×30 행렬)
- 관측일이 적으면 900개 숫자가 불안정하게 튐
- **Ledoit-Wolf가 하는 일**: "단순한 기준 공분산 + 실제 데이터"를 적절히 섞어 안정화
- 결과: 데이터가 적어도 극단적이지 않은 추정값 확보

### 🎓 개념 4: Fallback 4단계

| 상황 | Fallback 타입 | 처리 |
|------|------------|------|
| 평상시·위기 모두 48일+ | **separate** | 정상 분리 추정 |
| 평상시만 48일+ | **scaled** | Σ_crisis = Σ_stable × 1.5 |
| 위기만 48일+ | **scaled_reverse** | Σ_stable = Σ_crisis / 1.5 |
| 둘 다 48일 미만 | **single** | 구분 포기, 단일 Σ |

**1.5배의 근거** (역사적 관찰):
- 2008 금융위기: 평상시 변동성 대비 1.67배
- 2020 COVID: 2.14배
- 2022 긴축: 1.44배
- **평균 ≈ 1.5배** (보수적 프록시)

## 5. 판단 과정

### 🧭 주요 결정 사항

#### 결정 1: HMM 전체 기간 학습 vs 윈도우별 재학습

**선택**: **전체 기간 학습 (Deployment Simulation)**

**이유**:
- 24개월 윈도우에서 위기 관측이 5~15일로 너무 적음 → HMM 학습 실패 가능성
- 현실적 운용 방식: "현재 모델을 과거에 적용해보는" 접근이 더 실무적
- 레짐 라벨은 "고정 렌즈"로만 사용, 실제 의사결정(μ, Σ 추정, 최적화)은 IS 데이터만 사용

**대안 고려**:
- Expanding Annual Refresh (연 1회 재학습): v4.2 후속 과제로 분리

#### 결정 2: 4개 레짐 유지 vs 2개 레짐으로 통합

**선택**: **4개 레짐 유지 (BIC 최소값 증거)**

**이유**:
- BIC(n=2) - BIC(n=4) = **+4,617** (Kass-Raftery 기준 "매우 강한 증거")
- 데이터가 4개 레짐 구조를 명백히 지지
- Σ는 2그룹으로 통합 (HMM 상세 vs 공분산 단순성 절충)

#### 결정 3: Σ 그룹화 기준 (VIX vs 경보 레벨)

**선택**: **VIX 평균 기준 자동 그룹화**

**이유**:
- HMM 레짐 번호(0, 1, 2, 3)는 **임의적**, 번호 자체에 의미 없음
- VIX 평균으로 정렬하여 하위 2개 = Stable, 상위 2개 = Crisis
- HY_spread 차이(1.17%p)도 같은 방향으로 뒷받침 → 이중 검증

#### 결정 4: 대칭 Fallback 구조 (v4.1 개선)

**초기 설계**: 비대칭 (Crisis 관측수만 체크)
**발견 버그**: 윈도우 12~13이 single로 잘못 분류됨
**수정**: 대칭 4단계 (Stable·Crisis 각각 독립 판정)
**결과**: single 7개 → 0개, 모든 윈도우 경로 2 활성화

## 8. FAQ

### ❓ Q1. 왜 2016년 데이터부터 시작하나요?

**A**: Step 1 데이터 수집이 2016년부터이기 때문입니다. 2006-2015 데이터 추가 시 2008 금융위기 포함하여 Crisis 관측을 풍부하게 확보할 수 있으나, 이는 v5 후속 과제입니다.

### ❓ Q2. 레짐 3(위기)이 왜 전체의 18%밖에 안 되나요?

**A**: 2016-2025 기간은 상대적으로 큰 위기가 적었던 시기 (2020 COVID, 2022 긴축, 2023 SVB 정도). Crisis 레짐은 실제 위기 구간만 포착하므로 비중이 낮은 것이 정상입니다.

### ❓ Q3. Full-sample HMM 학습은 look-ahead bias 아닌가요?

**A**: **엄격하게는 그렇지만**, 이 프로젝트는 "Deployment Simulation" 철학을 채택합니다 — "현재 모델을 과거에 적용했을 때 어떤 성과였을지"를 시뮬레이션. 의사결정(μ, Σ 추정, MV 최적화)은 IS 데이터만 사용하여 실제 look-ahead를 회피합니다.

### ❓ Q4. Σ_stable과 Σ_crisis는 얼마나 다른가요?

**A**: 표본 구간에 따라 다르나, 일반적으로 **Crisis 시 주식 변동성이 10~30% 증가**하고 **자산 간 상관계수가 10~30% 절대값 증가**합니다. 그러나 MV 최적화에서 이 차이가 비중에 큰 영향을 주지 못해 경로 2가 효과 없었습니다 (Step 9 해설 참조).

### ❓ Q5. Fallback의 1.5배는 어디서 나온 숫자인가요?

**A**: 역사적 위기 3회(2008 GFC, 2020 COVID, 2022 긴축)의 **SPY 변동성 평균 배율**을 단순 평균한 값입니다. 1.44 + 1.67 + 2.14 = 5.25 / 3 ≈ 1.75 (보수적으로 1.5 채택).

### ❓ Q6. 4그룹 Σ는 어디에 쓰이나요?

**A**: Step 8에서 `regime_covariance_4group.pkl`로 저장했지만, **v4.1 Step 10에서 활용 안 됨**. v4.2 후속 작업에서 "2그룹 vs 4그룹 성과 비교"에 사용될 예정입니다.
