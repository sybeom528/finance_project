## 5. 공분산 추정 방법

### 확정 구조
```
Level 1 (자산군 간): 블록 대각 — 대표 ETF 3개(SPY, AGG, GLD) 간 3×3
Level 2 (주식 그룹): PCA 팩터 모형(K 자동 선택) + 레짐 조건부
Level 2 (채권 그룹): Ledoit-Wolf (N=4, 안정)
Level 2 (대안 그룹): 직접 추정 (N=2, 파라미터 1개)
```

### PCA 팩터 모형 상세
- K 자동 선택: 누적 설명 분산 >= 80%인 최소 K (하한 3, 상한 10)
- IS 윈도우별 K 변동 허용 (시장 환경 적응)
- 각 IS에서 선택된 K 값을 로그로 저장

### 레짐 조건부 Ledoit-Wolf
- IS 내 HMM 레짐별로 Σ_stable, Σ_crisis 분리 추정
- 경보 L0~L1 → Σ_stable, L2~L3 → Σ_crisis
- Fallback (위기기 관측일 부족 시):
  - >= 48일: 정상 분리 추정
  - 20~47일: Σ_crisis = Σ_stable × 1.5 (스트레스 스케일링)
  - < 20일: 단일 공분산 (레짐 분리 포기)

### 채택/탈락 근거 요약
- Ledoit-Wolf: Level 2 채권/대안에서 채택 (N 작음, 안정적)
- 블록 대각: Level 1에서 채택 (이중 계산 차단, 파라미터 3개)
- PCA 팩터 모형: Level 2 주식에서 채택 (N=24 → LW만으론 비율 1.8:1 불안정)
- DCC-GARCH: 미채택 (N=24에서 파라미터 700+, 수렴 불안정, 계산 수 시간)
- 랜덤 행렬 이론: 미채택 (구현 복잡, 금융 검증 사례 제한)
- Graphical LASSO: 미채택 (λ 튜닝 필요, 역공분산 기반으로 변환 필요)
- 팩터 모형이 DCC-GARCH + 랜덤 행렬의 핵심 효과를 포함하면서 구현 간단

```
5. 교체 시: 왕복 거래비용 30bps 부과
```

### "섹터별 시총 1위" 근거
- 기계적: 사람의 주관 배제
- 중립적: 성장주/가치주 편향 없음 (성향 차이는 최적화에서 표현)
- 섹터 분산 자동 보장
- 공격형의 성장주 선호: 높은 주식 비중 + 높은 개별 상한으로 충분히 표현

### 생존자 편향 방지
- NVDA(2015년 시총 200위) 같은 사후 성공 종목이 자동 탈락
- "매년 그 시점의 1위"를 사용하므로 미래 정보 개입 없음

### 교체 완충(Buffer) 규칙 근거
- 섹터 1위-2위 상관관계 평균 0.75~0.85 (거의 동일 노출)
- 교체 비용(왕복 30bps) > 기대 초과 수익(연 0.1~0.3%) → 교체하지 않는 것이 합리적
- 3위 이하로 밀림 = 구조적 변화(사업 악화, 경쟁사 부상) → 교체 합리적
- 기대 교체 빈도: 연 1~2개 종목

### 재구성 규칙 타당성 검증 (Perturbation Test)
- 원래 규칙 + 변형 5가지(시총 2위, 랜덤 1~2위, 시점 변경, 완충 없음, 완충 3위)
- 통과 기준: |Sharpe_원래 - Sharpe_변형| / Sharpe_원래 < 10%
- 10% 근거: 거래비용 영향 범위(Sharpe 0.03~0.08)와 동일 수준
- 학술적 기준은 추후 확인 시 변동 가능

## 9. 투자자 성향 프로파일

### generate_profile(gamma) 연속 함수
```python
def generate_profile(gamma):
    t = (gamma - 1) / 9  # 0(공격) ~ 1(보수)
    return {
        'gamma': gamma,
        'target_vol':  0.22 - 0.15 * t,
        'max_equity':  0.90 - 0.60 * t,
        'min_bond':    0.00 + 0.40 * t,
        'max_mdd':    -(0.35 - 0.25 * t),
        'max_single':  동적 산출 (아래 10장 참조),
    }
```

### 4개 프리셋
| 성향 | γ | 목표 변동성 | 주식 상한 | 채권 하한 | MDD 허용 |
|------|---|-----------|----------|----------|---------|
| 보수형 | 8 | 7% | 30% | 40% | -10% |
| 중립형 | 4 | 10% | 55% | 20% | -18% |
| 적극형 | 2 | 15% | 75% | 10% | -25% |
| 공격형 | 1 | 22% | 90% | 0% | -35% |

### gamma 연속 곡선 분석
- gamma 1.0~10.0 (0.5 단위, 19개 지점)
- 각 지점에서 MV 최적화 + W-F 백테스트 → Sharpe/MDD 곡선
- Config별 최적 gamma 비교 → "대안데이터가 최적 리스크 수준을 낮추는가?"

### 설문 기반 리스크 스코어: 이번 프로젝트 범위 밖
- 이유: 프로젝트 목적(대안데이터 검증)과 불일치, 행동 데이터 없이 검증 불가, 분석 결과에 영향 없음
- generate_profile(gamma)가 있으므로 향후 설문→gamma 매핑 함수만 앞단에 추가하면 확장 가능

## 10. 비중 상한 (HHI 기반)

### 산출 공식

```
비중 상한 = min(MDD 역산값, HHI Cap)

MDD 역산: (MDD 허용 × 1/3) / 극단 하락률 × 배수
HHI Cap:  HHI 기준을 충족하는 최대 개별 비중
```

### 자산 유형별 배수 (역사적 데이터 기반)

극단 하락률 수집 (2008 GFC, 2020 COVID, 2022 금리인상):
- 인덱스 ETF: 1개월 최악 -35% (SPY -33.9%, IWM -41.5% 등의 보수적 중앙값)
- 섹터 ETF: 1개월 최악 -45% (XLE -57.9%, XLF -38.4% 등의 중앙값)
- 개별 종목: 수개월 최악 -50% (META -76.7%, BA -72.4% 등의 보수적 중앙값)

배수 = 개별 종목 극단 하락(50%) / 해당 유형 극단 하락
- 인덱스 ETF: 50/35 = 1.43 → **1.4** (반올림)
- 섹터 ETF: 50/45 = 1.11 → **1.1** (반올림)
- 개별 종목: 50/50 = 1.00 → **1.0** (기준)

### HHI(Herfindahl-Hirschman Index) 기반 Cap

HHI = Σ(w_i²), DOJ 시장 집중도 기준 차용
- 보수형/중립형: HHI < 0.10 (비집중) → 최대 비중 ~15%
- 적극형: HHI < 0.15 (보통 집중) → 최대 비중 ~20%
- 공격형: HHI < 0.20 (고집중 미만) → 최대 비중 ~25%

### 최종 확정 비중 상한표

## 16. ML 수익률 예측 모델 — 레짐 설계

> 작성일: 2026-04-16

### 배경 및 문제 제기

ML 수익률 예측 모델의 패널 피처로 `regime` 변수를 포함하기로 결정.
초기 검토안이었던 Walk-Forward IS 윈도우(150일)마다 레짐 모델을 재학습하는 구조는
세 가지 이유로 부적합하다는 결론을 내림.

| 문제 | 내용 |
|------|------|
| 레짐 다양성 부족 | 150일(≈7개월) 안에 위기·안정·회복 레짐이 모두 관측될 보장 없음 |
| 레짐 정의 불안정 | 윈도우마다 레짐 기준이 달라져 수익률 예측 모델이 일관된 신호를 학습할 수 없음 |
| HMM 수렴 불안정 | 관측치가 적을수록 HMM 파라미터 추정 오차가 커짐 |

---

### 채택 방식: 전체 시계열 고정 학습

```
2016~2025 전체 데이터 → 레짐 모델 1회 학습
        ↓
전체 시계열에 레짐 라벨 부여
        ↓
Walk-Forward 모든 윈도우에 동일 라벨 적용
```

**약한 Look-Ahead 존재 — 허용 가능 수준으로 판단**

레짐 판별 입력(VIX, HY spread, GDELT 집계 감성)은 가격 수익률이 아닌
매크로·감성 지표이며, VIX=12이면 어느 시점에서 봐도 안정 레짐으로 판별되므로
실질적 정보 누출량이 크지 않음. 단, 이를 한계 사항으로 명시.

**방식별 비교**

| 방식 | look-ahead | 안정성 | 현실성 |
|------|-----------|--------|--------|
| 150일 Rolling 재학습 | 없음 | ❌ 매우 불안정 | ❌ 비현실적 |
| **전체 시계열 고정** | 약함 (허용) | ✅ 안정 | ✅ 실무와 동일 |
| Expanding Window | 없음 | ✅ 안정 | △ 구현 복잡 |

---

### 레짐 모델 입력 피처

개별 종목 매핑 불필요 — 시장 전체 집계로 충분.
GDELT를 개별 종목 수익률 예측에 직접 연결하는 것보다
**시장 전체 감성 집계 → 레짐 판별**이 GDELT의 특성에 더 적합한 활용 방식.

| 구분 | 피처 |
|------|------|
| GDELT (시장 전체 집계) | `gdelt_mkt_tone_avg`, `gdelt_mkt_tone_std`, `gdelt_mkt_event_cnt` |
| 매크로 | `VIX_level`, `HY_spread`, `T10Y2Y`, `MOVE_index` |

### 재보정(Re-calibration) 설계

시장 구조 변화 판단 시 전체 시계열 대상 재학습 후 Walk-Forward에 재적용.
재학습 트리거는 **객관적 기준으로 고정** (사후 성능 기반 주관적 판단 금지 — look-ahead).

```
정기: 매년 1월 (연간 종목 재구성 시점과 동기화)
긴급: 새 데이터의 레짐 분포가 기존과 KL divergence 임계값 초과 시
      또는 HMM log-likelihood가 일정 수준 이하로 하락 시
```

**레짐 라벨 일관성 처리 (필수)**

HMM 재학습 시 레짐 번호가 뒤바뀌는 label switching 문제를
VIX 평균 기준 정렬로 고정.

```python
# VIX 평균 낮은 순서대로 레짐 0(안정), 1(중립), 2(위기) 재부여
vix_mean_by_regime = data.groupby(raw_labels)['VIX'].mean()
mapping = {old: new for new, old in
           enumerate(vix_mean_by_regime.sort_values().index)}
aligned_labels = raw_labels.map(mapping)
```

### 17-1. K 결정 — K=3 채택

**비교 대상: K=3 vs K=4**

| 항목 | K=3 (안정 / 중립 / 위기) | K=4 (안정 / 보통 / 경계 / 위기) |
|------|------------------------|-------------------------------|
| 해석 용이성 | 직관적, 투자 의사결정과 직접 연결 | 경계-위기 구분이 모호 |
| 위기 레짐 관측일 | ~250일 (2520일 중 약 10%) | ~90일 (1/4로 감소) → 추정 불안정 |
| 파라미터 수 (full, p=10) | 204개 (비율 12:1) | 284개 (비율 9:1, 위험 경계 진입) |
| ML 피처 다양성 | 3개 범주로 충분 | 4번째 레짐이 수익률 예측에 추가 정보 제공 불확실 |
| 채터링 위험 | 낮음 | K=3 대비 상태 전환 빈도 증가 가능 |

**채택: K=3**

- 위기 레짐을 세분화하더라도 ML 예측 모델의 수익률 예측 정확도가
  개선된다는 이론적 근거가 약함
- 관측일 부족으로 인한 추정 불안정 리스크가 세분화의 이점을 상쇄
- K=3 vs K=4의 최적성은 BIC(Bayesian Information Criterion)로 사후 검증

```python
# BIC 기반 최적 K 검증 코드
from hmmlearn.hmm import GaussianHMM
import numpy as np

bic_scores = {}
for k in [2, 3, 4, 5]:
    model = GaussianHMM(n_components=k, covariance_type='full', n_iter=200, random_state=42)
    model.fit(X)
    log_likelihood = model.score(X)
    n_params = k * k + k * p + k * p * (p + 1) / 2  # 전이확률 + 평균 + 공분산
    bic = -2 * log_likelihood * len(X) + n_params * np.log(len(X))
    bic_scores[k] = bic

optimal_k = min(bic_scores, key=bic_scores.get)
```

```
위기 레짐 실효 비율 250/35 = 7.1:1  250/14 = 17.9:1  250/7 = 35.7:1 *
────────────────────────────────────────────────────────────────────
* tied의 공분산은 전체 2520일이 공동 추정 → 위기 레짐은 평균(μ)만 단독 추정
```

#### 핵심 경제적 질문: 레짐마다 피처 간 상관 구조가 다른가?

```
[안정 레짐]                        [위기 레짐]
VIX: 12~18 (소폭 등락)             VIX: 30~80 (급등)
HY_spread: 3~4% (안정)             HY_spread: 5~12% (급등)

VIX ↔ HY_spread 상관: ~0.3         VIX ↔ HY_spread 상관: ~0.85
→ 각자 독립적으로 움직임            → 동시에 같은 방향으로 폭발
```

레짐마다 피처 간 상관 구조가 근본적으로 다르다. 이 차이를 각 옵션이 어떻게 처리하는지가 선택의 핵심이다.

#### 각 옵션의 경제적 의미

**`full` — 레짐별 고유 공분산 행렬**

```
안정 레짐 공분산: VIX-HY 상관 낮음 → 각 지표가 독립적으로 움직임
위기 레짐 공분산: VIX-HY 상관 높음 → 공포 지표들이 동시 폭발하는 패턴 포착 ✅
```

위기의 핵심 특성("모든 공포 지표가 동시에 움직인다")을 정확히 포착.
p=7로 줄인 결과 위기 레짐 실효 비율 7.1:1 → 안전 범위(≥5:1) 충족.

**`diag` — 레짐별 독립 분산만 추정**

```
# diag의 위기 판단 방식
"VIX만 높다  → 위기 가능성 50%"
"HY만 높다   → 위기 가능성 50%"
"둘 다 높다  → 위기 가능성 50%"  ← 동시성 정보 완전 손실 ❌
```

통계적으로 안정(17.9:1)하지만, 위기 레짐의 핵심 신호인 지표 간 동시 급등 패턴을 포착하지 못함.
→ 위기 조기 감지 저하, ML 피처 품질 저하.

**`tied` — 전 레짐 공유 공분산**

```
공유 공분산 = 안정 레짐 상관(0.3) + 위기 레짐 상관(0.85)의 평균값(~0.5)

결과:
  안정 레짐 판단 시: VIX-HY 상관을 0.5로 가정 (실제 0.3 → 과대 추정)
  위기 레짐 판단 시: VIX-HY 상관을 0.5로 가정 (실제 0.85 → 과소 추정)
  → 양쪽 레짐 모두에서 틀린 공분산 행렬 사용 ❌
```

레짐 간 상관 구조 차이를 전혀 반영하지 못함 → 세 옵션 중 본 목적에 가장 부적합.

#### 종합 비교

```
────────────────────────────────────────────────────────────────────
항목                     full          diag          tied
────────────────────────────────────────────────────────────────────
위기 레짐 실효 비율       7.1:1 ✅      17.9:1 ✅✅   35.7:1 ✅✅
동시 급등 패턴 포착       ✅            ❌            ❌ (평균화됨)
레짐별 상관 차이 포착     ✅            ❌            ❌
위기 조기 감지            강함          약함          보통
채터링 위험               낮음          중간          낮음
ML 피처 품질              높음          중간          낮음
────────────────────────────────────────────────────────────────────
```

#### 채택: `full` — 최종 확정

p=7로 피처를 줄인 덕분에 위기 레짐 실효 비율 7.1:1 확보.
경제적으로도 위기 레짐 동시 급등 패턴 포착에 가장 적합.
수치 불안정 방어를 위해 `min_covar` 파라미터를 반드시 설정.

```python
from hmmlearn.hmm import GaussianHMM

model = GaussianHMM(
    n_components=3,           # K=3: 안정/중립/위기
    covariance_type='full',   # 레짐별 고유 공분산 — 동시 급등 패턴 포착
    n_iter=200,
    random_state=42,
    min_covar=1e-3            # 공분산 행렬 특이행렬 근접 방어 (위기 레짐 안정화)
)
```

> **기각 이유 요약**
> - `diag`: 통계적으로 안정적이나 위기 핵심 신호(동시성) 포착 불가
> - `tied`: 레짐 간 상관 구조 차이를 전혀 반영하지 못함 → 세 옵션 중 목적에 가장 부적합

### 17-3. 레짐 모델 피처 선택 프로세스

#### 배경: 상관분석만으로 부족한 이유

단순 상관분석은 선형 관계만 측정하며, 레짐별로 상관 방향이 역전되는 현상을 감지 못 함.

예시:
```
안정 레짐: VIX ↔ HY_spread 상관 0.3 (낮음, 각자 독립 움직임)
위기 레짐: VIX ↔ HY_spread 상관 0.9 (매우 높음, 동시 급등)
전체 기간 단순 상관: 0.5~0.6 (중간값 → 두 지표 중 하나 제거하면 위기 신호 손실)
```

따라서 피처 선택은 경제적 역할 기반 그룹핑을 1차 기준으로 삼고,
상관분석은 그룹 내 중복 제거 도구로 활용한다.

#### 3단계 피처 선택 프로세스

**Step 1 — 경제적 역할 기반 그룹핑**

서로 다른 경제적 채널을 대표하는 그룹으로 먼저 분류:

| 그룹 | 역할 | 후보 피처 |
|------|------|----------|
| A. 주식시장 공포 | 옵션 내재 변동성, 꼬리 위험 | VIX_level, SKEW_zscore |
| B. 신용·유동성 | 신용 위험 프리미엄 | HY_spread, HY_spread_chg |
| C. 금리·경기 | 수익률 곡선 형태, 통화정책 | T10Y2Y, MOVE_index |
| D. 실물 경기 | 경기 선행·동행 지표 | Cu_Au_ratio_chg, Sahm_indicator |
| E. GDELT 감성 | 뉴스 기반 시장 심리 | gdelt_mkt_tone_avg, gdelt_mkt_event_cnt |

설계 원칙: 각 그룹에서 최소 1개, 최대 2개를 유지 → 총 피처 수 p ≤ 10

**Step 2 — 그룹 내 상관 분석 (중복 제거)**

동일 그룹 내 피처들 간 상관계수를 계산하여 0.8 이상이면 1개만 잔존:

```python
import pandas as pd

def remove_redundant_within_group(df, group_features, threshold=0.8):
    """그룹 내 상관 0.8 이상 피처 제거 — 경제적 해석력 높은 피처 우선 보존"""
    corr = df[group_features].corr().abs()
    to_drop = set()
    for i in range(len(group_features)):
        for j in range(i + 1, len(group_features)):
            if corr.iloc[i, j] >= threshold:
                # 두 피처 중 해석력이 낮은 파생 지표를 제거
                to_drop.add(group_features[j])
    return [f for f in group_features if f not in to_drop]
```

**Step 3 — 그룹 간 상관 최종 검토**

Step 2 이후 잔존 피처들 간 교차 상관을 확인.
그룹 간 상관이 0.8 이상이면 대표 피처 1개만 남기고 제거.

```python
# 최종 상관 히트맵으로 시각화 후 육안 검토
import seaborn as sns
import matplotlib.pyplot as plt

final_features = [...]  # Step 2 이후 잔존 목록
corr_matrix = df[final_features].corr()
mask = corr_matrix.abs() < 0.5  # 0.5 미만은 시각적으로 흐리게

sns.heatmap(corr_matrix, annot=True, fmt='.2f', cmap='RdYlGn',
            center=0, vmin=-1, vmax=1, mask=mask)
plt.title('레짐 모델 입력 피처 상관 행렬 (|r| ≥ 0.5 표시)')
```

|------|----------|------|

### 17-4. 파라미터 수·데이터 비율 검증

#### 전체 데이터 비율 (K=3, p=7, full)

```
전이확률 행렬:     K×K = 9
평균 벡터:         K×p = 21
공분산 행렬(full): K × p×(p+1)/2 = 3 × 28 = 84
초기 확률:         K = 3
─────────────────────────────
총 파라미터:       117개

2016-2025 관측일:  ~2520일
전체 비율:         2520 / 117 ≈ 21.5 : 1   ← 권장 10:1 이상 충족 ✅
```

#### 레짐별 실효 비율 (실제 제약 조건)

전체 비율은 안전해 보이나, 파라미터 추정은 레짐별로 독립적으로 이루어진다.
위기 레짐의 공분산 행렬 28개는 오직 위기 레짐으로 분류된 날에서만 학습된다.

```
─────────────────────────────────────────────────────────────
레짐            추정 관측일    레짐 내 파라미터    실효 비율
─────────────────────────────────────────────────────────────
안정 (0)        ~1,760일      μ(7) + Σ(28) = 35    50.3 : 1  ✅✅
중립 (1)        ~510일        μ(7) + Σ(28) = 35    14.6 : 1  ✅
위기 (2)        ~250일        μ(7) + Σ(28) = 35     7.1 : 1  ✅
─────────────────────────────────────────────────────────────
* 안정:중립:위기 = 70:20:10 비율 가정 (역사적 비율 근거)
```

위기 레짐 7.1:1은 권장 기준(≥5:1)을 충족하며, p=7로 피처를 줄인 결과다.
(p=10일 경우 위기 레짐 실효 비율: 250/65 = 3.8:1 → ❌)

#### p=10 vs p=7 위기 레짐 비율 비교

```
─────────────────────────────────────────────
피처 수   레짐당 파라미터   위기 실효 비율
─────────────────────────────────────────────
p=10      μ(10)+Σ(55)=65    250/65 = 3.8:1  ❌
p=7       μ(7)+Σ(28)=35     250/35 = 7.1:1  ✅
─────────────────────────────────────────────
```

피처 수를 10→7로 줄이는 것만으로 위기 레짐 추정 안정성이 크게 개선된다.
이것이 17-3의 3단계 피처 선택 프로세스를 통해 p≤7을 목표로 삼는 근거다.

### 17-5. Walk-Forward 윈도우 구성 — 최종 확정

ML 수익률 예측 모델의 IS/OOS 구성에 대해 다음 안들을 비교 검토하였다.

| 구성 | IS | OOS | 독립 관측 수 | 비고 |
|------|-----|-----|------------|------|
| 안 1 (초기) | 150일 | 30일 | 150 / 30 = 5개 | 통계 검정력 매우 약함 |
| 안 2 (BL 동기화) | 500일 | 63일 | 500 / 30 ≈ 17개 | BL 리밸런싱(분기)과 동기화 |

**채택: IS=500일 / OOS=63일**

- 30일 forward return 타겟 사용 시 독립 관측: 500 / 30 ≈ 17개 (통계적으로 의미 있는 최소 수준)
- OOS 63일 = 분기(BL 리밸런싱 주기)와 동기화 → 예측 수익률이 실제 리밸런싱에 그대로 반영
- 윈도우 수: (2520 - 500) / 63 ≈ 32개 (Section 12의 백테스팅 윈도우 수와 동일)

> 참고: OOS 기간 중 30일 forward return은 날짜가 중복(overlapping)되므로,
> 통계 검정 시 Newey-West 표준오차 또는 Bootstrap 사용 필요.
