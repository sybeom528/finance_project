## 2. BIC 공식의 수정

### 2.1 HMM 완전 파라미터 수

HMM의 파라미터는 4개 군으로 구성됩니다:

| 파라미터 | 자유도 | 의미 |
|----------|--------|------|
| π (초기 상태 분포) | `n - 1` | 첫 시점의 상태 확률 |
| A (전이 행렬) | `n · (n - 1)` | 상태 간 전이 확률 (각 행 합 = 1) |
| μ (방출 평균) | `n · d` | 각 상태별 `d`차원 평균 벡터 |
| Σ (방출 공분산, full) | `n · d(d+1)/2` | 각 상태별 대칭 공분산 행렬 |
| **합계 k** | **`(n-1) + n(n-1) + n·d + n·d(d+1)/2`** | — |

### 2.2 이전 공식의 오류

v4.1까지 사용된 공식은 π 자유도를 누락했습니다:
```
이전: k = n(n-1) + n·d + n·d(d+1)/2   (π 누락)
수정: k = (n-1) + n(n-1) + n·d + n·d(d+1)/2
```

### 2.3 실측 영향 (d=5, full covariance)

| n | k_old | k_new | Δ |
|---|-------|-------|---|
| 2 | 42 | 43 | +1 |
| 3 | 66 | 68 | +2 |
| 4 | 92 | 95 | +3 |
| 5 | 120 | 124 | +4 |
| 6 | 150 | 155 | +5 |
| 7 | 182 | 188 | +6 |
| 8 | 216 | 223 | +7 |

### 2.4 결과: 최적 n 선택에 영향 없음

모든 n에서 `k_new = k_old + (n-1)`로 일정 상수 추가 → BIC는 각 n에서 `+(n-1)·log(T)` 증가. **상대 순위는 거의 동일**합니다.

그러나 **공식의 학술적 완전성**을 위해 수정 공식을 채택합니다.

## 4. BIC 단독 기준의 한계 실증

### 4.1 이론적 한계

HMM은 다음과 같은 이유로 BIC의 regularity conditions를 완전히 만족하지 않습니다:

| 조건 | HMM 위반 |
|------|---------|
| Identifiability | label switching (상태 순서 치환) |
| Non-boundary parameters | 전이 확률이 0/1에 수렴 가능 |
| Non-singular Fisher information | 고차원 공분산에서 자주 경계 |

결과적으로 BIC의 "과적합 penalty `k·log(T)`"가 HMM의 실제 복잡도를 과소평가할 수 있습니다 (Celeux & Durand, 2008).

### 4.2 실증적 한계: 가짜 엘보우

본 데이터에서 **n=6→7에서 ΔBIC=-121**로 평탄해져 엘보우처럼 보였으나, **n=7→8에서 ΔBIC=-1,311로 다시 급감**. 이는:

- n을 더 확장하면 (n=9, 10, 15 등) **계속 하락 가능성 높음**
- BIC만으로 "멈출 지점"을 결정할 수 없음
- 순수 BIC 추종 = **무한히 복잡한 모델 선호 경향**

### 4.3 AIC, ICL도 동일 문제

| 기준 | 최적 n | 문제 |
|------|--------|------|
| BIC | 8 | 단조 감소 |
| AIC | 8 | 단조 감소 (더 심함) |
| ICL | 8 | 단조 감소 |

**HMM 전문 지표 ICL도 동일한 문제** → 정보 기준 **일체를 단일 기준으로 사용 불가**

### 6.2 학술적 방어 포지션

"BIC가 n=8을 선호하나, HMM의 BIC는 regularity conditions 미충족 특성상 단조 감소 경향이 있어 단일 기준으로 부적절. Celeux & Durand (2008)의 제안에 따라 다기준 평가를 병행한 결과, 해석성·파이프라인 안정성·Rare state 회피 등을 종합하여 **n=4**를 정책적으로 채택. 이는 Guidolin & Timmermann (2007) 등 금융 실증 문헌의 주류 관행과 일치."

### 8.3 B1 (Alert 당일 적용) — 메커니즘 상세

```
1. 장 마감 (16:00 EST):
   VIX_t가 SPX 옵션 close 가격 기반 확정
   → alert_t = f(VIX_t, HY_t, ...) 계산 가능
   → 즉 close_t 시점에 "오늘 시장이 어땠는지" 이미 관측됨

2. Look-ahead 발생:
   alert_t로 당일 비중 결정 → return_t 평가
   = "오늘 장이 나쁠 것을 이미 안 상태에서 오늘 리밸런싱"

3. 수정 (shift(1)):
   alert_t는 내일(t+1)부터 적용 가능
   → return_t는 정상 비중으로 평가 (전일까지의 alert 반영)
```

## 11. Step 10 통계 검정 Family 정의 (v4.2b 명시)

### 11.1 Bonferroni 보정 분모 = 3의 의미

**코드**: `Step10_Ablation_Final.ipynb` Cell 8
```python
boot_df['bonferroni_sig'] = boot_df['p_value'] < (0.05 / 3)
```

### 11.2 Family 정의: "성향·Config별 M0 대비 비교"

본 파이프라인의 **family-wise error 단위**는 다음과 같이 정의됩니다:

```
family = (profile, config)
→ 각 family 내에서 M0 대비 3개 비교: M1 vs M0, M2 vs M0, M3 vs M0
→ Bonferroni α/3 = 0.0167 적용
```

**추가 비교 (family-wise 외)**: M3 vs M1 직접 검정은 경로 2 부가가치 검증용 보조 통계 → Bonferroni 적용 제외.

### 11.3 대안 해석: "전체 64 비교 family"

엄격한 통계학적 관점에서 **전 비교**를 하나의 family로 보면:
- 4 성향 × 4 Config × 4 비교 (M1/M2/M3 vs M0 + M3 vs M1) = **64 비교**
- Bonferroni α/64 ≈ 0.00078
- **유의 건수 거의 0으로 수렴** (M2_보수형의 "60/40 우위"도 이 기준에는 비유의)

### 11.4 채택 근거: "성향별 family" 채택

다음 4가지 이유로 **"성향·Config별 family" 해석 채택**:

1. **독립적 의사결정 가정**: 투자자는 자기 성향에 맞는 하나의 Config만 선택 → 4×4=16 독립 의사결정
2. **Feature-specific family**: Bonferroni의 보수성은 "한 번에 여러 추론"에 적용. 성향별 선택은 추론이 분리됨
3. **관련 문헌**: 포트폴리오 전략 비교 논문에서 성향별·Config별 family 보편적 (Lo 2002, Bailey & López de Prado 2014)
4. **실증적 결과**: Block Bootstrap + FDR 결과는 "성향별 family" 기준과 일관

### 11.5 명시적 한계 선언

- 본 파이프라인의 유의 결과는 **"각 성향·Config 하위체계 내 통계적 유의"**로 한정 해석
- "64 전체 family 기준의 엄격 유의"를 주장하지 않음
- 이 한계는 decision_log §11.3에 명시되어 학술적 방어 가능
