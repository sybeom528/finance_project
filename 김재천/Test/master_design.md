## 1. 프로젝트 개요

### 목표
Black-Litterman 포트폴리오에 ML 기반 수익률 예측(Q 벡터)을 통합하여
시장 국면(레짐)을 고려한 동적 자산 배분 시스템 구축

### 핵심 인과 체인
```
레짐 판별(HMM) → ML 수익률 예측 → Black-Litterman → 포트폴리오 비중 → Sharpe 개선
```

### 분석 기간
```
2016-01-01 ~ 2025-12-31 (10년, ~2520 거래일)
```

### 투자 유니버스 (30개)
```
인덱스 ETF (5):  SPY, QQQ, IWM, EFA, EEM
채권 ETF (4):   TLT, AGG, SHY, TIP
대안 ETF (2):   GLD, DBC
섹터 ETF (11):  XLK, XLF, XLE, XLV, VOX, XLY, XLP, XLI, XLU, XLRE, XLB
개별 종목 (8):  8개 GICS 섹터 시총 1위 (연간 재구성)
```

## 2. 전체 아키텍처

```
[Step 1] 데이터 수집
    주가 30종목 + 대안데이터 + 외부지표
         ↓
[Step 2] 전처리 / EDA
    결측치 처리, 수익률 계산, Granger 검정
         ↓
[Step 3] 레짐 판별 — HMM
    K=3 (안정/중립/위기), 전체 시계열 1회 학습
    출력: 날짜별 regime ∈ {0, 1, 2}
         ↓
[Step 4] ML 수익률 예측 — Walk-Forward
    입력: 모멘텀 + 변동성 + regime
    출력: Q 벡터 (30×1 예측 수익률)
         ↓
[Step 5] Black-Litterman
    Q + 시장 균형 π → μ_BL
    Ω: Ablation 대상 (방법 A / B / 베이지안)
         ↓
[Step 6] 포트폴리오 최적화
    MV (주) + RP / HRP (비교)
    공분산: 계층형 추정
         ↓
[Step 7] 경보 시스템 (별도 HMM)
    L0~L3 경보 → 긴급 비중 축소
         ↓
[Step 8] Walk-Forward 백테스트
    IS / OOS 슬라이딩 — [미결: 분기 vs 월]
         ↓
[Step 9] 리스크 분석 + Ablation
    VaR/CVaR, 스트레스 11개, 축 4개 비교
```

## 4. 레짐 모델 — HMM

### 목적
ML 피처로 사용할 시장 국면 라벨 생성
(Step 7 경보 시스템의 HMM과 별개)

### 설계 확정
```
K = 3           (안정=0 / 중립=1 / 위기=2)
covariance_type = 'full'
p = 7           (매크로 피처)
min_covar = 1e-3 (수치 안정화)
```

### 입력 피처 (잠정 7개)
```
그룹 A. 주식시장 공포:  VIX_level
그룹 B. 신용·유동성:    HY_spread
그룹 C. 금리·경기:      T10Y2Y, MOVE_index
그룹 D. 실물 경기:      (v5 도입 예정 — UNRATE z-score, claims_zscore 등 검토)
그룹 E. GDELT 감성:     gdelt_mkt_tone_avg, gdelt_mkt_event_cnt
※ GDELT 활용 여부 미결 → 대체 피처 검토 필요
```

### 파라미터 안정성 검증
```
총 파라미터:       117개 (K×K + K×p + K×p(p+1)/2 + K)
전체 비율:         2520 / 117 = 21.5:1  ✅
위기 레짐 실효비율: 250 / 35  = 7.1:1   ✅ (≥5:1 기준 충족)
```

### 학습 방식
```
[설계 의도] Full-sample 학습 + 시점별 Forward 추론

1) 학습 단계 (1회):
   - 전체 시계열 2016~2025 대상 1회 학습 → 전이행렬 A, 방출분포(μ, Σ) 추정
   - 의도: 짧은 IS 윈도우(예: 24개월)로 매번 재학습할 경우
     (a) 위기 레짐 관측이 거의 없는 윈도우에서 추정 불안정,
     (b) 윈도우마다 레짐 정의가 달라져 비교 불가능,
     (c) Label switching 보정 매번 수행 필요 → 운영 복잡도 ↑.
   - 따라서 "레짐 정의는 고정 (full-sample 학습), 시점별 판단은 Forward로" 분리.

2) 추론 단계 (시점별):
   - 각 시점 t의 레짐 분류는 Forward 알고리즘으로 t시점까지 관측치만 사용
   - Viterbi(전체 관측 사용) 대비 Forward(인과적)로 교체 — 2026-04-19 update
   - 실측 차이: Forward vs Viterbi 불일치 39일 (1.57%)
   → Strict OOS와 호환: 미래 관측이 t시점 분류에 들어가지 않음

재보정 트리거:
  정기: 매년 1월 (전이행렬·방출분포 재추정)
  긴급: KL divergence 임계값 초과 시
```

### Label Switching 처리
```python
# VIX 평균 낮은 순서대로 0(안정), 1(중립), 2(위기) 재부여
vix_mean_by_regime = data.groupby(raw_labels)['VIX'].mean()
mapping = {old: new for new, old
           in enumerate(vix_mean_by_regime.sort_values().index)}
aligned_labels = raw_labels.map(mapping)
```

### K=3 채택 근거
```
K=4 대비:
  위기 레짐 관측일: ~250일(K=3) vs ~90일(K=4) → K=4 추정 불안정
  파라미터 비율:     12:1(K=3)  vs  9:1(K=4)  → K=3 안전 범위
  해석 용이성:       3단계 = 투자 행동 3가지와 1:1 대응
BIC로 사후 검증 예정
```

### covariance_type = full 채택 근거
```
레짐별 피처 간 상관 구조가 다름:
  안정 레짐: VIX ↔ HY_spread 상관 ~0.3 (독립적)
  위기 레짐: VIX ↔ HY_spread 상관 ~0.85 (동시 폭발)
→ full만이 이 동시 급등 패턴 포착 가능
→ p=7로 줄여 위기 레짐 실효 비율 7.1:1 확보
```

## 6. Black-Litterman

> 🚧 **현 파이프라인(v4.x)에서는 미구현** — v5 도입 예정.
> 본 절은 v5 설계 명세. v4.x는 BL 없이 MV 최적화 + 레짐별 공분산 + 경보 시스템으로 운영.
> π/Q/μ_BL 계산, Ω Ablation은 v5에서 진행.

### 목적
ML 예측(Q)과 시장 균형(π)을 베이지안 결합
→ 극단적 비중 방지 + 분산 유지

### 수식
```
μ_BL = [(τΣ)⁻¹ + Ω⁻¹]⁻¹ × [(τΣ)⁻¹π + Ω⁻¹Q]

P = I₃₀  (절대 View, 종목별 독립 예측이므로 소거)
τ = 1/IS기간
π = δ × Σ × w_mkt  (CAPM 역산 시장 균형 수익률)
Q = ML 예측 수익률
Ω = ML 예측 신뢰도 (Ablation 대상)
```

### Ω 산출 방법 — Ablation 대상
```
방법 A (변동성 기반):
  Ω_i = τ × σ²_i
  장점: 즉시 사용 가능, BL 원논문 일치
  단점: ML 실측 성능 미반영

방법 B (OOS 오차 기반):
  Ω_i = Var(OOS 예측 오차₁ ... 오차ₙ₋₁)
  장점: ML 성능 직접 반영
  단점: 초반 윈도우 사용 불가

베이지안 (A→B 점진 전환):
  Ω_i = w × Var(오차) + (1-w) × (τ × σ²_i)
  w = n / (n + k),  k: 최적화 파라미터
  k 결정 방법: Walk-Forward 완료 후
               L(k) = Σ(Ω추정 - 실제오차²)² 최소화
```

### δ (위험회피계수) ✅ **확정 (v4.2d): 고정값 (성향별)**
```
보수형: γ = 8
중립형: γ = 4
적극형: γ = 2
공격형: γ = 1

채택 근거:
  (1) 성향 프리셋(§7)과 1:1 대응 — 투자자가 자기 성향만 고르면 됨
  (2) 시장 Sharpe 역산은 매 윈도우 변동 → 비교 어려움
  (3) BL이 미구현이므로 현 시점에서 δ는 문서 정책 차원
  (4) v5 BL 도입 시 동일 γ 값을 δ로 사용
```

## 7. 포트폴리오 최적화

### 전략
```
주 전략:  MV (Mean-Variance) — γ가 직접 반영, 성향 맞춤
비교 전략: RP (Risk Parity)  — μ 추정 불필요, MV 과적합 검증
          HRP (Hierarchical) — 공분산 역행렬 불필요, 가장 강건
```

### 공분산 추정 (계층형)
```
Level 1 (자산군 간):  블록 대각 — 대표 ETF 3개(SPY, AGG, GLD) 간 3×3
Level 2 주식 (24개):  PCA 팩터 모형 (K 자동 선택, 누적 분산 80%)
Level 2 채권 (4개):   Ledoit-Wolf
Level 2 대안 (2개):   직접 추정
```

### 투자자 성향 (4개 프리셋)
```
보수형 γ=8: 목표변동성 7%,  주식 상한 30%, 채권 하한 40%
중립형 γ=4: 목표변동성 10%, 주식 상한 55%, 채권 하한 20%
적극형 γ=2: 목표변동성 15%, 주식 상한 75%, 채권 하한 10%
공격형 γ=1: 목표변동성 22%, 주식 상한 90%, 채권 하한 0%
```

### 비중 상한 (HHI 기반)
```
인덱스 ETF: MDD역산 × 1.4
섹터 ETF:  MDD역산 × 1.1
개별 종목: MDD역산 × 1.0
```

### 리밸런싱
```
정기:  OOS 기간마다 (분기 또는 월 — Walk-Forward와 동기화)
긴급:  L2 경보 발동 시 즉시 주식 비중 축소
```

## 9. Walk-Forward 백테스트

### 구조 ✅ **확정 (v4.2d)**
```
채택: 안 A — 분기

  IS=504일 / OOS=63일 / 윈도우 31개
  리밸런싱: 분기 1회 (연 4회)
  거래비용: 편도 15bps × 4회 = 연 ~120bps

채택 근거: §5 「Walk-Forward 구조」 참조
```

### 각 윈도우 수행 순서
```
1. IS 데이터로 ML 재학습
2. OOS 첫날 Q 생성 → BL → μ_BL
3. MV 최적화 → 비중 w
4. OOS 기간 보유 → 수익률 기록
5. 경보 발동 시 긴급 비중 조정
6. 다음 윈도우로 이동
```

### 거래비용
```
편도 10bps + 슬리피지 5bps = 왕복 30bps
```

## 10. 리스크 분석

### 측정 지표
```
VaR / CVaR
리스크 기여도 (종목별)
```
