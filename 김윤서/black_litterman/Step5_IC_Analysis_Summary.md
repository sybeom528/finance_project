# Step 5 Black-Litterman — IC 분석 및 설계 결정 정리

## 1. 프로젝트 맥락

Step 5는 11개 GICS 섹터 × 시총 Top5 = **55개 개별주식 + 6개 채권/대안 ETF = 61개 자산**을 대상으로 Black-Litterman 모델을 구현한다.

| 구성 요소 | 방법 | 근거 |
|-----------|------|------|
| Prior (π) | Risk Parity (ERC 최적화) | 채권 포함 시 시가총액 가중 불가 — Maillard et al. (2010) |
| View (Q) | 6/9/12개월 연환산 로그수익률 평균 | 횡단면 모멘텀 — Asness et al. (2013) |
| Uncertainty (Ω) | Idzorek z-score 방식 | 신뢰도 c → Ω_ii = ((1-c)/c)·τ·Σ_ii |
| τ | 1/252 | 하루치 분산 스케일 — He & Litterman (1992) |

---

## 2. τ(tau) 파라미터에 대한 분석

### τ를 변경해도 성능이 바뀌지 않는 이유

BL 공식에서 τ가 관여하는 두 항목:

```
Prior precision : (τΣ)⁻¹  ∝  1/τ
Omega           : Ω_ii = ((1-c)/c) · τ · Σ_ii  →  Ω⁻¹  ∝  1/τ
```

**두 정밀도 모두 1/τ에 비례** → τ가 바뀌어도 Prior:View 비율은 불변.

τ = 1/252 → τ = 1 로 바꿔도 μ_BL이 변하지 않는다.

> **실질적으로 Prior/View 균형을 조정하는 파라미터는 τ가 아니라 Idzorek의 c_min/c_max다.**
> c를 높이면 Ω_ii가 작아지고 → View precision이 높아져 → Q가 μ_BL에 더 강하게 반영된다.

## 3. IC(Information Coefficient) 개념

### 횡단면 IC vs 시계열 IC

| | 횡단면 IC | 시계열 IC |
|---|---|---|
| **질문** | "이 시점에 모멘텀 높은 종목이 더 오르는가?" | "이 자산의 모멘텀이 높을 때 그 자산이 더 오르는가?" |
| **단위** | 시점당 Spearman 상관계수 1개 | 자산당 Spearman 상관계수 1개 |
| **BL Q 검증** | **직접 검증 (올바른 방법)** | 간접 — 개별 타이밍 전략 |
| **계산 방식** | t 시점에서 61개 자산의 모멘텀 순위 vs 미래수익 순위 | 단일 자산의 모멘텀 시계열 vs 미래수익 시계열 |

우리 Q는 "여러 자산 중 어느 자산이 더 오를 것인가"라는 **상대적 뷰**이므로 **횡단면 IC**가 올바른 평가 지표다.

### Overlapping vs Non-overlapping IC

| | 매 거래일 계산 (overlapping) | 21일 간격 (non-overlapping) |
|---|---|---|
| **관측 수** | 2500+ | 119 |
| **선행수익률 겹침** | 인접 관측 20일 겹침 | 겹침 없음 |
| **t-통계량** | 자기상관으로 부풀림 | 독립 관측 → 신뢰 가능 |
| **189일 IC 예시** | +0.0026 (양수처럼 보임) | -0.0013 (사실상 0) |

overlapping IC는 같은 수익률 사건이 여러 번 중복 반영되어 t-통계량을 과장한다.

## 4. 시계열 IC -0.4 ~ -0.5 문제

### 원인

단일 종목 시계열 IC를 60일 rolling window로 계산하면:
- window=126일 모멘텀의 경우 인접 두 관측값이 **125일 겹침**
- rolling window 60개 안에서 이 극단적 자기상관이 rank 순위를 고정
- 개별 주식의 평균회귀 효과 + 오버랩 편향 → IC -0.4~-0.5

### 실제 의미

| IC 측정 방식 | 값 (예시) | 의미 |
|---|---|---|
| 시계열 IC (60일 rolling) | -0.4 ~ -0.5 | 오버랩 편향 + 개별 주식 평균회귀 — **사용 불가** |
| Non-overlapping 횡단면 IC | -0.03 ~ 0.00 | BL Q 신뢰성 실제 측정값 |

## 5. Non-overlapping 횡단면 IC 분석 결과

### 윈도우별 전체 평균 IC

| 윈도우 | Mean IC | Hit Rate | t-통계량 | 해석 |
|--------|---------|----------|----------|------|
| 21일  | -0.0317 | 45.4%    | -1.78*   | 단기 반전 효과 유의 |
| 42일  | -0.0271 | 53.8%    | -1.47    | n.s. |
| 63일  | -0.0209 | 50.4%    | -1.09    | n.s. |
| 126일 | -0.0100 | 52.1%    | -0.48    | n.s., 거의 0 |
| 189일 | -0.0013 | 52.9%    | -0.06    | **사실상 0** — 가장 중립 |
| 252일 | -0.0118 | 46.2%    | -0.56    | n.s. |

### Mean IC 음수인데 Hit Rate 50% 이상인 이유

**수익률 분포의 비대칭성 (모멘텀 크래시):**

```
IC 분포:
  53.8% 기간  : 소폭 양수 IC (+0.01 ~ +0.05)  ← 모멘텀 작동
  46.2% 기간  : 가끔 대폭 음수 IC (-0.3 ~ -0.6) ← 모멘텀 크래시

평균 = 0.538 × (+0.03) + 0.462 × (-0.09) = -0.027
```

→ 대부분의 기간에서 모멘텀은 소폭 작동하지만,  
→ 2020 COVID 충격, 2022 금리 급등 같은 시장 반전 시 IC가 대폭 음수로 떨어져 평균을 끌어내림.

**Daniel & Moskowitz (2016) "Momentum Crashes"** 에서 검증된 현상.

## 6. 자산별 IC 분석 결과

### IC가 양수인 자산군

| 윈도우 | 양수 IC 자산 | 해석 |
|--------|------------|------|
| 63일   | GE(+0.307), WELL(+0.130), NEM(+0.118), SPG(+0.107) | 산업재·부동산·소재 — 2~3개월 모멘텀 유효 |
| 126일  | BA(+0.158), TSLA(+0.096), TMUS(+0.090) | 중기 모멘텀 작동 |
| 189일  | SHY(+0.382) | 단기국채 — 금리 방향성 9개월 지속 |
| 252일  | AGG(+0.164), META(+0.077) | 채권 + 소수 주식 장기 모멘텀 |

### 히트맵에서 관찰된 패턴

- **채권/대안 (SHY, AGG, TLT, GLD)**: 중장기(126~252일)에서 초록 많음 → 금리 추세의 지속성 반영
- **개별 주식 대부분**: 전반적으로 빨강 → 단기 평균회귀 효과 지배
- **예외적 주식 (GE, BA, TSLA)**: 특정 중기 윈도우에서만 양수 IC → 섹터 모멘텀이 있을 때만 작동

### 주의: 최신 유니버스 기준으로 계산

IC 분석은 **2025년 말 기준 유니버스(55개 주식)를 고정**하고 2016~2025 전체 기간을 분석한다.  
백테스트에서는 유니버스가 매년 갱신되므로 완전히 동일하지 않다.  
(예: TSLA는 2020년 S&P500 편입 전 데이터도 IC 계산에 포함됨)

> **논문 서술 필요**: "IC 분석은 생존편향이 존재하는 최신 유니버스 기준으로 계산되었으며, 실제 백테스트 유니버스와 완전히 일치하지 않는다."

## 7. BL Q 신뢰성 종합 평가

| 평가 항목 | 결과 | 판단 |
|-----------|------|------|
| 전체 평균 IC | -0.003 ~ -0.032 (모두 음수) | 전체 자산 평균으로는 모멘텀 효과 미미 |
| t-통계량 유의성 | 21일만 유의(-1.78*), 나머지 n.s. | 통계적으로 대부분 0과 구분 불가 |
| Hit Rate | 42~189일은 50% 이상 | 과반 기간에서 방향은 맞음 |
| 양수 IC 자산 존재 | 채권(SHY, AGG) + 일부 산업재/부동산 | 서브셋에서는 유효한 신호 존재 |
| 백테스트 Sharpe | Neutral 0.850, Conservative 0.871 > SPY 0.832 | 실제 성능은 SPY 초과 |

### 성능이 괜찮은 이유 (IC 약함에도 불구하고)

1. **Prior 자체가 좋은 출발점**: Risk Parity Prior는 다각화된 포트폴리오를 산출
2. **BL의 자동 조절**: Idzorek Omega는 IC 약한 자산의 Q 영향력을 자동으로 낮춤  
   (신뢰도 c가 낮을수록 Ω 커짐 → View precision 낮아짐)
3. **Step 3 ML Q로 대체 시 개선 기대**: XGBoost/TabPFN Q는 모멘텀보다 높은 예측력 보유
4. **현재 Step 5는 모멘텀 베이스라인**: ML Q 통합 전 기준점으로서의 역할

## 8. 향후 적용 방향

```
현재 (Step 5):
  모멘텀 Q (126/189/252일 평균) → IC 약함 → BL 효과 제한적
  → Prior (Risk Parity) 중심으로 작동

Step 3 ML Q 통합 후:
  XGBoost/TabPFN Q → 높은 예측력 → c_max 상향 조정 가능
  → BL이 Q를 더 강하게 반영 → 진정한 Black-Litterman 효과
```

**Q 영향력 강화 방법**: τ 변경(효과 없음) → `c_max` 상향 조정
```python
# 현재
compute_omega_idzorek(Q, cov, tau, c_min=0.10, c_max=0.90)

# ML Q 통합 후 (신뢰도 상향)
compute_omega_idzorek(Q, cov, tau, c_min=0.30, c_max=0.95)
```
