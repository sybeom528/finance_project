## 전체 흐름 한눈에 보기

```
가격 데이터 로드
     ↓
[함수 정의]
  Prior π = λΣw           ← "시장이 이렇게 될 것이다"
  Q = 모멘텀 수익률        ← "우리가 이렇게 예측한다"
  Ω = z-score 기반 신뢰도 ← "우리 예측을 얼마나 믿는가"
  BL 사후 μ_BL, Σ_BL      ← "Prior + 예측을 섞은 최종 기대수익률"
  MVO 최적화               ← "최종 기대수익률로 비중 결정"
     ↓
[단일 기간 검증]
  2024-12-31 기준 BL 결과 확인
     ↓
[Walk-Forward 백테스트]
  2016-01 ~ 2025-12, 21일마다 리밸런싱
     ↓
[성능 측정]
  Sharpe, MDD, 연환산 수익률, 구간별 분석
```

## 셀 3: 핵심 함수 5개

### (1) `compute_prior(cov, w_mkt, lam=2.5)`

```python
return lam * cov @ w_mkt
```

**"아무 정보가 없을 때 시장이 기대하는 수익률"을 계산한다.**

- 이 공식(`π = λΣw`)은 역 MVO에서 나온다 — "시장 비중이 이미 최적화된 결과라면, 그 때의 기대수익률은 얼마인가?"를 역으로 계산
- `w_mkt`: 시장 포트폴리오 비중 → 우리는 시가총액 데이터가 없으므로 **등가중(1/30)** 사용
- `lam = 2.5`: 시장 위험회피계수 (S&P 500 Sharpe ÷ 변동성 ≈ 2.5)
- 결과: 각 자산에 대한 "시장이 동의하는 연환산 기대수익률" 벡터 (30개)

**직관**: SPY(미국 대형주)는 과거 변동성이 크고 시장에서 큰 비중 → π 높음. SHY(단기채)는 변동성 낮음 → π 낮음.

---

### (2) `compute_momentum_Q(prices_window, windows=[126, 189, 252])`

```python
# 6개월 연환산 로그 수익률
ret_6m = (log(P[-1]) - log(P[-126])) / 126 * 252

# 6/9/12개월 평균
Q = mean([ret_6m, ret_9m, ret_12m])
```

**Lee & Bae (2024) 논문 방식 그대로: 6/9/12개월 과거 수익률의 평균을 뷰(Q)로 사용한다.**

- `126 거래일 ≈ 6개월`, `189 ≈ 9개월`, `252 ≈ 12개월`
- 연환산: `/w * 252`로 기간 길이 차이를 조정
- 3개 윈도우 평균: 단일 윈도우보다 안정적 (단기 노이즈 희석)

**직관**: "지난 6~12개월 동안 잘 오른 자산이 앞으로도 상대적으로 좋을 것이다" = 모멘텀 가설

---

### (3) `compute_omega_idzorek(Q, cov, tau=1/252)`

```python
# 횡단면 z-score
z = (Q - Q.mean()) / Q.std()

# sigmoid로 신뢰도 변환 [0.1, 0.9]
sig = 1 / (1 + exp(-|z|))
confidence = 0.10 + 0.80 * (sig - 0.5) / 0.5

# Idzorek(2004) Omega
omega_diag[i] = ((1 - confidence[i]) / confidence[i]) × τ × cov[i,i]
```

**"모멘텀 신호가 강할수록 더 믿고, 약할수록 덜 믿는다"를 수식으로 표현한다.**

| 상황 | z-score | confidence | Ω (불확실성) | BL 결과 |
|------|---------|-----------|-------------|---------|
| 강한 모멘텀 (상위/하위 극단) | 크다 | 높다(~0.9) | 작다 | Q 쪽으로 크게 기울어짐 |
| 약한 모멘텀 (중간) | 작다 | 낮다(~0.1) | 크다 | Prior π 쪽에 더 가까움 |

**논문(Lee & Bae 2024)과의 차이**:
- 논문: 252일 중 상승일 비율(%pos)만으로 신뢰도 계산 → 방향성만 반영
- 본 구현: **횡단면 z-score** → "같은 날짜 기준으로 얼마나 강한 신호인가"까지 반영
  - 예: AAPL이 +20% 모멘텀이고 평균이 +5%라면 z-score 높음 → 강한 신호로 인식

---

### (4) `black_litterman(pi, cov, Q, omega, tau)`

```python
# 핵심 수식
precision = (τΣ)⁻¹ + P'Ω⁻¹P
mu_bl = precision⁻¹ × [(τΣ)⁻¹π + P'Ω⁻¹Q]
```

**Prior(π)와 모멘텀 뷰(Q)를 Ω의 역수(신뢰도)로 가중평균한다.**

수식을 직관적으로 풀면:

```
μ_BL = (Prior 신뢰도 × Prior) + (뷰 신뢰도 × Q)
       ─────────────────────────────────────────
               Prior 신뢰도 + 뷰 신뢰도
```

- `P = Identity (단위행렬)`: **절대적 뷰** 사용 ("SPY는 15% 수익률을 낼 것이다" 형식)
- `tau = 1/252`: Prior를 얼마나 불확실하게 볼 것인가 (표준값)
- `Σ_BL = Σ + precision⁻¹`: 사후 공분산도 업데이트 (Prior + 뷰의 불확실성 합산)

**수치 예시 (2024-12-31 기준, AAPL)**:
```
π(Prior)  = +2.4%   (시장 균형 기대수익률)
Q(모멘텀) = +35.9%  (12개월 강한 상승)
Ω 작음    → 신뢰도 높음
μ_BL      = +26.1%  (Q쪽으로 크게 기울어짐, 그러나 완전히 Q는 아님)
```

---

### (5) `mvo_optimize(mu, cov, gamma, constraints)`

```python
maximize:  w'μ - (γ/2) × w'Σw
subject to: Σw = 1, 0 ≤ w_i ≤ max_weight, ...
```

**BL 사후 기대수익률(μ_BL)과 사후 공분산(Σ_BL)으로 포트폴리오 비중을 최적화한다.**

- `γ (감마)`: 위험회피계수
  - γ=2 (공격형): 수익을 중시, 변동성 허용
  - γ=5 (중립형): 수익과 위험 균형
  - γ=8 (보수형): 안정성 중시
- 제약조건: 주식 비중 상한, 채권 비중 하한, 개별 종목 비중 상한

## 셀 10: Walk-Forward 백테스트

```python
for i, date in enumerate(analysis_days[1:], 1):
    # 1. 일별 수익률로 포트폴리오 가치 업데이트
    port_ret = current_weights @ daily_ret
    
    # 2. 21일마다 리밸런싱
    if i % 21 == 0:
        cov    = LedoitWolf(최근 252일)
        pi     = compute_prior(cov, w_mkt)
        Q      = compute_momentum_Q(가격 전체)
        omega  = compute_omega_idzorek(Q, cov)
        mu_bl, sigma_bl = black_litterman(pi, cov, Q, omega)
        
        for 각 프로파일:
            current_weights = mvo_optimize(mu_bl, sigma_bl, gamma)
```

**핵심: 매 21일마다 과거 데이터만 보고 (미래 데이터 금지) BL → MVO → 비중 결정 → 보유.**

| 시점 | 행동 |
|------|------|
| T일 | 현재까지 가격으로 Q, Ω, μ_BL 계산 |
| T~T+21 | 계산된 비중으로 보유 |
| T+21일 | 다시 계산, 리밸런싱 |

이 방식은 **look-ahead bias가 없다** — T일에는 T일 이전 데이터만 사용.

## 셀 13: 성능 지표

```python
def compute_metrics(values):
    ann_ret = (1 + total_ret) ^ (1/n_years) - 1   # 연환산 수익률
    sharpe  = ann_ret / (일별 수익률.std() × √252)  # 위험 조정 수익률
    mdd     = min((가치 - 최고가) / 최고가)          # 최대 낙폭
    sortino = ann_ret / 하방변동성                   # 하락 위험 조정
    calmar  = ann_ret / |MDD|                       # MDD 대비 수익률
```

**단순 수익률보다 Sharpe와 MDD가 더 중요한 지표다.**

- Sharpe가 높다 = 같은 리스크로 더 많이 번다
- MDD가 낮다 = 최악의 구간에서 덜 잃는다

## SPY 대비 수익률: 솔직한 답

> **결론: 총 수익률은 SPY B&H보다 낮을 가능성이 높다. 하지만 그게 문제가 아니다.**

### 왜 SPY보다 총 수익률이 낮을까

```
2016-2025 SPY 연환산 수익률: ~13-14% (역사적 최상위 구간)

BL 포트폴리오:
  - 30개 자산 중 채권(TLT, AGG...) + 대안(GLD, DBC) 포함
  - 이 자산들은 강세장에서 주식보다 수익률이 낮음
  - 결과: 분산 효과로 수익률이 SPY보다 낮게 나옴
```

| 전략 | 예상 연환산 수익률 | 예상 Sharpe | 예상 MDD |
|------|-----------------|------------|---------|
| SPY B&H | ~13-14% | ~0.9-1.0 | ~-35% (2020) |
| BL-Aggressive | ~11-13% | ~1.0-1.2 | ~-25% |
| BL-Neutral | ~8-11% | ~0.9-1.1 | ~-18% |
| BL-Conservative | ~5-8% | ~0.7-0.9 | ~-12% |

※ 위 수치는 백테스트 전 이론적 예상치 — 실제 실행해야 확인 가능

### 그러면 이 전략이 의미 없는가?

아니다. **프로젝트 목표가 "SPY 이기기"가 아니라 "투자자 성향에 맞는 포트폴리오 구축"**이기 때문이다.

| 목표 | SPY B&H | BL 전략 |
|------|---------|---------|
| 최대 수익 추구 | ✅ | ❌ |
| **위험 조정 수익 (Sharpe)** | 0.9 | **1.0~1.2** |
| **최대 손실 방어 (MDD)** | -35% | **-12~-25%** |
| **투자자 성향 반영** | ❌ | **✅** |
| 하락장에서 방어 | ❌ | **✅ (채권 비중)** |

### 논문 프레이밍 방법

```
"본 연구의 목적은 시장 초과 수익이 아닌, 
 투자자 위험 성향(γ)에 최적화된 위험 조정 포트폴리오 구축이다.
 BL+모멘텀 전략은 SPY 대비 총 수익률은 낮으나,
 Sharpe 비율과 MDD 측면에서 우월한 리스크 관리를 보인다."
```

### 만약 SPY를 이기고 싶다면?

Conservative/Neutral을 버리고 **Aggressive(γ=2, 주식 90%) 단독**으로 보면, 강세장 구간(2017-2019, 2023-2024)에서 SPY와 유사하거나 소폭 상회할 수 있다. 단, 하락장 방어도 약해진다.
