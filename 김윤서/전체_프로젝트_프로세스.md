## 전체 파이프라인 개요

```
[Step 1] 데이터 수집
    yfinance (자산 30개 + 외부지표 12개) + FRED (8개) + GDELT (뉴스 감성)
            ↓
[Step 2] 전처리 & EDA & 피처 엔지니어링
    정상성 검정 / Granger 인과 / 파생변수 생성 / GDELT 피처 통합
            ↓
[Step 3] 모델링 (롤링 윈도우 150일 학습 → 30일 예측)
    RF + XGBoost + ExtraTrees → Q(전망값), Ω(불확실성) 도출
            ↓
[Step 4] 블랙-리터만
    Q, Ω → 사후 기대수익률(μ) 도출
            ↓
[Step 5] MVO (평균-분산 최적화)
    μ + Σ + 위험회피계수(γ) → 성향별 최적 비중
            ↓
[Step 6] 백테스트 & 리스크 분석
    IS/OOS 성과 평가 / 스트레스 테스트 / 거래비용 반영
            ↓
[Step 7] 동적 리밸런싱 & 대시보드
    Streamlit 대시보드 구축
```

query = """
SELECT
    PARSE_DATE('%Y%m%d', CAST(DATE AS STRING)) AS date,

  AND (
        Themes LIKE '%ECON_STOCKMARKET%'
     OR Themes LIKE '%ECON_FRBRESERVE%'
     OR Themes LIKE '%ECON_INFLATION%'
     OR Themes LIKE '%ECON_UNEMPLOYMENT%'
  )
  AND SourceCommonName IN ('reuters.com','cnbc.com','wsj.com','ft.com','bloomberg.com')

"""

### 2.3 피처 엔지니어링 — 파생변수 전체 목록

#### A. 가격 기반 파생변수 (yfinance)

| 피처명 | 계산 방식 | 의미 |
|---|---|---|
| `mom_20` | 20일 수익률 평균 | 단기 모멘텀 |
| `mom_60` | 60일 수익률 평균 | 중기 모멘텀 |
| `mom_120` | 120일 수익률 평균 | 장기 모멘텀 |
| `vol_20` | 20일 수익률 표준편차 | 단기 변동성 |
| `vol_60` | 60일 수익률 표준편차 | 중기 변동성 |
| `rel_spy` | 종목 수익률 - SPY 수익률 | 시장 대비 상대강도 |
| `ma_gap_20` | 현재가 / 20일 이동평균 - 1 | 단기 이동평균 괴리율 |
| `ma_gap_60` | 현재가 / 60일 이동평균 - 1 | 중기 이동평균 괴리율 |

#### B. 외부지표 파생변수 (yfinance + FRED)

| 피처명 | 원천 | 의미 |
|---|---|---|
| `vix_level` | ^VIX | 시장 공포 수준 |
| `vix_term_spread` | VIX3M - VIX9D | VIX 기간구조 (선행 신호) |
| `yield_curve` | T10Y2Y (FRED) | 수익률 곡선 기울기 |
| `hy_spread` | BAMLH0A0HYM2 (FRED) | 하이일드 신용 스프레드 |
| `cu_au_ratio` | 구리선물 / 금선물 | 경기 선행 지표 |
| `dxy_level` | DX-Y.NYB | 달러 강도 |
| `sahm_indicator` | SAHMREALTIME (FRED) | 경기침체 조기 경보 |

#### C. GDELT 파생변수 ← 핵심

| 피처명 | 계산 방식 | 의미 |
|---|---|---|
| `fin_sentiment` | GCAM c15 긍정 - 부정 | 금융 특화 감성 점수 |
| `sent_momentum` | fin_sentiment - 20일 이동평균 | 감성 추세 이탈도 |
| `sent_volatility` | 20일 감성 표준편차 | 시장 의견 불일치 정도 |
| `sent_zscore` | (감성 - 60일 평균) / 60일 std | 감성 극단치 여부 |
| `news_spike` | 기사수 > 20일 평균×2 | 시장 주목 이벤트 발생 (0/1) |
| `crisis_signal` | avg_shock × fin_sentiment | 충격강도 × 감성 복합 위기지표 |
| `omega_proxy` | mention_tone_std 기반 | 뉴스 기사 간 의견 일관성 |

```python
def create_all_features(price_df, fred_df, gdelt_df):
    """
    전체 피처 생성 → 패널 데이터(long format) 변환

    출력 형태:
        date        ticker  mom_20  vol_20  ...  fin_sent  sent_z  target
        2023-01-01  SPY     0.012   0.018   ...   0.31     0.42      3
        2023-01-01  QQQ     0.015   0.021   ...   0.31     0.42      4
        2023-01-01  XLK     0.019   0.023   ...   0.31     0.42      4
    """
    ret = price_df.pct_change()
    spy_ret = ret['SPY']
    records = []

    for ticker in price_df.columns:
        df = pd.DataFrame(index=price_df.index)
        df['ticker']       = ticker

        # A. 가격 기반
        df['mom_20']       = ret[ticker].rolling(20).mean()
        df['mom_60']       = ret[ticker].rolling(60).mean()
        df['mom_120']      = ret[ticker].rolling(120).mean()
        df['vol_20']       = ret[ticker].rolling(20).std()
        df['vol_60']       = ret[ticker].rolling(60).std()
        df['rel_spy']      = ret[ticker] - spy_ret
        df['ma_gap_20']    = price_df[ticker] / price_df[ticker].rolling(20).mean() - 1
        df['ma_gap_60']    = price_df[ticker] / price_df[ticker].rolling(60).mean() - 1

        # B. 외부지표 (날짜 기준 merge)
        df['vix']          = price_df['^VIX']
        df['yield_curve']  = fred_df['T10Y2Y']
        df['hy_spread']    = fred_df['BAMLH0A0HYM2']
        df['cu_au_ratio']  = price_df['HG=F'] / price_df['GC=F']

        # C. GDELT 감성 (날짜 기준 merge, 종목 무관하게 동일값)
        df['fin_sent']     = gdelt_df['fin_sentiment']
        df['sent_mom']     = gdelt_df['sent_momentum']
        df['sent_z']       = gdelt_df['sent_zscore']
        df['news_spike']   = gdelt_df['news_spike']
        df['crisis_sig']   = gdelt_df['crisis_signal']
        df['omega_proxy']  = gdelt_df['omega_proxy']

        # 타겟: 30일 후 수익률 등급 (0~4)
        future_ret         = ret[ticker].shift(-30)
        df['target']       = pd.qcut(
            future_ret.dropna(), q=5, labels=[0,1,2,3,4]
        ).reindex(df.index)

        records.append(df)

    panel_df = pd.concat(records).dropna()
    return panel_df.reset_index().rename(columns={'index': 'date'})
```

## Step 3. 모델링 — Q, Ω 도출

### 3.1 핵심 아이디어

> 논문(LLM 방식)의 "100번 질문 → 평균=Q, 분산=Ω"를  
> ML 앙상블로 대체: "여러 모델 예측 → 평균=Q, 분산=Ω"

```
[배깅 계열] RandomForest, ExtraTrees
    → 트리별 예측의 평균 = Q, 표준편차 = Ω

[부스팅 계열] XGBoost
    → 클래스 예측 확률 = Q, 확률 분포의 분산 = Ω

[AI 에이전트] LLM 기반 (논문 방식 참고)
    → 여러 번 호출 후 평균 = Q, 표준편차 = Ω

[TabPFN] 소규모 테이블 데이터 특화 모델 (검토 중)
    → 예측 확률 = Q, 불확실성 = Ω

모델 간 예측 평균 = 최종 Q (전망값)
모델 간 예측 분산 = 최종 Ω (불확실성)
        ↓
레짐에 따라 Ω 동적 조정
        ↓
블랙-리터만 입력
```

### 3.2 레짐 판별 방식 (미확정 — 두 가지 방안 검토 중)

> **회의록 결정사항:**  
> 레짐 판별은 ML 예측 모델과 **별도로** 독립적인 모델로 수행  
> 전체 시계열에 대해 먼저 레짐을 판별한 뒤, 그 결과를 ML 피처로 투입  
> (롤링 윈도우 내에서 레짐을 다시 학습하는 방식은 의미 없다고 결론)

**방안 A — HMM (Hidden Markov Model)**
```
전체 시계열 입력 (VIX, 스프레드, GDELT 감성 등)
        ↓
HMM이 자동으로 숨겨진 상태(레짐) 학습
        ↓
각 날짜마다 레짐 레이블 (0, 1, 2, 3) 부여
        ↓
ML 모델의 피처 컬럼으로 투입
```

**방안 B — 규칙 기반 Classifier (현재 초안)**
```python
def classify_regime(gdelt_df, vix, spread):
    """
    레짐 분류 기준 (초안 — 추후 HMM으로 대체 가능):
    ┌─────────┬──────────────────┬──────────┬─────────────────────────┐
    │ 레짐    │ 감성 조건        │ VIX      │ Ω 조정 배수             │
    ├─────────┼──────────────────┼──────────┼─────────────────────────┤
    │ Crisis  │ 평균 - 2σ 이하   │  > 30    │ × 3.0 (AI 뷰 거의 무시) │
    │ Bear    │ 평균 - 1σ 이하   │    -     │ × 1.5 (보수적 반영)     │
    │ Bull    │ 평균 + 1σ 이상   │  < 18    │ × 0.8 (적극 반영)       │
    │ Neutral │ 그 외            │    -     │ × 1.0 (기본값)          │
    └─────────┴──────────────────┴──────────┴─────────────────────────┘
    """
    mean = gdelt_df['fin_sentiment'].mean()
    std  = gdelt_df['fin_sentiment'].std()

    conditions = [
        (gdelt_df['fin_sentiment'] < mean - 2*std) & (vix > 30),
        (gdelt_df['fin_sentiment'] < mean - std)   & (spread > 5),
        (gdelt_df['fin_sentiment'] > mean + std)   & (vix < 18),
    ]
    return np.select(conditions, ['Crisis','Bear','Bull'], default='Neutral')
```

### 3.3 롤링 윈도우 학습

```
타임라인:

Day 1          Day 150   Day 180
│←── 학습 ──→│←예측→│  → 1차 Q, Ω 도출

Day 31         Day 180   Day 210
   │←── 학습 ──→│←예측→│  → 2차 Q, Ω 도출

(30일마다 반복)
```

```python
def rolling_predict(panel_df, train_days=150, predict_days=30):
    """
    패널 데이터 기반 롤링 윈도우 예측
    
    입력 피처:
        가격 기반 8개 + 외부지표 4개 + GDELT 6개 = 총 18개
    
    타겟:
        30일 후 수익률 등급 (0~4, 5분위 분류)
    
    모델:
        RF (배깅): 트리별 예측 분산 → Ω
        XGBoost (부스팅): 클래스 확률 → Ω
        ExtraTrees (배깅 변형): 다양성 확보
    """
    from sklearn.ensemble import RandomForestClassifier, ExtraTreesClassifier
    from xgboost import XGBClassifier

    FEATURES = ['mom_20','mom_60','vol_20','vol_60','rel_spy',
                'ma_gap_20','vix','hy_spread','cu_au_ratio',
                'fin_sent','sent_mom','sent_z','news_spike',
                'crisis_sig','omega_proxy','regime_encoded']

    dates   = panel_df['date'].unique()
    results = []

    for i in range(0, len(dates) - train_days - predict_days, predict_days):
        train_dates = dates[i : i + train_days]
        pred_dates  = dates[i + train_days : i + train_days + predict_days]

        train = panel_df[panel_df['date'].isin(train_dates)]
        pred  = panel_df[panel_df['date'].isin(pred_dates)]

        X_train = train[FEATURES]
        y_train = train['target']
        X_pred  = pred[FEATURES]

        models = {
            'RF' : RandomForestClassifier(n_estimators=200, random_state=42),
            'XGB': XGBClassifier(n_estimators=200, random_state=42),
            'ET' : ExtraTreesClassifier(n_estimators=200, random_state=42),
        }

        proba_list = []
        for name, model in models.items():
            model.fit(X_train, y_train)
            proba_list.append(model.predict_proba(X_pred))  # (n, 5)

        # 3개 모델 예측 확률 스택
        proba_stack = np.stack(proba_list)  # (3, n, 5)

        # Q: 기대 등급 (확률 × 등급의 가중합) → 모델 간 평균
        grades         = np.array([0,1,2,3,4])
        expected_stack = proba_stack @ grades              # (3, n)
        Q_values       = expected_stack.mean(axis=0)       # (n,) 모델 간 평균
        raw_Omega      = expected_stack.var(axis=0)        # (n,) 모델 간 분산

        # Ω: 레짐에 따라 동적 조정
        regime_mult = {'Bull':0.8, 'Neutral':1.0, 'Bear':1.5, 'Crisis':3.0}
        regime_col  = pred['regime'].map(regime_mult).values
        Omega_values = raw_Omega * regime_col

        pred = pred.copy()
        pred['Q']     = Q_values
        pred['Omega'] = Omega_values
        results.append(pred)

    return pd.concat(results)
```

### 3.4 Q, Ω → 종목별 집계

```python
def aggregate_Q_Omega(result_df):
    """
    패널 결과 → 종목별 Q 벡터, Ω 대각행렬 생성
    블랙-리터만 모델 직접 입력 형태로 변환
    """
    latest = result_df.groupby('ticker').last()

    Q_vec   = latest['Q'].values           # (종목수,)
    Omega   = np.diag(latest['Omega'].values)  # (종목수, 종목수) 대각행렬

    return Q_vec, Omega
```

## Step 4. 블랙-리터만 모델

### 4.1 구성 요소

| 기호 | 의미 | 계산 방법 |
|---|---|---|
| π | 시장 균형 수익률 (Prior) | CAPM 역산: π = δΣw_mkt |
| Σ | 공분산 행렬 | yfinance 수익률 기반 |
| Q | 투자자 뷰 (전망값) | ML 모델 앙상블 평균 |
| Ω | 뷰 불확실성 | ML 모델 앙상블 분산 × 레짐 배수 |
| τ | 시장 vs 뷰 신뢰 비율 | 검증셋 Grid Search로 튜닝 |
| P | 피킹 행렬 | 단위행렬 I (절대적 뷰 사용) |
| μ | 사후 기대수익률 (Posterior) | 블랙-리터만 수식 결과 |

### 4.2 수식 구현

```python
def black_litterman(pi, Sigma, P, Q, Omega, tau=0.05):
    """
    블랙-리터만 공식:
    μ = [(τΣ)⁻¹ + PᵀΩ⁻¹P]⁻¹ × [(τΣ)⁻¹π + PᵀΩ⁻¹Q]

    핵심 메커니즘:
    - Ω 낮음 (확신도 높음) → μ가 Q에 가깝게 이동 (AI 뷰 강하게 반영)
    - Ω 높음 (확신도 낮음) → μ가 π에 가깝게 유지 (시장 평균 유지)
    - Crisis 레짐 → Ω × 3.0 → μ ≈ π (시장 평균 비중으로 회귀)
    """
    M1  = np.linalg.inv(tau * Sigma)
    M2  = P.T @ np.linalg.inv(Omega) @ P
    mu  = np.linalg.inv(M1 + M2) @ (M1 @ pi + P.T @ np.linalg.inv(Omega) @ Q)
    return mu


def compute_equilibrium_returns(Sigma, w_mkt, delta=2.5):
    """
    CAPM 역산으로 시장 균형 수익률(π) 계산
    delta: 시장 위험회피계수 (보통 2~3)
    w_mkt: 시장 시가총액 비중 (ETF 비중 등으로 대체 가능)
    """
    return delta * Sigma @ w_mkt
```

### 4.3 τ 튜닝 (검증셋 Grid Search)

```python
def tune_tau(panel_val, Sigma, w_mkt, tau_grid=None):
    """
    검증셋에서 Sharpe Ratio 최대화하는 τ 탐색
    논문 방식 그대로 적용
    """
    if tau_grid is None:
        tau_grid = [0.01, 0.025, 0.05, 0.1, 0.25, 0.5]

    best_tau, best_sharpe = 0.05, -np.inf

    for tau in tau_grid:
        # 검증셋으로 임시 포트폴리오 구성 후 Sharpe 계산
        sharpe = evaluate_sharpe(panel_val, Sigma, w_mkt, tau)
        if sharpe > best_sharpe:
            best_sharpe = sharpe
            best_tau    = tau

    print(f"최적 τ: {best_tau} (Sharpe: {best_sharpe:.3f})")
    return best_tau
```

## Step 5. MVO — 성향별 최적 비중 산출

### 5.1 위험회피계수(γ)별 성향 구분

| 성향 | γ 범위 | 특징 | 주요 자산 비중 |
|---|---|---|---|
| 초공격형 | γ = 1 | 고수익 추구, 고위험 감내 | 주식 ETF 80%+ |
| 공격형 | γ = 2 | 수익 우선, 리스크 일부 감내 | 주식 60~80% |
| 중립형 | γ = 4 | 수익/리스크 균형 | 주식 40~60% |
| 보수형 | γ = 8 | 안정성 우선 | 채권/대안 50%+ |
| 초보수형 | γ = 16 | 원금 보전 최우선 | 채권 70%+ |

### 5.2 MVO 구현

```python
from scipy.optimize import minimize

def mvo_optimize(mu, Sigma, gamma, constraints=None):
    """
    평균-분산 최적화:
    목적함수: max(μᵀw - γ/2 × wᵀΣw)
    
    제약조건:
    - 비중 합계 = 1
    - 개별 비중 0 ≤ w ≤ 0.3 (단일 자산 집중 방지)
    - 섹터 비중 ≤ 0.4 (섹터 집중 방지)
    
    gamma: 위험회피계수 (클수록 변동성 회피)
    """
    n = len(mu)

    def neg_utility(w):
        return -(w @ mu - gamma/2 * w @ Sigma @ w)

    constraints = [{'type': 'eq', 'fun': lambda w: w.sum() - 1}]
    bounds      = [(0, 0.30)] * n  # 개별 자산 최대 30%

    result = minimize(
        neg_utility,
        x0=np.ones(n)/n,
        method='SLSQP',
        bounds=bounds,
        constraints=constraints
    )
    return result.x


def build_fund_products(mu, Sigma):
    """
    γ별 5개 펀드 상품 비중 동시 산출
    """
    gammas = {'초공격형':1, '공격형':2, '중립형':4, '보수형':8, '초보수형':16}
    funds  = {}

    for name, gamma in gammas.items():
        weights      = mvo_optimize(mu, Sigma, gamma)
        funds[name]  = weights
        print(f"{name} (γ={gamma}): 상위 3 자산 = {get_top3(weights)}")

    return funds
```

def stress_test(portfolio_returns, gdelt_df, threshold_z=-2):
    """
    GDELT sent_zscore < -2 구간을 스트레스 구간으로 자동 식별
