# Gu, Kelly, Xiu (2020) — 횡단면 랭크 활용 방식 해설

> 논문: *Empirical Asset Pricing via Machine Learning*  
> 저널: The Review of Financial Studies (2020), Impact Factor 5.4  
> 핵심 아이디어: 머신러닝으로 주식 수익률을 예측하고, 그 예측값으로 포트폴리오를 구성

---

## 1. 이 논문이 뭘 하려는 건가요?

**한 줄 요약**: "주식이 다음 달에 얼마나 오를지 머신러닝으로 예측할 수 있을까?"

주식 시장에는 약 30,000개의 주식이 있고, 각 주식마다 크기(시가총액), 모멘텀(최근 수익률), 변동성 등 **94개의 특성(피처)**이 있습니다.  
이 논문은 이 94개 특성을 머신러닝 모델에 넣어서 **다음 달 수익률**을 예측합니다.

---

## 2. 횡단면 랭크란 무엇인가요?

### 비유로 이해하기

> 반에서 수학 점수를 비교한다고 해봅시다.
> - A반 평균이 70점, B반 평균이 90점이라면
> - A반에서 75점이면 "평균 위"지만, B반에서 75점이면 "평균 아래"
> - **절대 점수보다 "반에서 몇 등인가"가 더 공정한 비교**

주식도 마찬가지입니다.  
- 2008년 금융위기 때 모멘텀 값이 -30%라면 → 그냥 보면 나쁜 것 같지만 다른 주식들도 다 -30%라면 평균 수준
- 2021년 강세장 때 모멘텀 값이 +5%라면 → 작아 보이지만 다른 주식들보다 높다면 상위권

→ **"이 주식이 이 시점에 다른 주식들 중 몇 등인가"** 로 바꾸는 것이 횡단면 랭크

---

## 3. 논문에서 어떻게 구현했나요?

### 3-1. 피처(X)에 횡단면 랭크 적용

논문 각주 29번:
> *"We cross-sectionally rank all stock characteristics period-by-period and map these ranks into the [-1, 1] interval"*

**과정**:

```
[1단계] 2020년 1월 기준, 6,000개 주식의 모멘텀 값 수집
  AAPL: +7%
  MSFT: +5%
  NVDA: +12%
  JPM:  -2%
  ...  (6,000개)

[2단계] 크기 순으로 줄세우기 (순위 계산)
  1등: NVDA (+12%)
  2등: AAPL (+7%)
  3등: MSFT (+5%)
  ...
  6000등: JPM (-2%)

[3단계] 순위를 -1 ~ +1 사이 숫자로 변환
  1등(상위) → +1.0
  중간      →  0.0
  6000등    → -1.0
```

이렇게 하면 **2008년이든 2021년이든 항상 -1~+1 사이** 값이 나옵니다.

### 3-2. 왜 -1 ~ +1로 변환하나요?

| 문제 | 해결 |
|------|------|
| 모멘텀은 -50%~+200%, 시가총액은 1조~1000조 → 단위가 너무 다름 | 모두 -1~+1로 통일 |
| 2008년 모멘텀 -30% vs 2021년 모멘텀 +20% → 같은 값이지만 의미가 다름 | 각 시점에서의 상대 위치로 변환 |
| 극단값(outlier)이 모델을 망침 | 랭크 변환은 극단값에 덜 민감 |

### 3-3. 수식으로 표현하면

$$z_{i,t} = x_t \otimes c_{i,t}$$

- $c_{i,t}$: 주식 i의 t시점 특성값 → **횡단면 랭크 변환 적용**
- $x_t$: 거시경제 지표 (모든 주식에 공통)
- $z_{i,t}$: 최종 피처 (두 가지의 조합, 총 920개)

## 4. 예측값(Y)으로 무엇을 썼나요?

**예측 대상(Y)은 실제 수익률** (퍼센트)을 그대로 사용했습니다.

```
예측값(Ŷ): 모델이 예측한 다음 달 수익률
실제값(Y): 실제로 관찰된 다음 달 수익률

목표: Ŷ가 Y와 최대한 가깝도록 모델 학습
```

> 피처(X)는 랭크 변환 → 예측 대상(Y)은 실제 수익률 그대로  
> (Y를 랭크로 변환하지 않음)

---

## 5. 학습/검증/테스트 분리 방법

논문은 60년(1957~2016) 데이터를 아래처럼 나눴습니다:

```
[학습(Train)]    1957~1974 (18년) → 모델 파라미터 학습
[검증(Val)]      1975~1986 (12년) → 하이퍼파라미터 튜닝
[테스트(Test)]   1987~2016 (30년) → 최종 성능 평가 (OOS)
```

매년 학습 데이터를 1년씩 늘리면서 재학습 (Walk-Forward 방식)

---

## 6. 예측 결과를 어떻게 포트폴리오로 만들었나요?

여기서 **예측값 기준 횡단면 랭크**가 다시 등장합니다.

```
[매달 말]
1. 모든 주식에 대해 다음 달 수익률 예측값 계산
2. 예측값 기준으로 주식을 10개 그룹(Decile)으로 분류

   Decile 10 (예측 수익률 상위 10%) → 매수 (Long)
   Decile 1  (예측 수익률 하위 10%) → 매도 (Short)

3. Long - Short = 롱숏 포트폴리오 수익률 계산
```

### 실제 결과 (Table 7, NN3 기준)

| 그룹 | 예측 수익률 | 실제 수익률 | Sharpe Ratio |
|------|-----------|-----------|-------------|
| Decile 1 (최하위) | -0.03% | -0.43% | -0.19 |
| Decile 5 (중간) | +0.71% | +0.69% | +0.55 |
| Decile 10 (최상위) | +1.83% | +1.69% | +0.80 |
| **10-1 스프레드** | **+1.86%** | **+2.12%** | **+1.20** |

→ **예측값 상위 종목이 실제로도 더 많이 올랐습니다!**  
→ 연간 Sharpe Ratio 1.35 달성 (일반 투자자 0.51 대비 2.6배)

## 7. 가장 중요한 피처는 무엇이었나요?

모든 모델이 공통적으로 중요하게 선택한 피처 Top 5:

| 순위 | 피처명 | 의미 |
|------|--------|------|
| 1 | mom1m | 단기 반전 (1개월 전 수익률) |
| 2 | mvel1 | 시가총액 (기업 크기) |
| 3 | mom12m | 모멘텀 (12개월 수익률) |
| 4 | chmom | 모멘텀 변화율 |
| 5 | maxret | 최근 최대 수익률 |

→ **가격 추세 관련 피처(모멘텀, 반전)가 가장 강력한 신호**

## 8. 우리 프로젝트에 주는 시사점

| 논문 방법 | 우리 프로젝트 적용 방안 |
|----------|----------------------|
| 피처를 횡단면 랭크 → [-1,1] 변환 | `groupby('date')[feature].rank(pct=True) * 2 - 1` |
| 30,000개 종목, 60년 데이터 | GICS 11섹터 × 3~5종목, 10년 데이터 |
| Y = 실제 수익률 (회귀) | Y = 5분위 클래스 (분류) |
| 예측값으로 Decile 정렬 → 포트폴리오 | ML 예측값 Q → BL → MVO |
| 모든 자산 단일 모델 (패널) | Long-Panel 동일 방식 |

### 핵심 인사이트

> 논문이 ETF 아닌 **개별 주식**을 쓴 이유:  
> 개별 주식끼리 횡단면 비교(이달에 어떤 주식이 상대적으로 오를지)가 가능하기 때문.  
> ETF는 자산 성격이 너무 달라 같은 기준으로 랭크하기 어렵습니다.
