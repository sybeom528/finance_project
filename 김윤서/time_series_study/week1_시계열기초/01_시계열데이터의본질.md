## 1. 시계열 데이터란 무엇인가

**시계열(Time Series)** 은 시간 순서대로 수집된 관측치의 집합입니다.

수학적으로는 이렇게 표기합니다.

$$ \{X_t\}_{t=1}^{T} = \{X_1, X_2, X_3, \ldots, X_T\} $$

여기서 $t$ 는 시간 인덱스, $X_t$ 는 그 시점의 관측값입니다.

### 일반 데이터(Tabular)와 무엇이 다른가

| 구분 | 예시 | 행 사이의 관계 |
|---|---|---|
| **Tabular 데이터** | 고객별 나이·소득·구매여부 | 행끼리 독립 — 고객 A와 고객 B는 무관 |
| **시계열 데이터** | 일별 MSFT 종가 | 행끼리 종속 — 어제 가격이 오늘 가격에 영향 |

이 차이가 모든 것을 결정합니다. 머신러닝 모델은 대부분 "행이 독립이다"를 가정하고 만들어졌기 때문에, 시계열에 그대로 쓰면 미묘한 오류가 발생합니다.

## 2. i.i.d. 가정 — 왜 중요한가

### i.i.d.의 정의

**i.i.d.** = **i**ndependent and **i**dentically **d**istributed (독립이며 동일한 분포)

두 가지 조건을 동시에 만족해야 합니다.

1. **독립 (Independent)**: 각 관측치가 서로 영향을 주지 않음
   - 예: 동전을 100번 던질 때, 1번째 결과가 2번째 결과에 영향 없음
2. **동일 분포 (Identically Distributed)**: 모든 관측치가 같은 확률 분포에서 나옴
   - 예: 항상 같은 동전(앞면 나올 확률 0.5)을 사용

### 거의 모든 ML 모델의 숨겨진 전제

XGBoost, Random Forest, Logistic Regression, MLP 등 표 데이터 기반 모델은 **모두 i.i.d.를 가정**합니다.
그래서 학습할 때 "행 순서를 섞어도(shuffle) 결과가 같아야 한다"고 보는 겁니다.

```python
# XGBoost는 이런 식으로 학습
# 행 순서를 무작위로 섞어도 동일한 모델이 나옴
X_shuffled, y_shuffled = shuffle(X, y, random_state=42)
model.fit(X_shuffled, y_shuffled)  # ← OK
```

### 주가가 i.i.d.를 위반하는 이유

**(1) 독립성 위반 — 자기상관(Autocorrelation)이 존재**

오늘 MSFT가 $400이면 내일은 $401 근처일 확률이 높지, 갑자기 $50으로 떨어질 확률은 매우 낮습니다.
→ "어제 가격"이 "오늘 가격"의 강력한 예측 변수입니다.
→ 두 관측치는 명백히 **독립이 아닙니다**.

**(2) 동일 분포 위반 — 분포가 시간에 따라 변함**

2010년 MSFT 가격 분포(평균 약 $25, 변동성 작음)와 2024년 분포(평균 약 $400, 변동성 큼)는 완전히 다릅니다.
→ 같은 분포에서 추출된 표본이 아닙니다.
→ **identically distributed가 아닙니다.**

### 위반 시 무슨 일이 벌어지는가

XGBoost로 주가 예측 모델을 만들고 train/test를 무작위로 섞어 평가하면, **비현실적으로 높은 정확도가 나옵니다.**
이유는 train set에 2024-01-15 데이터가 있고 test set에 2024-01-16 데이터가 있을 때, 두 행이 거의 같은 가격대이기 때문입니다.
→ 모델이 "예측"하는 게 아니라 "이웃 데이터를 외운" 것에 가깝습니다.
→ 실전에서는 미래 데이터가 없으니 이 트릭이 안 통합니다 → **실전 성능 폭락**

이것이 시계열 분석에서 **반드시 시간 순서대로 train/test를 분할**해야 하는 이유입니다(Walk-Forward).

## 3. 가격(Price) 대신 수익률(Return)을 쓰는 이유

현재 프로젝트 코드를 보면 `log_return`, `fwd_ret` 같은 컬럼이 등장합니다.
왜 가격을 그대로 안 쓰고 굳이 수익률로 변환할까요?

### 이유 1 — 정상성(Stationarity) 확보

가격은 추세를 가지고 끝없이 변합니다 (비정상). 반면 수익률은 0 근처에서 진동합니다 (정상).

```
가격 시계열                       수익률 시계열
$    │      ╱╲                  %  │
400  │    ╱╲╱                  +5  │  ╲ ╱╲    ╱╲
300  │  ╱╲╱                     0  │───╳───╳───╳───
200  │ ╱                       -5  │     ╲    ╱
100  │╱                            │
     └──────────  시간              └──────────  시간
     평균이 시간에 따라 증가         평균이 0 근처에서 일정
     (비정상)                       (정상)
```

대부분의 시계열 분석 도구(ARIMA, 자기상관 분석 등)는 **정상성을 전제로** 만들어졌습니다. 비정상 시계열에 그대로 적용하면 결과가 왜곡됩니다.

### 이유 2 — 스케일 통일

MSFT 가격은 $400, AAPL은 $200, 페니스톡은 $5. 같은 모델에 넣으려면 스케일을 맞춰야 합니다.
수익률은 모두 % 단위라 종목 간 직접 비교가 가능합니다.

### 이유 3 — 비즈니스 의미가 더 명확

투자자가 진짜로 관심 있는 건 "내일 가격이 얼마냐"가 아니라 **"오늘 대비 얼마나 오르내리느냐(수익률)"** 입니다.
B-L 모델의 입력도 결국 *기대 수익률* 이지 *기대 가격* 이 아닙니다.

## 4. 단순 수익률 vs 로그 수익률 — 왜 log를 씌우는가

### 두 가지 정의

**단순 수익률 (Simple Return)**
$$ r_t = \frac{P_t - P_{t-1}}{P_{t-1}} = \frac{P_t}{P_{t-1}} - 1 $$

**로그 수익률 (Log Return)**
$$ r_t^{\log} = \ln\left(\frac{P_t}{P_{t-1}}\right) = \ln(P_t) - \ln(P_{t-1}) $$

값이 작을 때(±5% 이내) 두 값은 거의 같습니다 ($\ln(1+x) \approx x$).
하지만 시계열 분석에서는 **로그 수익률**을 선호합니다. 두 가지 결정적 이유가 있습니다.

### 이유 (1) — 시간 누적이 덧셈으로 단순해진다

여러 기간의 수익률을 합칠 때:

**단순 수익률은 곱셈**으로 누적됩니다 (수식이 복잡):
$$ R_{\text{전체}} = (1 + r_1)(1 + r_2)\cdots(1 + r_n) - 1 $$

**로그 수익률은 덧셈**으로 누적됩니다 (계산이 깔끔):
$$ R_{\text{전체}}^{\log} = r_1^{\log} + r_2^{\log} + \cdots + r_n^{\log} $$

→ 시계열 모델은 덧셈 연산이 곱셈보다 압도적으로 다루기 쉽습니다.
→ 평균, 분산, 정규성 가정 등 통계적 성질이 훨씬 깔끔하게 정의됩니다.

### 이유 (2) — 분포가 정규분포에 더 가까워진다

금융 시계열은 **로그 정규 분포(log-normal distribution)** 를 따른다고 알려져 있습니다.
가격에 로그를 씌우면 → 정규분포에 가까워집니다 → 통계 모델이 잘 동작합니다.

```python
# 직접 확인하는 코드 (다음 실습 노트북에서 돌려볼 예정)
import numpy as np
import yfinance as yf

msft = yf.download('MSFT', start='2020-01-01', end='2024-12-31')
prices = msft['Close']

simple_return = prices.pct_change().dropna()
log_return = np.log(prices / prices.shift(1)).dropna()

# 정규성 검정 (Jarque-Bera) — p-value가 클수록 정규분포에 가까움
from scipy.stats import jarque_bera
print('단순 수익률 JB p-value:', jarque_bera(simple_return)[1])
print('로그 수익률 JB p-value:', jarque_bera(log_return)[1])
# 보통 로그 쪽 p-value가 더 큼
```
