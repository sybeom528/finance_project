# 프로젝트 예측 구조 시각화

> GRU/LSTM 시퀀스 모델로 Q, Ω를 산출해 Black-Litterman에 넘기는 전체 흐름

---

## 1. 전체 Walk-Forward 타임라인

```
날짜 축 ──────────────────────────────────────────────────────────────▶

         Fold 1
         ├──────────── IS (231일) ────────────┤purge│emb│── OOS (21일) ──┤
         │                                    │(21) │(21)│               │

                Fold 2 (21일 오른쪽 이동)
                ├──────────── IS (231일) ────────────┤purge│emb│── OOS ──┤

                       Fold 3
                       ├──────────── IS (231일) ─────────────┤purge│emb│OOS┤

                              ...   (약 79개 fold, 10년치)
```

- **IS(학습 구간)**: 252일 설정, 뒤 21일 purge → 실제 학습 231일
- **purge**: IS 끝 21일 제거 — fwd_ret_21d 레이블이 OOS 가격을 참조하기 때문
- **embargo**: 21일 공백 — lookback이 OOS를 침범하지 않도록
- **OOS(예측 구간)**: 21일, 이 기간의 예측값이 BL에 들어감
- **step**: 21일씩 우측 이동 (매달 리밸런싱)

### IS / OOS 란 무엇인가요?

> **IS = In-Sample** (인샘플) = 모델이 **학습에 사용한** 데이터  
> **OOS = Out-of-Sample** (아웃오브샘플) = 모델이 **학습 중에 본 적 없는** 미래 데이터

**왜 나눠야 할까요?**

시험 공부에 비유해봐요.

- **IS** = 교과서와 문제집 (공부하는 자료)
- **OOS** = 실제 시험지 (공부한 적 없는 새 문제)

만약 시험 문제를 미리 보고 공부했다면 점수가 좋겠지만, 그건 "진짜 실력"이 아니에요.  
AI 모델도 마찬가지입니다. 학습에 쓴 데이터로 평가하면 당연히 잘 나오지만, **실제 미래에서도 잘 할지는 모릅니다.**

그래서 OOS 데이터(모델이 한 번도 본 적 없는 미래 구간)로 평가해야 진짜 성능을 알 수 있어요.

**그런데 왜 한 번만 나누지 않고 "walk-forward"로 계속 이동할까요?**

일반 AI 수업에선 "전체 데이터의 80%로 학습, 20%로 테스트"를 한 번만 해요.  
금융에서 이렇게 하면 심각한 문제가 생깁니다:

```
잘못된 방법 (일반 AI):

2016년──────────────────────────────────2025년
├────────────── 학습 (80%) ──────────────┤테스트┤

문제: 2020년 코로나 폭락이 학습 데이터에 포함됨
     → 테스트 때 "코로나는 이렇게 회복했었지" 를 이미 알고 있는 상태로 예측
     → 이건 미래를 미리 알고 맞추는 것 = 실전에서 쓸 수 없음
```

```
올바른 방법 (Walk-Forward):

시간 →
[IS 학습] → [OOS 예측]          ← fold 1: 2016년으로 학습, 2017년 1월 예측
      [IS 학습] → [OOS 예측]    ← fold 2: 21일 앞으로 이동
           [IS 학습] → [OOS 예측] ← fold 3: 또 21일 앞으로 이동
                ...
```

각 fold에서 IS로 GRU를 학습시키고, OOS에서 예측합니다.  
OOS 예측값은 그 당시 시점에서 "진짜 미래를 모른 채 뽑아낸 예측"이기 때문에 신뢰할 수 있어요.  
이 OOS 예측값들이 쌓여서 **Q (기대수익률 뷰)** 가 되고 Black-Litterman에 들어갑니다.

## 2. 하나의 Fold 확대

```
일(day) : 1        60       231   252   273       294
           │         │        │     │     │         │
IS 원본    ├─────────────────────────┤
IS 실사용  ├─────────────────┤
           (1일 ~ 231일)     │purge│
                             └─────┘
                                   │← 21일 →│← 21일 →│
                                   embargo    OOS 구간
                                             (예측 대상)
```

**각 구간의 역할:**

| 구간 | 날짜 범위 | 역할 |
|------|-----------|------|
| IS 실사용 | day 1 ~ 231 | GRU 학습 |
| purge | day 232 ~ 252 | 버림 (레이블 누수 방지) |
| embargo | day 253 ~ 273 | 비워둠 (lookback 누수 방지) |
| OOS | day 274 ~ 294 | GRU로 예측 → BL 입력 |

### purge(퍼지)가 필요한 이유

우리 모델이 예측하려는 값은 **fwd_ret_21d** = "오늘 기준 21거래일 뒤 수익률"입니다.

예를 들어 day 231의 레이블을 계산하려면 day 252(21일 뒤)의 주가가 필요해요.  
그런데 day 252는 OOS 직전 구간입니다. 즉:

```
IS 마지막 샘플 (day 231)의 레이블 y = fwd_ret_21d
  → 계산에 day 252 주가가 사용됨
  → day 252는 이미 OOS 근처 데이터 ← 미래 정보가 학습에 스며든다!
```

이걸 **데이터 누수(data leakage)**라고 합니다. 시험 문제를 미리 보고 공부한 것과 같아요.  
그래서 IS 끝 21일(day 232~252)은 학습에서 **아예 제거(purge)**합니다.

### embargo(엠바고)가 필요한 이유

purge를 했다고 끝이 아닙니다. OOS 첫 날(day 274)에서 예측할 때,  
GRU는 직전 60일치 데이터를 봅니다: **day 215 ~ day 274**.

그런데 day 253~273 구간(embargo)도 이 lookback 범위에 들어옵니다.  
이 구간은 IS 학습 데이터 근처라서, 학습 때 레이블로 사용된 미래 주가가 간접적으로 영향을 줄 수 있어요.

```
OOS day 274 예측 시 lookback:
  day 215 ~ 274  (60일)
       ↑
  day 253~273 구간이 포함됨
  → 이 구간은 IS 레이블 계산에 쓰인 가격과 겹칠 수 있음

해결: day 253~273을 "비어있는 공백 구간(embargo)"으로 만들어서
     이 구간의 데이터가 학습에 영향 미치지 않도록 보장
```

정리하면:
- **purge** = IS 레이블이 OOS 가격을 직접 참조하는 것 차단
- **embargo** = IS 학습 패턴이 OOS 입력에 간접적으로 스며드는 것 차단

## 3. IS 안에서 Sequential 샘플 만들기 (Sliding Window, T=60)

```
IS 데이터 (231일 × 10종목):

day:   1   2   3  ...  60  61  62  ... 230  231
       │   │   │        │   │   │        │    │

샘플 0: ├──────── T=60 ────────┤
        day 1 ~ day 60
        X shape: (60, 17)  →  y = fwd_ret_21d[day 60]
                                   (day 60 기준 21일 뒤 수익률)

샘플 1:     ├──────── T=60 ────────┤
            day 2 ~ day 61
            X shape: (60, 17)  →  y = fwd_ret_21d[day 61]

샘플 2:         ├──────── T=60 ────────┤
                day 3 ~ day 62
                X shape: (60, 17)  →  y = fwd_ret_21d[day 62]

...

샘플 171:                      ├──────── T=60 ────────┤
                                day 172 ~ day 231
                                X shape: (60, 17)  →  y = fwd_ret_21d[day 231]
```

**IS 내 총 샘플 수**: 231 - 60 + 1 = **172개**
(종목 k개 패널이면 172 × k개)

### T=60은 어디서 나온 숫자인가요?

**T**는 **lookback window** — "GRU가 한 번에 몇 일치를 보고 예측할까"를 결정하는 숫자입니다.  
60 거래일 ≈ **약 3개월**입니다. 이 숫자는 연구자가 직접 정하는 **설계값(hyperparameter)**이에요.

**왜 60인가?**

| T 값 | 결과 | 문제점 |
|------|------|--------|
| T = 5 (1주일) | 너무 짧음 | 노이즈(일시적 등락)만 보이고, 중기 패턴을 놓침 |
| T = 60 (3개월) | 적당함 | 모멘텀·추세를 충분히 볼 수 있고, 샘플 수도 172개로 충분 |
| T = 120 (6개월) | 너무 길어짐 | IS 231일 중 샘플 수가 231−120+1=112개로 줄어들어 학습 부족 |

금융 연구에서 **3~6개월 lookback**이 모멘텀 신호를 잡는 데 가장 효과적이라는 것이 여러 논문(GKX 2020, Moskowitz 등)으로 검증되어 있어요.

**샘플 수 공식:**

```
IS 실사용 일수 - T + 1 = 샘플 수
     231     - 60 + 1 = 172개
```

T를 늘리면 각 샘플이 더 많은 과거를 보지만 샘플 수가 줄고,  
T를 줄이면 샘플 수는 늘지만 각 샘플이 보는 과거가 짧아집니다.  
T=60은 이 트레이드오프에서 균형점입니다. 나중에 T=40, T=90으로 바꿔서 비교해볼 수도 있어요.

## 4. 하나의 샘플 내부 구조

```
샘플 하나 = (X, y)

X: shape (60, 1)   ← 피처: log_ret_1d 하나만
┌──── 1개 피처 ────┐
│  log_ret_1d      │
├──────────────────┤ ← day t-59
│  -0.012          │
├──────────────────┤ ← day t-58
│   0.008          │
├──────────────────┤ ← day t-57
│   0.003          │
│    ...           │
│    ...           │
├──────────────────┤ ← day t-1
│  -0.021          │
├──────────────────┤ ← day t  (오늘)
│   0.011          │
└──────────────────┘
  └── T = 60일 ───┘

y: scalar
  fwd_ret_21d[t] = (price[t+21] - price[t]) / price[t]
  = "오늘(day t) 기준, 21일 뒤의 초과 수익률"
```

### 피처: 수익률(`log_ret_1d`) 하나만 사용하는 이유

현재 모델은 **일별 log 수익률 하나(F=1)**만 피처로 씁니다.

`log_ret_1d` = 오늘 주가가 어제보다 몇 % 변했는지를 log 변환한 값

```
log_ret_1d = ln(오늘 종가 / 어제 종가)

예시:
  어제 종가 100원 → 오늘 101원 : log_ret_1d ≈ +0.010  (1% 상승)
  어제 종가 100원 → 오늘  98원 : log_ret_1d ≈ -0.020  (2% 하락)
```

**왜 log 변환을 쓰나요?**  
단순 수익률(`(오늘-어제)/어제`)은 +50%, -50% 같은 극단값에서 비대칭이 생겨요.  
log 변환하면 상승과 하락이 대칭적으로 표현돼서 GRU 학습이 안정적입니다.

**왜 지금은 이 피처 하나만 쓰나요?**  
- 가장 단순한 구조로 먼저 GRU의 기본 성능을 확인하는 것이 목적
- 피처가 적을수록 과적합 위험이 줄고, 결과 해석이 쉬움
- 성능이 확인되면 이후에 `mom_1m`, `vix`, `t10y2y` 등을 추가해 개선 가능

> 피처 1개 × 60일 = **60개 숫자**가 GRU에 입력됩니다.  
> "이 종목의 최근 60거래일치 일별 수익률 흐름"만으로 21일 뒤를 예측하는 구조입니다.

### 나중에 추가할 수 있는 피처들 (참고)

| 그룹 | 피처 예시 | 의미 |
|------|-----------|------|
| 모멘텀 | `mom_1m`, `mom_3m`, `mom_6m` | 최근 1~6개월 누적 수익률 |
| 변동성 | `vol_20d`, `vol_60d` | 최근 20/60일 수익률 표준편차 |
| 기술적 | `rsi_14`, `bb_pct`, `ma_gap_20_60` | 과매수/과매도, 이동평균 괴리 |
| 매크로 | `vix`, `t10y2y`, `indmom` | 공포지수, 금리차, 섹터 모멘텀 |

## 5. OOS 구간에서의 예측

```
OOS 구간 (21일):

day:  274  275  276  ...  294
       │    │    │         │

day 274 예측:
  X_input = [day 215 ~ day 274]  ← 바로 직전 60일 (IS+embargo 영역에서 가져옴)
  GRU → ŷ = fwd_ret_21d 예측값
            = "day 274 기준 21일 뒤 수익률 예측"

day 275 예측:
  X_input = [day 216 ~ day 275]
  GRU → ŷ = fwd_ret_21d 예측값

...

day 294 예측:
  X_input = [day 235 ~ day 294]
  GRU → ŷ = fwd_ret_21d 예측값
```

**OOS 21일 전체의 예측값 → 평균 or 마지막 값 → Q 벡터 (종목별 기대수익률)**

### "직전 60일치를 본다"는 게 정확히 무슨 의미인가요?

**왜 하루치만 보면 안 될까요?**

day 274 하루의 데이터만 보면 이런 정보밖에 없어요:

```
day 274 하루치 (17개 숫자):
  log_ret_1d  = +0.011  (오늘 1.1% 올랐다)
  vix         = 19.1    (오늘 VIX는 19)
  mom_3m      = 0.091   (3개월 수익률)
  ...
```

이걸로 "앞으로 21일 뒤 수익률"을 예측하기엔 맥락이 너무 없어요.  
"오늘 1% 올랐다"는 것만으로는 이게 3개월 상승세의 끝인지, 폭락 후 반등인지 알 수 없습니다.

**60일치를 본다 = 60행짜리 표(1,020개 숫자)를 입력으로 넣는다**

```
입력 X: shape (60행 × 17열)

         log_ret  mom_1m  mom_3m  vol_20d  vix   t10y2y  ...
day 215  [ 0.003,  0.041,  0.078,  0.181,  18.2,  -0.4,  ... ]  ← 60일 전
day 216  [ 0.008,  0.045,  0.079,  0.183,  17.9,  -0.4,  ... ]
day 217  [-0.012,  0.043,  0.082,  0.187,  18.2,  -0.4,  ... ]
...      (중략)
day 273  [-0.021,  0.058,  0.088,  0.198,  19.0,  -0.3,  ... ]
day 274  [ 0.011,  0.061,  0.091,  0.201,  19.1,  -0.5,  ... ]  ← 오늘
```

GRU는 이 표를 **위에서 아래로 한 줄씩** 읽어내려갑니다.  
한 줄 읽을 때마다 "지금까지 읽은 내용의 요약(hidden state h)"을 업데이트해요:

```
day 215 읽음 → h₁ = "일단 이날은 이랬구나"
day 216 읽음 → h₂ = h₁ + "어제보다 조금 올랐네"
day 217 읽음 → h₃ = h₂ + "이틀 연속 상승이네"
   ...
day 274 읽음 → h₆₀ = "60일치 흐름을 전부 압축한 최종 요약"
                  ↓
           FC layer → ŷ (21일 뒤 수익률 예측)
```

60일을 다 읽고 나온 **마지막 요약 h₆₀ 하나**만 예측에 씁니다.  
이 요약 안에 "3개월 동안 이 종목이 어떤 흐름이었는지"가 압축되어 있어요.

**60일 흐름이 있어야 알 수 있는 것들:**

| 패턴 | 60일 흐름 예시 | GRU가 파악하는 의미 |
|------|----------------|---------------------|
| 모멘텀 | mom_1m, mom_3m 모두 꾸준히 플러스 | 상승 추세 지속 가능성 |
| 과열 후 조정 | 3개월 급등 → rsi_14 80 이상 → 최근 하락 | 과열 해소 국면 |
| 변동성 확대 | vol_20d가 최근 30일 사이 2배로 증가 | 불확실성 고조, 예측 어려움 |
| 매크로 악화 | vix가 15→25로 상승, t10y2y 역전 | 시장 전반 위험 증가 |

> **한 줄 요약**: GRU가 "60일치를 본다" = day 215~274의 17개 피처 × 60행 표를 입력으로 받아,  
> 하루씩 순서대로 읽으며 흐름을 파악하고, 마지막에 "이 60일의 요약"으로 21일 뒤를 예측한다.

## 7. GRU 내부 동작 (T=60 시간축을 따라)

```
입력 시퀀스 X: (60, 1)   ← log_ret_1d 하나

day t-59:  x_{t-59} ──▶ [GRU cell] ──▶ h_{t-59}
                              ↑
day t-58:  x_{t-58} ──▶ [GRU cell] ──▶ h_{t-58}
                              ↑
day t-57:  x_{t-57} ──▶ [GRU cell] ──▶ h_{t-57}
              ...
              ...
day t-1:   x_{t-1}  ──▶ [GRU cell] ──▶ h_{t-1}
                              ↑
day t:     x_t      ──▶ [GRU cell] ──▶ h_t  ← 마지막 hidden state
                                           │
                                           ▼
                                       FC layer
                                           │
                                           ▼
                                      ŷ (scalar)
                                 = fwd_ret_21d 예측값
```

GRU cell은 "지금 입력 + 직전 hidden state"를 보고 hidden state를 갱신합니다.
60번 반복 후 나온 **마지막 h_t**가 "60일치 시계열 요약"이고, 이걸로 예측합니다.

## 8. BL 연결: GRU 출력 → Q, Ω

```
Fold k의 OOS 21일 예측 완료:

종목     day274  day275  ...  day294   ← OOS 21일
─────────────────────────────────────
MSFT     0.023   0.019  ...   0.031
INTC    -0.011  -0.008  ...  -0.014
AAPL     0.041   0.038  ...   0.044
...

↓ 종목별 평균 (or 마지막 날)

Q 벡터 (종목별 기대수익률 뷰):
  Q = [MSFT: 0.024, INTC: -0.011, AAPL: 0.041, ...]  shape (n_stocks,)

Ω 벡터 (종목별 예측 불확실성):
  Ω = diag( var(OOS 잔차_MSFT), var(OOS 잔차_INTC), ... )  shape (n×n)

↓

Black-Litterman:
  μ_BL = f(Q, Ω, π, Σ, τ)
  Σ_BL = g(Q, Ω, π, Σ, τ)

↓ MVO (성향별 γ)

w* = 최적 포트폴리오 비중
```
