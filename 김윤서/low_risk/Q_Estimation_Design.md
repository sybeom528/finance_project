## 1. BL에서 Q의 역할

Black-Litterman 핵심 공식:

```
μ_BL = π + τΣP'(PτΣP' + Ω)⁻¹(Q - Pπ)
```

| 기호 | 의미 | 단위 |
|------|------|------|
| π | CAPM 균형 기대수익 | 월별 수익률 |
| P | 뷰 벡터 (저위험 +, 고위험 −, 시총 가중) | — |
| **Q** | **"내 뷰: 이 포트폴리오가 낼 수익률"** | **월별 수익률** |
| Ω | 뷰 불확실성 = τ × P'ΣP | 분산 |
| τ | BL 스케일 파라미터 (보통 0.05~0.1) | — |

### Q - P·π 의 의미: "CAPM 대비 초과 뷰"

```
Q - P·π > 0  → BL이 저위험 tilt 강화 (의도한 방향)
Q - P·π = 0  → CAPM과 동일 (뷰 없음)
Q - P·π < 0  → BL이 저위험 tilt 약화 또는 역전 ← 위험!
```

**Q는 반드시 P·π와 같은 단위(월별 수익률)여야 한다.**

## 2. 핵심 문제: Q_hist/Q_momentum이 왜 실패했나

### 2011-2025 저위험 이상현상 역전

강세장·테크 불마켓에서는 고변동성 종목이 오히려 더 올랐다:

```
P·R_t = (저위험 그룹 수익) - (고위험 그룹 수익)
      = 음수가 빈번
```

| 방법 | 윈도우 | 결과 | 문제 |
|------|--------|------|------|
| Q_hist | 60개월 | 거의 항상 양수이지만 0에 가까움 | 과집중 유발, 변동성 ↑ |
| Q_momentum | 12개월 | 최근 추세 반영 → 음수 빈번 | Q < P·π → BL 역방향 작동 → MDD -42% 폭발 |

**핵심**: Q가 음수가 되면 `Q - P·π < 0`이 되어 BL이 고위험 종목으로 기울어진다. 저위험 전략이 고위험 전략으로 뒤집히는 것이다.

## 3. 월별 데이터 + 일별 공분산 설계

### 설계 목적

월별 수익률만 사용하면 N=500일 때 p/T = 500/60 = 8.3 → 공분산 행렬 추정 불안정.  
일별 수익률로 Σ를 추정하면 T가 1260으로 늘어나 p/T = 0.40으로 개선된다.

### 올바른 단위 처리

```python
# 1. 일별 수익률로 Σ 추정 (학습 기간 60개월 × 21일 = 1260일)
daily_rets = price_close.pct_change().dropna()
Sigma_daily = LedoitWolf().fit(daily_rets.tail(1260)).covariance_

# 2. 월별 단위로 스케일링 (IID 근사)
Sigma_monthly = Sigma_daily * 21

# 3. π, Q, μ_BL은 전부 월별 수익률 단위 유지 → 단위 일관성 확보
```

### 근사 오차 인식

| 조건 | Σ_daily × 21 vs 실제 Σ_monthly |
|------|-------------------------------|
| 일반 기간 | 근사적으로 성립 |
| 위기/고변동성 | 실제 Σ_monthly > 21 × Σ_daily (ARCH 효과, 상관관계 급등) |
| 저변동성 기간 | 실제 Σ_monthly ≈ 21 × Σ_daily (비교적 정확) |

ARCH 효과가 85% 종목에서 확인되므로(03_VolatilityEDA) 스케일링 오차가 존재한다.  
그러나 Ledoit-Wolf 수축(α ≈ 0.2~0.3)을 병용하면 1차 근사로 충분히 합리적이다.

## 4. 구현 완료된 Q 산정 방식

### 4-1. Q_FIXED

```python
Q = Q_OPTIMAL  # 05_Q_Sensitivity에서 Sharpe 최대화 기준으로 탐색한 고정값 (≈ 0.003)
```

**의미**: "저위험 포트폴리오는 매달 약 0.3%의 초과수익을 낸다"는 고정 뷰.  
시장 상황과 무관하게 일정한 뷰 강도를 유지한다.

**장점**: 단순하고 견고함. walk-forward 탐색으로 정당화 가능.  
**한계**: 시장 레짐 변화에 무반응. 위기 때도 동일한 뷰 강도 유지 → 방어력 부족.

**성과**: Sharpe 0.931~1.074 (노트북별 차이), MDD -13.70% ~ -18.81%

### 4-2. Q_hist

```python
def compute_Q_hist(P, ret_matrix):
    # ret_matrix: 학습 기간(60개월) 월별 수익률 행렬 (T × N)
    p_vec = P.reindex(ret_matrix.columns).fillna(0).values
    return float((ret_matrix.values @ p_vec).mean())
    # = mean(저위험 그룹 수익 - 고위험 그룹 수익) over 60M
```

**의미**: "학습 기간 60개월 동안 실제로 관찰된 저위험-고위험 수익 스프레드 평균"을 Q로 사용.  
과거가 미래를 대표한다는 가정.

**장점**: 데이터 기반, look-ahead bias 없음.  
**한계**:
- 2011-2025 강세장에서 스프레드가 0 근처 또는 음수 → Q 과소평가 → 과집중 유발
- 60개월 평균이라 최근 시장 변화에 둔감

**성과**: Sharpe 0.896, MDD -21.57% (Q_FIXED 대비 Sharpe 열세, 변동성 ↑)

---

### 4-3. Q_momentum

```python
def compute_Q_momentum(P, ret_matrix, window=12):
    # 최근 12개월만 사용
    n = min(window, len(ret_matrix))
    p_vec = P.reindex(ret_matrix.columns).fillna(0).values
    return float((ret_matrix.values[-n:] @ p_vec).mean())
```

**의미**: Q_hist와 공식은 동일하되 최근 12개월 스프레드만 반영.  
"최근 추세가 지속된다"는 단기 모멘텀 가정.

**장점**: 시장 변화에 민감하게 반응.  
**한계**: **민감함이 독이 된다.**
- 고변동성 종목이 강세인 달이 이어지면 Q가 음수로 전환
- Q - P·π < 0 → BL이 고위험 방향으로 기울어짐 → 전략 역전
- MDD -42.00%, Sharpe 0.597 (최하위)

**교훈**: 단기 스프레드는 신뢰성이 낮다. 음수 방지 장치 없이 쓰면 위험하다.

### 4-4. Q_lambda ★ 07 노트북 최고 성과

```python
def compute_Q_lambda(lam, q_base, lam_mean=2.5):
    scale = np.clip(lam / lam_mean, 0.1, 3.0)
    return q_base * scale

# lam: 학습 기간 SPY 초과수익 / SPY 분산 (시장 위험회피계수 추정값)
# lam_mean = 2.5 (He & Litterman 2002 장기 평균)
```

**의미**: 위험회피계수 λ로 Q 강도를 조절.

```
시장 안정 (수익 ↑, 변동성 ↓) → λ ↑ → Q 강화 → 저위험 tilt 강하게
시장 불안 (수익 ↓, 변동성 ↑) → λ ↓ → Q 약화 → 포트폴리오 보수적으로
```

**장점**:
- 역사적 스프레드 사용 안 함 → 음수 문제 없음
- 시장 상황에 자동 적응
- clip(0.1, 3.0)으로 극단값 방지

**한계**: λ는 SPY 수익률/분산으로 추정하므로 시장 전체 위험회피를 대리하는 값. 저위험 프리미엄과 λ의 관계가 항상 성립하지 않을 수 있다.

**성과**: Sharpe 1.112, MDD -11.84% (07 노트북 기준 전체 1위)

### 4-5. Q_ff3

```python
# Fama-French 3팩터 회귀로 종목별 기대수익 추정 후 P 가중합
Q_ff3 = P @ expected_returns_ff3
```

**의미**: 시장(Mkt-RF), 규모(SMB), 가치(HML) 팩터로 각 종목의 기대수익을 예측하고, P 벡터로 가중합해 저위험-고위험 스프레드를 산출.

**장점**: 이론 기반, 팩터 모델 체계 활용.  
**한계**: **팩터 미스매치.**
- FF3는 저변동성 이상현상을 직접 설명하는 팩터가 없음
- 월별 변동이 극심, 음수 빈번
- Sharpe 0.586, MDD -51.48% (최하위)

### 4-6. Regime3

```python
REGIME3_MAP = {
    '저변동성': Q_OPTIMAL,       # 강세장 → Q 전력
    '중간':     Q_OPTIMAL * 0.5, # 중간 → Q 절반
    '고변동성': 0.0,             # 위기 → 뷰 없음 (CAPM으로 후퇴)
}
Q = REGIME3_MAP[regime]

# regime: SPY 12개월 롤링 변동성 × expanding 분위수로 3분류
```

**의미**: 시장 레짐을 3단계로 분류하고, 고변동성 국면에서 뷰를 완전히 철회.  
"위기 때는 모른다고 선언"하는 하드스탑 전략.

**장점**: 고변동성 구간에서 BL 조정 제거 → 낙폭 방어  
**한계**: 경계값이 불연속적. 레짐 전환 시점에 Q가 갑작스럽게 바뀜.

**성과**: Sharpe 0.912, MDD -19.76%

### 4-7. Regime+λ ★ 10 노트북 최고 성과

```python
Q = 0.0 if regime == '고변동성' else q_lambda(lam, Q_OPTIMAL)
```

**의미**: Regime3의 하드스탑(고변동성 → Q=0)과 Q_lambda의 연속 스케일링을 결합.

**성과**: Sharpe 0.955, MDD -18.05% (10 노트북 전체 1위)

### 4-8. GARCH_conf

```python
# GARCH 예측 정확도(MAPE)를 12개월 롤링 평균 → expanding 분위수로 [0.3, 1.0] 정규화
Q = Q_OPTIMAL * garch_conf_norm  # conf: 0.3(예측 불량) ~ 1.0(예측 정확)
```

**의미**: GARCH가 변동성을 잘 예측하는 시기에는 Q를 크게(뷰 신뢰), 못 맞히는 시기에는 Q를 작게(뷰 철회).  
"모델 신뢰도에 비례한 뷰 강도 조절."

**장점**: GARCH 예측 품질이 자동으로 Q에 반영됨. GARCH_conf가 높으면 vol 예측이 정확한 시점 → 저위험 분류도 믿을 수 있는 시점.  
**한계**: MAPE 기반 신뢰도가 Q 강도와 항상 비례하는지 불확실. 저변동성 레짐에서 예측이 쉬워지는 구조적 편향 가능.

**성과**: Sharpe 0.939, MDD -19.25%

### 4-9. π_ratio

```python
def q_pi_ratio(P, pi, q_base, spread_ref):
    spread = abs(float(P @ pi))         # 현재 CAPM이 암시하는 저위험-고위험 스프레드
    spread_ref = expanding_median(...)  # 과거 스프레드 중앙값 (look-ahead bias 없음)
    return q_base * np.clip(spread / spread_ref, 0.1, 3.0)
```

**의미**: CAPM 균형(π)이 저위험-고위험 간 스프레드를 크게 부여하는 시기에는 Q도 함께 키운다.  
"CAPM이 이미 저위험 프리미엄을 인정할 때 내 뷰도 강하게."

**장점**:
- 역사적 실현 스프레드 사용 안 함 → 음수 문제 없음
- CAPM 균형과 Q가 동일 방향으로 움직임 → 이론적 일관성
- MDD 최소(-17.82%)

**한계**: P·π 절댓값은 λ, Σ에 동시 의존해 변동이 크다. 스프레드가 작을 때 Q도 작아져 뷰가 약해지는 구조가 항상 옳지는 않다.

**성과**: Sharpe 0.910, MDD -17.82% (MDD 최소)

## 5. 신규 아이디어

### 5-1. Q_vol_spread — 변동성 예측 모델을 Q에 직접 연결

**핵심 아이디어**: 이미 GARCH로 예측한 `vol_predicted`가 있다. 저위험-고위험 그룹의 예측 변동성 격차가 클수록 다음 달 수익 스프레드도 클 것이라는 가설을 검증하고 Q를 추정한다.

```python
# 매 리밸런싱 시점 t에서:
vol_low_pred  = vol_pred[P > 0].mean()           # 저위험 그룹 예측 변동성
vol_high_pred = vol_pred[P < 0].mean()           # 고위험 그룹 예측 변동성
vol_spread_t  = vol_high_pred - vol_low_pred     # 예측 변동성 격차 (항상 양수)

# 학습 기간에서 회귀 추정 (walk-forward, look-ahead bias 없음):
# actual_ret_spread_t+1 = α + β × vol_spread_t + ε
# Q_t = α̂ + β̂ × vol_spread_t
```

**왜 유망한가**:
- 음수 문제 없음: vol_spread는 항상 양수 → Q 항상 양수 방향
- 파이프라인 일관성: GARCH vol 예측이 P 구성에도, Q 추정에도 동시 사용됨
- "변동성 예측을 잘할수록 Q 추정도 정교해진다"는 자기강화 구조

**한계**: α, β 추정에 충분한 학습 기간이 필요. vol_spread → ret_spread 관계 자체가 기간에 따라 달라질 수 있음.

---

### 5-2. Q_IC_weighted — IC로 가중한 역사 스프레드

**핵심 아이디어**: Q_hist는 과거 스프레드를 동등하게 평균한다. 그러나 저위험 신호(vol)가 수익률을 잘 예측했던 시기의 스프레드가 더 신뢰할 만하다. IC(정보계수)로 가중하면 신호가 강했던 시점이 Q에 더 반영된다.

```python
# 12개월 롤링 IC (Spearman correlation between vol and next-month return)
ic_rolling = ic_df['ic'].rolling(12).mean()

# IC 기반 가중치 (음수 IC → 신호 역전 → 0으로 clip)
weights = ic_rolling.reindex(train_dates).clip(lower=0)
weights /= weights.sum() if weights.sum() > 0 else 1.0

# IC 가중 스프레드
P_R_series = ret_matrix.values @ P.values
Q_ic = float((P_R_series * weights.values).sum())
```

**왜 유망한가**:
- 저위험 이상현상이 강하게 작동한 시기의 스프레드만 Q에 반영
- IC < 0인 기간(신호 역전)은 자동으로 down-weight → 음수 완화
- 데이터 기반으로 Q가 시장 환경에 적응

**한계**: IC 자체가 노이즈하다. 12개월 롤링 IC로 가중하면 최근 IC가 낮을 때 Q가 과소평가될 수 있다.

---

### 5-3. Q_shrink — CAPM 균형으로 수축

**핵심 아이디어**: 정보가 없을 때는 Q를 P·π(CAPM 균형 스프레드)쪽으로 수축한다. α=1이면 Q = Q_hist(완전 신뢰), α=0이면 Q = P·π(뷰 없음). α를 IC나 시장 레짐에 연동하면 동적 수축이 된다.

```python
Q_pi = float(P @ pi)          # CAPM 균형 스프레드 (뷰 없음 = 이 값으로 수렴)
Q_hist_val = compute_Q_hist(P, ret_matrix)

alpha = ic_rolling_12m.clip(0, 1)  # IC가 높을수록 Q_hist에 가중
Q_shrink = alpha * Q_hist_val + (1 - alpha) * Q_pi
```

**왜 유망한가**:
- α=0일 때 Q - P·π = 0 → BL이 CAPM으로 자동 후퇴 (안전 장치)
- Q_hist가 음수여도 수축이 완충 역할
- IC와 연동하면 신호 강도에 비례한 자동 조절

**한계**: α 설계 방식에 따라 결과가 크게 달라짐. 추가 튜닝 필요.
