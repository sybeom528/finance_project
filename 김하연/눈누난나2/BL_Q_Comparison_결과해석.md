## 3. Q 값 자체의 분포 분석 (핵심 발견)

| 방법 | Q 평균(%/yr) | Q std | 95% CI |
|---|---|---|---|
| **Idzorek** | 11.66 | 8.52 | [11.46, 11.86] |
| **변동성스프레드** | 11.66 | 6.99 | [11.50, 11.83] |
| **변동성모멘텀** | 11.66 | 8.37 | [11.46, 11.86] |
| RS(2국면) | 13.94 | 9.76 | [13.70, 14.17] |
| RS(3국면) | 13.69 | 9.70 | [13.46, 13.92] |
| GARCH | 17.21 | 15.62 | [16.83, 17.58] |
| GJR-GARCH | 16.00 | 15.18 | [15.64, 16.36] |
| **pi(기준)** | 11.66 | 8.03 | [11.47, 11.85] |

### 발견 1: Idzorek · 변동성스프레드 · 변동성모멘텀의 Q 평균 = pi 평균

세 방법의 Q 평균(11.66%)이 pi(11.66%)와 완전히 일치하고, 95% CI도 pi의 CI와 겹친다.

**이유**: 이 세 방법은 모두 `Q = pi ± f(signal)` 형태이며 신호의 횡단면 평균이 0(z-score 정규화)이기 때문에 Q의 평균이 pi의 평균과 같아진다. 즉 **개별 종목의 뷰 방향은 다르지만, 시장 전체에 대한 평균 기대수익률은 pi와 동일**하다.

→ BL 모델이 이 뷰를 반영해도 aggregate 기대수익률이 바뀌지 않으므로 동일가중과 사실상 동일한 포트폴리오가 나온다.

### 발견 2: GARCH · GJR-GARCH는 체계적 낙관 편향

GARCH Q 평균(17.21%)이 pi(11.66%)보다 **5.55%p** 높다. 95% CI가 전혀 겹치지 않아 통계적으로 유의한 차이다.

**이유**: GARCH 방법은 `Q = pi × (hist_vol / garch_vol)`을 사용한다. 롤링 252일 창의 역사적 변동성(hist_vol)이 최근 EWMA 조건부 변동성(garch_vol)보다 대체로 높은 상태에서 끝나기 때문이다 — 시장이 하락한 뒤 진정되면 창 끝의 garch_vol이 낮아지고 비율이 1보다 커진다. 이 구조적 편향이 Q를 pi보다 지속적으로 높게 만들고, 결국 과도하게 낙관적인 뷰로 운용되어 성능이 저하된다.

### 발견 3: RS(2/3국면)은 중간

RS 방법의 Q 평균(13.94%, 13.69%)은 pi(11.66%)보다 약 2%p 높다. CI가 pi CI와 겹치지 않아 통계적으로 구별되지만, 편향 크기가 GARCH보다 작다.

**이유**: Bull 국면에서 scale > 1, Bear 국면에서 scale < 1이지만 Bull 비중이 더 많아 평균적으로 약간 낙관적이다. 이 정도 편향은 성능에 큰 영향을 주지 않아 RS가 여전히 양호한 성능을 낸다.

## 5. 종합 결론

### 왜 대부분의 방법이 동일가중과 같은가?

BL 모델의 포트폴리오 출력은 Q의 **방향과 크기** 모두에 민감해야 한다. 그런데:

1. **Idzorek · 변동성스프레드 · 변동성모멘텀**: Q 평균 = pi 평균 → 집합적 뷰가 없음 → posterior가 prior(pi)에 수렴 → equal-weight에 가까운 결과
2. **RS(2/3국면)**: Q가 pi를 약간 상회하지만 차이가 작고, 모든 종목에 동일한 국면 배율이 적용 → 횡단면 종목 선택 신호가 없고 시장 타이밍 신호만 있음 → equal-weight에 가까운 결과

### 방법별 활용 권고

| 상황 | 권장 방법 | 이유 |
|---|---|---|
| 기본 운용, 안정성 우선 | **Idzorek** | 동일가중과 동일하나 해석 용이, 구현 단순 |
| 위험 조정 수익 극대화 | **변동성스프레드** | 샤프 0.798로 전 방법 최고 |
| 위기 방어 (VIX 활용 가능) | **내재변동성** | GFC +60.8%, COVID +44.6% |
| GARCH 계열 사용 시 | **사용 비권장** | 체계적 낙관 편향으로 Q가 pi보다 5%p 이상 높음 |

### Q 설계 개선 제안

현재 모든 방법의 근본 문제는 **Q의 횡단면 평균이 pi에서 벗어나지 않거나 일방적으로 높다**는 것이다. 개선 방향:

```
Q_개선 = pi + scale × signal
  - scale ≈ 0.5 × pi.std()  (≈ 6~7%/yr per z-score unit)
  - signal: 각 방법의 신호를 횡단면 정규화
  - 조건: 횡단면 signal 평균 = 0 확보 (체계적 편향 제거)
```

GARCH 계열은 이 조건을 위반하므로 `Q = pi × ratio` 대신 `Q = pi + clip(ratio-1, -0.3, 0.3) × pi.std()` 형태로 재설계 필요.
