## 1. 실험 설계 배경

9차 Run(H=14, 단일 LSTM)에서 SPY·QQQ 모두 R²_OOS > 0을 달성했지만 두 가지 구조적 문제가 잔존했다.

- **Mean collapse**: pred_std/true_std ≈ 0.21 → 예측값이 실제 변동의 20%만 커버
- **상승 편향**: 하락 예측 정확도 SPY 2.8%, QQQ 7.8%

이를 개선하기 위해 **CEEMDAN(Complete Ensemble EMD with Adaptive Noise)** 을 도입, 논문(CGL-BL) 의 CEEMDAN-LSTM 구조를 채택한다.

| 단계 | 설명 |
|---|---|
| Stage 1 | CEEMDAN으로 log_return → IMF₁ … IMF_K 분해 (전체 시계열 1회) |
| Stage 2 | 각 IMF_k에 대해 독립 LSTM_k 훈련 (INPUT_SIZE=1, 단일 채널) |
| Stage 3 | fold별 예측 합산: ŷ = Σ_k ŷ_k (수학적 보장: Σ target_k = original_target) |

**파라미터**: IS=756, PURGE=21, EMB=21, OOS=21, STEP=21, SEQ_LEN=21, horizon=21 (BL 월별 리밸런싱 1:1 정합).

## 3. 버그 원인 분석 — QQQ IMF 수 불일치

QQQ의 catastrophic failure 원인은 **Stage 2·3에서 IMF 수를 전역 고정**한 데 있다.

### 현상

```
CEEMDAN 분해 결과:
  SPY: 10 IMFs  →  ceemdan_extra['SPY'].shape = (2514, 10)
  QQQ: 11 IMFs  →  ceemdan_extra['QQQ'].shape = (2514, 11)

버그 코드:
  N_IMFS = ceemdan_extra['SPY'].shape[1]  # = 10 (SPY 기준 전역 고정)

  for k in range(N_IMFS):   # range(10) — QQQ의 IMF₁₁ 누락
      ...

  # Stage 3 합산
  y_true_sum = Σ target_k for k in range(N_IMFS)   # k=0~9만 합산
  # QQQ의 IMF₁₁ 타깃이 빠진 y_true_sum ≠ original_target_21d
```

### 수학적 파괴 경로

CEEMDAN의 수학적 보장은 `Σ_k IMF_k = log_ret` (완전 재구성). 이로부터:

```
Σ_k [IMF_k.rolling(21).sum().shift(-21)] = original_target_21d
```

QQQ는 IMF₁₁이 빠진 10개만 합산 → y_true_sum ≠ original_target. **y_true 자체가 오염**되어 R²가 의미를 잃는다. min=-199.73은 특정 폴드에서 y_true_sum의 편향이 극단적으로 발산한 결과다.

### 수정 방법

```python
# 버그: N_IMFS 전역 고정
for k in range(N_IMFS):  # SPY=10, QQQ도 10 (잘못)

# 수정: ticker별 동적 결정
n_imfs_ticker = ceemdan_extra[ticker].shape[1]
for k in range(n_imfs_ticker):  # SPY=10, QQQ=11 (올바름)
```
