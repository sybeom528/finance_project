## 6. 관문 기준 재정의 (현행 유지)

> 제안서 §5 내용대로, 현재는 `hit_rate > 0.55` 관문을 그대로 유지. 이후 확인을 위해 아래 기록.

### 왜 현행 0.55 기준이 부적절한가

`2026-04-25_Run결과분석.md` §4 요약:
- previous baseline hit_rate = **0.92** (21일 윈도우 20일 겹침으로 강한 자기상관)
- LSTM hit_rate 0.64 → 관문 0.55 통과했으나 **baseline보다 28%p 낮음**
- `hit_rate > 0.55` = "동전 던지기보다 낫다" 수준 → 실질 학습 여부와 무관

### 강화 기준 (추후 재검토용)

```python
# 적용 방법 (§9 Cell 22 관문 판정 코드 변경)
# Before
gate_hit = summary['hit_rate']['mean'] > 0.55

# After (강화 기준)
prev_baseline_hit = bl['previous']['hit_rate']   # ≈ 0.92
margin = 0.02
gate_hit_strict = summary['hit_rate']['mean'] > prev_baseline_hit + margin
# 기준: hit_rate > 0.94 (0.92 + 0.02)
```

**margin 0.02 근거**: n_oos ≈ 21 × 105 = 2,205 표본 → 베르누이 비율 차이 0.02가 p < 0.05 유의한 최소 표본 약 2,000 부합.

**2차 Run 결과에 새 기준 적용 시 판정**:

| 지표 | 2차 Run | 새 기준 | 판정 |
|---|---|---|---|
| SPY Hit Rate | 0.6313 | > 0.94 (0.9265 + 0.02) | ❌ FAIL |
| QQQ Hit Rate | 0.5968 | > 0.94 (0.9252 + 0.02) | ❌ FAIL |
| SPY R²_OOS | -0.2118 | > 0 | ❌ FAIL |
| QQQ R²_OOS | -0.5472 | > 0 | ❌ FAIL |

→ 강화 기준으로는 두 관문 모두 명확 FAIL. 피처 확장 후 3차 Run에서 재판정 예정.
