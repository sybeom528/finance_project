## 논의 1. 파라미터 과잉과 과적합 방지

### 1-1. 배경

#### 현 구성의 숫자

| 항목 | 값 |
|---|---|
| LSTM 파라미터 수 | **199,297** |
| 훈련 시퀀스 수 (IS 231 − seq_len 126 → train 80%) | **84** |
| val 시퀀스 수 (20%) | **21** |
| 파라미터/훈련 샘플 비율 | **≈ 2,372배** |

#### 현 구성이 이렇게 된 이유

1. **관행적 기본값 수용**: `hidden=128, num_layers=2, dropout=0.2` 는 PyTorch LSTM 튜토리얼·학습자료의 표준 예시. Plan 작성 시 별다른 재검토 없이 차용.
2. **원 논문(Su et al. 2026 ESWA) 규모 재현**: 원 논문은 다변량 피처로 학습해 실질 샘플이 많았을 것. Phase 1 은 univariate 로 축소하면서 **모델 규모는 그대로 유지**.
3. **가설 검증 설계**: "univariate LSTM 이 통상적 수준에서 실패한다" 를 보이는 것이 Phase 1 목적 → 의도적으로 축소하지 않음.
4. **Walk-Forward 신뢰 가정**: Purge+Embargo 가 일반화 오차를 정직하게 추정해주리라 기대. 실제로는 fold 간 성능 편차 폭발로 신뢰 가능한 점추정 어려움.

#### 일반 딥러닝과의 차이점

BERT·GPT·ResNet 등 현대 딥러닝은 파라미터 > 샘플 비율에서도 잘 작동합니다. 그러나 **금융 시계열은 그 경험칙이 잘 통용되지 않는 영역**입니다:

| 조건 | 일반 딥러닝 | 금융 시계열 |
|---|---|---|
| Signal-to-Noise | 높음 (이미지·언어에 강한 구조 존재) | **매우 낮음** (가격 변동 대부분이 random) |
| 표본 독립성 | 대체로 만족 | 인접 시점 강한 의존, 비독립 |
| 정상성 (stationarity) | 대체로 유지 | 체제 전환 빈번 (COVID, 인플레, 금리 변화) |
| 전이학습 기반 | 대규모 사전학습 데이터 풍부 | 희소 |
| 관측량 | 이미지 수백만 장 가능 | 티커 1개 × 10년 ≈ 2,500일 한계 |

→ 금융 시계열에서는 **작은 모델 + 강한 정규화** 가 실용적이라는 것이 퀀트 문헌 (López de Prado 2018) 의 일관된 권고.

#### 실측 과적합 증상

- `best val loss` 매우 낮음 (0.000008 수준, §4 sanity 에서 관찰) → train/val 에서는 완벽
- 그러나 OOS R²_OOS 는 평균 음수, min -37 까지 추락
- fold 마다 다른 학습 결과 → variance 폭발
- 전형적인 "train 은 외웠지만 test 에서 무너지는" 과적합 패턴

### 1-2. 논의할 해결책

#### A. 모델 축소 (구조적 — 가장 직접적)

| 변경 | Before | 제안 | 파라미터 영향 |
|---|---|---|---|
| hidden_size | 128 | **32** 또는 16 | 199k → 약 4.5k 또는 1.3k |
| num_layers | 2 | **1** (+ head_dropout) | 상위 층 제거 |

**장점**: 파라미터를 2~3 자릿수 줄여 샘플 대비 비율 개선. 구현 간단 (LSTMRegressor 인자만 변경).

**단점**: 표현력 감소 → underfit 가능성. 단 현 상태는 명백한 overfit 이므로 축소 여지가 큼.

**참고**: Setting B 가 이미 hidden=64, 1-layer 로 설계되어 있음 (PLAN.md §의사결정 표). 설정 A 도 이 방향으로 수렴시키는 것이 일관성 있음.

#### B. 정규화 강화

| 항목 | Before | 제안 |
|---|---|---|
| dropout | 0.2 | **0.3~0.4** |
| weight_decay | 1e-4 | **1e-3** |
| Recurrent Variational Dropout | 미적용 | 도입 검토 (LSTM 특화) |

**장점**: 모델 구조 보존하면서 학습 시 무작위성·제약 추가. 튜닝 비용 낮음.

**단점**: dropout 너무 높이면 학습 불안정. 금융 시계열 noise 가 이미 많아서 0.5+ 는 위험.

**추가 고려**: 학습자료_주의사항 §4.4 의 "LSTM 1-layer dropout 무시 함정" — 현 `scripts/models.py` 가 이미 `num_layers=1` 시 별도 `nn.Dropout` 을 head 앞에 적용하도록 구현됨. num_layers=1 로 전환해도 dropout 정상 작동.

#### C. 학습 조기 중단 강화

| 항목 | Before | 제안 |
|---|---|---|
| EarlyStopping patience | 10 | **5** |
| max_epochs | 50 | **30** |
| LR scheduler patience | 5 | 3 |

**장점**: 과적합 전 단계에서 중단하여 generalization 개선.

**단점**: val=21 샘플이 작아 noise 민감 → 조기 중단 오판 가능. **val 크기 자체를 키우는 방법** (IS 안에서 val 비율 상향, 또는 seq_len 축소로 시퀀스 수 증가) 과 병행 필요.

#### D. 데이터 풀링 (멀티 티커 공동 학습)

현재: SPY 모델과 QQQ 모델을 **독립 학습**.

제안: **SPY + QQQ + VOO + IVV + VTI + DIA** 같은 여러 ETF 를 한 모델로 공동 학습.
- 티커 정보를 embedding 으로 추가하거나
- 단순 pool 하여 ticker-agnostic 학습

**장점**:
- 훈련 샘플 5~10배 증가 (84 → 500+)
- 체제 다양성(여러 지수의 서로 다른 변동 양상) 노출 → generalization 향상
- 시장 공통 구조를 학습할 기회

**단점**:
- ticker 별 특성 희석 가능
- 구현 복잡도 증가 (데이터 로더 multi-ticker, target_series 확장)
- Phase 1 범위 밖 — Phase 1.5 또는 Phase 2 검토

#### E. 앙상블

서로 다른 seed 로 k (예: 5~10) 개 모델 학습 → 예측 평균.

**장점**:
- variance 감소 — 현재 fold 간 편차 큰 문제 직접 완화
- 모델 변경 없이 적용 가능 (학습 함수만 반복 호출)

**단점**:
- 학습 시간 k배 (10분 → 100분)
- 체크포인트·메트릭 저장 복잡도 증가 (현재는 fold 당 1 모델)

#### F. 사전 학습 (장기)

대규모 금융 시계열 (수백 티커 × 수십 년) 로 pretrain → 특정 자산 fine-tune.

**장점**: NLP 의 transfer learning 과 유사한 혜택, 데이터 효율 극대화.

**단점**:
- 대규모 데이터 수집·전처리 부담
- Phase 1~3 범위 밖 (Phase 4 이후 또는 별도 연구 project)

## 논의 2. seq_len 재조정

### 2-1. 배경

현재 `seq_len = 126` (약 6개월 lookback). PLAN.md 에 "약 6개월" 이라고 명시되어 있으나, 근거는 원 논문·학습자료 관행 수용.

#### 현 구성의 문제

1. **훈련 샘플 수 심각히 부족**
   - IS=231 기준 훈련 시퀀스 = 231 − 126 = **105개**
   - train 80% 적용 시 실제 학습은 84개
   - 파라미터 199k 대비 턱없이 부족

2. **일별 수익률 ACF 가 매우 약함**
   - Phase 1 Step 1 측정값: SPY lag 1 = -0.13, lag 2 이후 |값| < 0.1
   - **오래된 정보의 예측력 거의 없음**
   - 긴 lookback 은 정보 기여 없이 **샘플 수만 깎아먹는 구조**

3. **val 크기 불안정**
   - IS=231, seq_len=126 → train 시퀀스 105개
   - val 20% = 21 시퀀스 — EarlyStopping noise 에 민감
   - seq_len 을 줄이면 val 도 자연히 커짐

#### 수치 트레이드오프 (IS=231 기준)

| seq_len | 훈련 시퀀스 수 | 대략 기간 | 파라미터/샘플 비 | 평가 |
|---|---|---|---|---|
| 21 | 210 | 1개월 | 하락 | 샘플 풍부, ACF lag 1 만 활용 가능 |
| 42 | 189 | 2개월 | 하락 | 타협 |
| **63** | **168** | **3개월** | 하락 | **권장 출발점** |
| 84 | 147 | 4개월 | 중간 | — |
| 126 (현재) | 105 | 6개월 | 최악 | 샘플 부족 |
| 252 | 없음 | 12개월 | N/A | IS 231 < 252 → 불가 |

### 2-2. 논의할 선택지

#### 옵션 1: seq_len = 63 (권장 출발점)
- 3개월 lookback — 분기 단위 추세 포착 가능
- 훈련 시퀀스 168개로 60% 증가
- val 21 → 약 33 으로 증가, EarlyStop 안정

#### 옵션 2: seq_len = 21 (보수적 최소)
- 1개월 lookback — 월별 체제만 반영
- 훈련 시퀀스 210개로 2배 증가
- 설정 A·B 의 시간 스케일이 유사해짐 (A 일별·월 lookback, B 월별·2년 lookback)
- 너무 짧아 LSTM 의미 퇴색 가능

#### 옵션 3: seq_len = 42 (중간)
- 2개월 lookback
- 훈련 시퀀스 189개
- 63 과 21 사이 절충

### 2-3. 권장

**1차 Run**: `seq_len=63` 으로 설정하여 샘플 수 확보 + 분기 lookback 유지.
**2차 비교 Run**: `seq_len=21` 로 1차와 비교하여 "짧을수록 나은가" 검증.

### 2-4. 변경 시 점검 항목

- `02_setting_A_daily21.ipynb` §5, §6 의 `SEQ_LEN` 상수 변경
- §8 `run_all_folds` 는 `seq_len` 을 인자로 받으므로 자동 반영됨

- `build_train_val_loaders` 의 `val_ratio=0.2` 유지 시 val 크기 자동 재계산 — 별도 수정 불필요

## 논의 3. 입력 피처 확장 (Univariate → Multivariate)

### 3-1. 배경

#### 현 구성

- **Input**: 일별 log-return 단일 채널 `(B, T, 1)`
- **Target**: 21일 누적 forward log-return `sum(log_ret[t+1:t+22])`

#### 관찰된 한계

1. **타깃의 자기상관이 매우 강하나 LSTM 이 활용 못함**
   - Previous baseline hit_rate = 0.92 = **21일 누적 수익률 자체의 인접 시점 부호 일치율**
   - 즉 "지금까지의 21일 누적 경향 ≈ 앞으로 21일 누적 경향" 관계가 강함
   - LSTM 은 일별 log-return 만 보므로 **이 누적 스케일의 자기상관을 직접 관찰 못함**
   - 일별 noise 속에서 저주파 누적 추세를 스스로 추출해야 하는 **이중 작업** 부담

2. **시간 스케일 불일치**
   - 입력: σ ≈ 0.011 (일별)
   - 타깃: σ ≈ 0.022 (21일 누적, 약 2배)
   - LSTM 이 스케일 점프를 학습해야 하는데 단변량으로는 힌트 부족

3. **원 가설 재검증 필요성**
   - Phase 1 원 가설: "univariate 한계"
   - 부분적으로 맞음 (R²_OOS 음수) 이지만, previous baseline 이 0.92 를 찍는 환경에서는 **"피처 공학 부족" 이 주 원인** 으로 해석 가능
   - 다변량 입력으로 재검증 필요

### 3-2. 추가 피처 후보

#### 피처 1: **Y_trailing (과거 21일 누적)** — 최우선 권장

```python
Y_trail[t] = sum(log_ret[t-20:t+1])   # 과거 21일, 현재 포함
```

- 타깃(21일 forward 누적) 과 **동일 시간 스케일**
- Previous baseline 의 핵심 신호 — 자기상관 — 을 모델이 직접 관찰 가능
- **누수 체크**: `Y_trail[t]` 는 `log_ret[t-20:t+1]` 만 사용 → 과거·현재만, 미래 없음 ✅
- 타깃 `Y_forward[t] = sum(log_ret[t+1:t+22])` 와 겹치는 term 없음 ✅

#### 피처 2: Rolling mean (단기·중기·장기)

```python
mean_5[t]  = log_ret.rolling(5).mean()[t]
mean_21[t] = log_ret.rolling(21).mean()[t]
mean_63[t] = log_ret.rolling(63).mean()[t]
```

- 단기·중기·장기 추세 요약
- 일별 noise 에서 저주파 신호 추출
- 누수: trailing rolling → 없음 ✅

#### 피처 3: Rolling std (변동성)

```python
std_5[t]  = log_ret.rolling(5).std()[t]
std_21[t] = log_ret.rolling(21).std()[t]
```

- 변동성 클러스터링 (학습자료_주의사항 §2.5 "fat-tail, 이분산") 반영
- 시장 스트레스 구간 신호
- 누수: 없음 ✅

#### 피처 4: 상대 위치 지표 (평균회귀 신호)

```python
z_21[t] = (log_ret[t] - mean_21[t]) / (std_21[t] + eps)
```

- Bollinger Band 스타일 Z-score
- 평균회귀 신호 포착 (ACF lag 1 음수 → 평균회귀 힌트)
- 누수: 없음 ✅

#### 피처 5: 거래량 파생 (Phase 1.5 이후)

```python
vol_ratio[t] = volume[t] / volume.rolling(21).mean()[t]
```

- 거래 강도 (시장 관심·유동성)
- 원 데이터 `SPY.csv` 의 `Volume` 컬럼 활용 가능
- 누수: 없음 ✅

#### 피처 6: 외부 지수 (Phase 2+ 검토)

- VIX (변동성 지수)
- 10년 국채 수익률
- 원유/달러 인덱스

- 매크로 체제 반영

## 논의 4. Phase 1 관문 재정의 (vs Previous Baseline)

### 4-1. 배경

#### 현 관문의 한계

기존 PLAN.md 확정 관문:

| 지표 | 임계값 | SPY 결과 | QQQ 결과 | 판정 |
|---|---|---|---|---|
| Hit Rate | > 0.55 | 0.6442 | 0.6006 | ✅ PASS |
| R²_OOS | > 0 | -0.1552 | -0.5242 | ❌ FAIL |

위 표만 보면 "Hit Rate 는 통과했고 R²_OOS 만 실패" 로 해석된다. 그러나 §0 에서 확인한 **previous baseline (직전 값 복사) 의 hit_rate 가 0.92** 라는 사실을 합쳐 보면, **0.55 임계값 자체가 의미를 잃는다**:

```
LSTM hit_rate = 0.64
previous baseline hit_rate = 0.92   ← 단순히 "직전과 같다" 만 답해도 92% 적중
임계값 0.55 = 동전 던지기 + α       ← 무의미한 기준
```

즉 0.55 통과는 **"동전 던지기보다는 낫다"** 는 약한 진술일 뿐이며, 모델이 실질적으로 **신호를 학습했는지** 와는 무관하다.

#### 왜 21일 누적 수익률에서 previous baseline 이 0.92 인가

`target[t] = sum(log_ret[t+1:t+22])`, `target[t-1] = sum(log_ret[t:t+21])` 두 윈도우는 **20일이 겹친다**. 따라서 두 값의 부호는 거의 항상 같음 (인접 시점 21일 누적 수익률은 강한 자기상관). previous baseline 의 0.92 hit_rate 는 모델 성능이 아니라 **타깃 자체의 자기상관** 을 측정한 결과.

→ 이 환경에서 LSTM 이 의미있는 학습을 했다고 주장하려면 **previous baseline 을 유의하게 넘어야** 한다.

### 4-2. 제안 — 관문 정의 변경

#### 변경안

| 항목 | Before | After |
|---|---|---|
| Hit Rate 관문 | `> 0.55` | `> previous_baseline.hit_rate + margin` (margin = 0.02) |
| R²_OOS 관문 | `> 0` | 유지 (zero baseline 대비 개선 의미 그대로) |
| baseline 비교 의무 | 표만 출력 | **PASS/FAIL 판정에 직접 반영** |
| 종합 PASS | 두 관문 모두 만족 | 두 관문 모두 만족 (정의만 강화) |

#### margin 0.02 근거

- `n_oos = 21 (일별) × 106 (fold) = 2,226` 표본
- 베르누이 비율 차이 검정에서 0.02 차이가 유의(p < 0.05)하려면 표본 약 2,000 필요
- 본 실험 표본 수가 그 임계 부근 → margin 0.02 가 통계적·실무적 양쪽에서 합리적 출발점

#### 적용 후 재판정 (Run 결과 그대로 적용 시)

| 지표 | 값 | 새 임계 | 새 판정 |
|---|---|---|---|
| SPY Hit Rate | 0.6442 | > 0.94 (0.92+0.02) | ❌ FAIL (큰 차이) |
| QQQ Hit Rate | 0.6006 | > 0.94 (0.92+0.02) | ❌ FAIL (더 큰 차이) |
| SPY R²_OOS | -0.1552 | > 0 | ❌ FAIL |
| QQQ R²_OOS | -0.5242 | > 0 | ❌ FAIL |

→ 새 관문 적용 시 **두 관문 모두 명확히 FAIL**. Hit Rate 의 "혼동을 일으키는 PASS" 가 사라지고 실패 원인 진단이 단일화된다.

### 4-3. 구현 방법
