### 2-6. 입력 피처 확장 (log_ret → 4채널) [2순위]

**파일**: `02_setting_A_daily21.ipynb §8` run_all_folds 호출 전 + 함수 정의부

#### 추가할 피처

| 피처 | 공식 | 누수 여부 | 역할 |
|---|---|---|---|
| Y_trailing | `rolling(21).sum()` | 없음 ✅ | previous baseline의 핵심 신호 직접 노출 |
| mean_21 | `rolling(21).mean()` | 없음 ✅ | 중기 추세 |
| std_21 | `rolling(21).std()` | 없음 ✅ | 중기 변동성 클러스터링 |

→ input_size = 1 (log_ret) + 3 (extra) = **4**

#### 피처 계산 및 전달 (§8 상단에 추가)

```python
# ticker 루프 전에 또는 루프 안에서 계산
lr_series = pd.Series(raw_dict[ticker]['log_return'].values)

y_trailing = lr_series.rolling(21).sum().values
mean_21    = lr_series.rolling(21).mean().values
std_21     = lr_series.rolling(21).std().values

extra = np.column_stack([y_trailing, mean_21, std_21])
extra = np.nan_to_num(extra, nan=0.0)  # rolling 초기 NaN → 0

results = run_all_folds(
    ticker,
    raw_dict[ticker]['log_return'].values,
    targets_dict[ticker].values,
    folds, SEQ_LEN,
    extra_features=extra,   # ← 추가
    input_size=4,           # ← 1 + 3
    ...
)
```

#### run_all_folds 함수 내부 수정 필요

현재 `run_all_folds`는 `extra_features`와 `input_size`를 받지 않음.

```python
# 시그니처에 추가
def run_all_folds(

    extra_features=None,   # ← 추가

# 내부 build_fold_datasets 호출부 수정
tr_ds_k, te_ds_k, _ = build_fold_datasets(
    series, tr_idx, te_idx, seq_len,

    extra_features=extra_features,   # ← 추가

# 모델 생성부 수정

```

**누수 검증 의무**: 새 피처 추가 후 §4 스타일 assert 검증 추가.

## 5. 나중에 확인할 사항 — Phase 1 관문 재정의

> 현재는 `hit_rate > 0.55` 관문 그대로 유지. 아래는 추후 성능 개선 후 재검토용.

### 문제

현행 관문 `hit_rate > 0.55`가 사실상 무의미함:

- **previous baseline** (직전 21일 누적 수익률 부호 복사) hit_rate = **0.92** (SPY 0.9236, QQQ 0.9209)
- LSTM hit_rate 0.64가 관문 0.55를 통과했으나 baseline보다 28%p 낮음
- 0.55 관문은 "동전 던지기 + α" 수준 → 실질 학습 여부와 무관

### 왜 previous baseline이 0.92인가

`target[t] = sum(log_ret[t+1:t+22])`와 `target[t-1] = sum(log_ret[t:t+21])`이 20일 겹침
→ 인접 시점 21일 누적 수익률이 강한 자기상관 → 직전 값 부호 복사만으로 92% 적중.

### 제안 변경안 (미래 검토용)

```python
# §9 관문 판정 코드 변경안
prev_baseline_hit = bl['previous']['hit_rate']   # ≈ 0.92
margin = 0.02
gate_hit_strict = summary['hit_rate']['mean'] > prev_baseline_hit + margin
# → 기준: hit_rate > 0.94 (0.92 + 0.02)
```

**새 기준 적용 시 현재 Run 재판정**:

| 지표 | SPY | QQQ | 새 기준 | 판정 |
|---|---|---|---|---|
| Hit Rate | 0.6442 | 0.6006 | > 0.94 | ❌ FAIL |
| R²_OOS | -0.1552 | -0.5242 | > 0 | ❌ FAIL |

→ 수정 후 재 Run에서 이 기준으로도 체크해 볼 것.

**margin 0.02 근거**: n_oos = 21 × 106 = 2,226 표본에서 베르누이 비율 차이 0.02가 p < 0.05 유의하려면 표본 약 2,000 필요 → 합리적 출발점.
