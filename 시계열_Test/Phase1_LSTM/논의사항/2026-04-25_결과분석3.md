## 4. 현재 상태 진단

### 4-1. 왜 모델 축소 후에도 성능이 오히려 나빠졌는가

| 요인 | 설명 |
|---|---|
| **정보 병목 (핵심 원인)** | 입력 1채널 (일별 log_return). 타깃의 핵심 자기상관 신호(21일 누적 수익률)를 LSTM이 직접 관찰 불가 |
| 용량 과잉 축소 가능성 | hidden 128→32: 파라미터 1/44. 미약한 신호조차 포착 못하는 underfit 진입 가능 |
| fold 간 분산 유지 | R²_OOS std: SPY 1.18, QQQ 1.91 — 여전히 불안정 |

**결론**: 정규화·구조 축소는 필요 조건이지만, 피처 공학 없이는 충분 조건이 안 됨.

### 4-2. hidden=64 먼저 올릴지 vs 피처 확장 먼저 할지

**피처 확장이 먼저가 맞다.** 이유:

현재 LSTM이 train_mean baseline조차 못 이기는 근본 원인은 **볼 수 있는 정보 자체가 빈약**하기 때문이다. hidden을 32→64로 늘리면 파라미터는 늘어나지만, 1채널로 들어오는 신호가 빈약하면 더 큰 모델이 있어도 쓸 게 없다.

```
현재:
  log_return (1채널) → hidden=32 LSTM → 예측
  타깃(21일 누적)의 자기상관을 직접 못 봄

피처 확장 후:
  [log_return, Y_trailing, mean_21, std_21] (4채널) → hidden=32 LSTM → 예측
  previous baseline이 쓰는 신호를 모델이 직접 관찰 가능
```

또한 피처가 4채널이 되면 입력 레이어 파라미터 자체가 늘어나 hidden=32여도 실질 capacity가 올라간다:

| 구성 | 파라미터 수 | 훈련샘플/fold | param/sample 비 |
|---|---|---|---|
| 현재: input=1, hidden=32 | 4,513 | 134 | 34 |
| 피처확장: input=4, hidden=32 | 4,769 | 134 | 36 |
| 피처확장: input=4, hidden=64 | 17,729 | 134 | 132 |

세 구성 모두 1차 Run의 2,372배 대비 압도적으로 개선된 수준이므로, 피처 확장 후 hidden=32로도 충분할 수 있다.

**hidden=64는 피처 확장 후 결과를 보고 결정.** 두 가지를 동시에 바꾸면 어느 쪽이 효과인지 분리가 안 된다.

## 5. 다음 단계 — 4차 Run (피처 확장)

2차·3차 Run에서 구조·정규화 최적화 완료. 이제 근본 원인인 **정보 병목** 해소가 필요.

> hidden 크기 조정은 피처 확장 이후에 판단한다. 두 변수를 동시에 바꾸면 원인 분리가 안 되므로, 4차 Run(피처 확장, hidden=32 유지) → 결과 확인 → 필요시 5차 Run(hidden=64) 순서로 진행.

### 5-1. 피처 공학 단계별 계획

피처를 한꺼번에 추가하면 어떤 피처가 효과인지 알 수 없으므로 단계적으로 진행한다.

#### 1단계 — 4차 Run (지금 적용)

| 피처 | 공식 | 역할 | 누수 |
|---|---|---|---|
| Y_trailing | `rolling(21).sum()` | previous baseline의 핵심 신호 직접 노출 | 없음 ✅ |
| mean_21 | `rolling(21).mean()` | 중기 추세 | 없음 ✅ |
| std_21 | `rolling(21).std()` | 중기 변동성 클러스터링 | 없음 ✅ |

→ input_size = 1 (log_ret) + 3 = **4채널**

Y_trailing이 핵심이다. 타깃(21일 forward 누적)과 동일 시간 스케일 신호를 모델에 직접 주입하므로 previous baseline이 0.92를 찍는 자기상관을 LSTM이 비로소 활용할 수 있게 된다.

#### 2단계 — 4차 Run 결과 보고 판단

| 피처 | 공식 | 역할 |
|---|---|---|
| z_21 | `(log_ret - mean_21) / std_21` | 평균회귀 신호 (ACF lag1 음수 → 단기 반전 경향) |
| mean_5 | `rolling(5).mean()` | 단기 추세 (주 단위) |
| Y_trailing_5 | `rolling(5).sum()` | 단기 누적 |

#### 3단계 — Phase 2 이후

거래량(`vol_ratio`), VIX, 금리 등 외부 시계열 — 데이터 수집 부담 및 누수 검증 복잡도 증가.

### 5-3. 4차 Run 성공 기준

| 지표 | 기대 방향 | 의미 |
|---|---|---|
| R²_OOS | > 0 | zero baseline 초과 (관문) |
| hit_rate | > 0.55 (현행 관문) | — |
