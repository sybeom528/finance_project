## 1. 배경 — 왜 모델 학습 전에 통계검정을 하는가

LSTM을 바로 돌리기 전에 데이터가 다음 두 조건을 만족하는지 확인해야 한다.

1. **적용 가능성**: 시계열이 정상(stationary)이어야 훈련-테스트 간 패턴 일관성이 보장됨
2. **기대 성능 범위**: 자기상관 크기를 사전에 측정해야 "잘 되면 이상한 것, 못 되면 당연한 것"을 구분할 수 있음

이 노트북은 "LSTM이 잘 될 것"을 보여주는 게 아니라, **"적용 가능하되 성능은 제한적일 것"을 수치로 미리 확인하는 문서**다.

## 3. 검정별 상세 해석

### 3-1. ADF + KPSS — 정상성 검정 (LSTM 적용의 전제 조건)

LSTM 학습이 의미 있으려면 훈련 구간에서 배운 통계 특성(평균·분산)이 테스트 구간에서도 유지되어야 한다. 이것이 **정상성**이다.

우리가 주가(Adj Close)가 아닌 **log_return**을 사용하는 이유가 여기 있다. 주가는 추세가 있어 비정상이지만, log_return은 이론적으로 정상이다. ADF + KPSS로 이를 실증 확인한 것.

```
ADF  귀무: 단위근 있음(비정상)  → p≈0.000 → 기각 → 정상성 지지 ✅
KPSS 귀무: 정상임              → p≥0.10  → 기각 못함 → 정상성 지지 ✅
두 검정이 모두 정상성을 지지 → 추가 차분(differencing) 불필요
```

> **왜 ADF와 KPSS 둘 다 쓰는가**: 두 검정은 귀무가설이 반대라 상호 보완적이다. ADF만 통과해도 KPSS가 반박할 수 있다. 둘 다 통과해야 정상성이 견고하게 확인된다.

---

### 3-2. ACF/PACF + Ljung-Box — 자기상관 검정 (학습 가능한 패턴이 있는가)

**가장 중요한 검정**이다. 자기상관이 없으면 LSTM이 학습할 패턴 자체가 없다.

```
Ljung-Box: lag 5/10/20/30 모두 p≈0 → "자기상관이 통계적으로 유의하게 존재"
그런데...
ACF 절댓값 최대: 0.13
→ lag 1 과거 수익률로 미래를 설명하는 R² ≈ 0.13² ≈ 1.7%
```

**핵심 구분**: 통계적 유의(p < 0.05) ≠ 경제적 예측력(실제로 쓸모 있는 크기)

표본이 2,514개로 크면 신뢰구간이 ±0.039로 좁아져서, ACF=0.04만 돼도 유의하게 잡힌다. "유의하다"는 것은 "0이 아니다"는 것이지 "크다"는 게 아니다.

이것이 **Phase 1 가설("univariate LSTM으로는 성능이 제한적")을 사전에 지지하는 증거**다. 실제로 Phase 1 결과(R²_OOS ≈ +0.016~+0.066)가 낮게 나온 것과 정확히 대응된다.

---

### 3-3. ARCH-LM — 변동성 클러스터링 검정

```
수익률 자체:    ACF max ≈ 0.13 (약함)
제곱 수익률:    ACF가 lag 130 이상까지 유의 (강함)
```

수익률의 방향(오를지 내릴지)보다 **변동성(얼마나 크게 움직일지)**이 훨씬 예측하기 쉽다는 뜻이다.

Univariate log_return만 입력으로 주면 LSTM이 이 변동성 구조를 간접적으로밖에 못 잡는다. 이것이 Phase 2에서 변동성 피처(vol_20d, ATR 등)를 명시적으로 추가하는 핵심 동기다.

---

### 3-4. Jarque-Bera — 정규성 검정 (Loss 함수 선택)

```
JB 통계량: SPY 24,051 / QQQ 6,561 → p≈0 → 정규분포 강력 기각
초과 첨도: SPY 15.1, QQQ 7.9 (정규분포=0)
```

수익률 분포의 꼬리가 정규분포보다 훨씬 두껍다(fat-tail). COVID 급락(-11.6%)처럼 극단적인 날이 정규분포가 예측하는 것보다 훨씬 자주 발생한다.

```
MSE 사용하면: 극단값에 제곱 페널티 → 모델이 그 몇 날을 맞추는 방향으로 왜곡
Huber loss:  |오차| < δ 구간은 MSE, 그 이상은 MAE → 이상치에 robust
```

→ **Huber loss 채택의 직접 근거**가 이 검정에서 나온다.

---

### 3-5. seq_len 검증 (PACF cutoff + 제곱 ACF + AR AIC/BIC)

| 기준 | SPY | QQQ |
|---|---|---|
| PACF cutoff | lag 3 | lag 2 |
| 제곱 ACF 소멸 | >130 lag | >130 lag |
| AR AIC 최적 p | 9 | 9 |
| AR BIC 최적 p | 9 | 1 |

선형 패턴은 최대 lag 9(약 2주)에 집중되어 있다. seq_len을 그 이상으로 늘려도 선형 예측력 향상은 없다. Phase 1에서 seq_len=63→21로 줄였을 때 오히려 성능이 올라간 것과 일치한다.

변동성 구조는 130 lag 이상 지속되지만, 훈련 샘플 수 감소와 연산 비용을 고려해 seq_len=21을 채택했다.

## 4. 종합 판단

```
검정                  결과                    판단
─────────────────────────────────────────────────────────
ADF + KPSS            정상성 ✅               LSTM 적용 가능 (전제 충족)
Ljung-Box             유의하지만 ACF≤0.13     패턴 있음, 하지만 설명력 1.7%
ARCH-LM               강한 ARCH 효과           변동성에 더 강한 구조 존재
Jarque-Bera           비정규 fat-tail          Huber loss 사용 필수
seq_len 검증          선형=lag 9 이하          seq_len=21 충분
─────────────────────────────────────────────────────────
종합     "LSTM 적용은 가능하지만 성능은 구조적으로 제한적"
         → Phase 1 R²_OOS ≈ +0.016~+0.066으로 실제 확인됨
```

이 노트북의 통계검정은 **"모델이 잘 될 것"을 증명하는 게 아니라**, "우리가 왜 높은 성능을 기대하면 안 되는지, 그리고 성능을 높이려면 무엇을 추가해야 하는지(변동성 피처, Huber loss)"를 수치로 보여준다. Phase 1 결과가 검정 예측 범위 안에서 나왔다는 사실이 분석이 올바르게 설계됐음을 역으로 보여준다.

## 5. Phase 2 이후 시사점

| 검정 | Phase 2+ 개선 방향 |
|---|---|
| ARCH-LM (강한 변동성 구조) | 변동성 피처(vol_20d, ATR, VIX) 명시적 입력 추가 |
| Jarque-Bera (fat-tail) | Huber loss 유지, 추후 Quantile Regression 고려 |
| ACF 한계 (선형 예측력 1.7%) | 비선형 패턴 포착을 위한 멀티피처 또는 어텐션 메커니즘 |
| seq_len (선형=9일) | seq_len=21 유지, 변동성 피처 추가 시 seq_len 재검토 |
