## 1. 설계 변경 배경

11차 Run에서 SPY R²=+0.127로 크게 향상됐으나, QQQ가 IMF 수 불일치 버그(N_IMFS=10 전역 고정, 실제 QQQ=11)로 R²=-2.789 FAIL했다. 두 가지를 동시에 수정한다.

1. **버그 수정**: `n_imfs_ticker = ceemdan_extra[ticker].shape[1]` (ticker별 동적 IMF 수)
2. **horizon 단축**: 21d → **5d** — 예측 난이도 완화로 두 자산 동시 통과를 목표

BL 연동 시 horizon 불일치는 `Q_5d × (21/5)` 선형 스케일링으로 근사 처리 가능.

## 4. 핵심 발견 — Mean Collapse 해소

### 4-1. pred_std / true_std 비교 (Phase 1 전체)

| Run | SPY pred/true | QQQ pred/true | 판정 |
|---|---|---|---|
| 9차 (H=14, 단순 LSTM) | 0.209 | 0.205 | ⚠️ COLLAPSE |
| 11차 (H=21, CEEMDAN, 버그) | — | 무효 | — |
| **12차 (H=5, CEEMDAN)** | **0.833** | **0.862** | **✅ 정상** |

12차 Run에서 pred_std/true_std가 0.83~0.86으로 **9차 Run(0.21) 대비 4배 이상 개선**. CEEMDAN 분해로 개별 IMF가 주파수별로 독립 학습하므로, 합산 예측이 원래 수익률의 진폭을 잘 복원한다. 이것이 CEEMDAN-LSTM 도입의 가장 두드러진 성과다.

### 4-2. best_epoch 분포 비교

| | 9차 Run (H=14) | 12차 Run (H=5, CEEMDAN) |
|---|---|---|
| best_epoch=1 (SPY) | 17/122 (13.9%) | **0/82 (0%)** |
| best_epoch=1 (QQQ) | 29/122 (23.8%) | **0/82 (0%)** |
| best_epoch≤3 (SPY) | 54/122 (44.3%) | **0/82 (0%)** |
| best_epoch≤3 (QQQ) | 74/122 (60.7%) | **0/82 (0%)** |

9차 Run에서 절반 가까이 되는 폴드가 3에포크 이내에 조기 종료됐던 반면, 12차 Run은 **모든 폴드에서 에포크 4+ 이상 충분히 학습**. 모델이 각 IMF의 주파수 패턴을 실제로 학습하고 있음을 의미한다.

- 11차 SPY R²=+0.127이 더 높다. H=21(BL 정합)의 이점 + CEEMDAN 효과 결합이 가장 강한 조합임을 시사. QQQ 버그 수정 후 H=21 재실험 가치 있음.

## 6. 구조적 문제 잔존 여부

| 문제 | 9차 Run | 12차 Run | 방향 |
|---|---|---|---|
| Mean collapse | pred/true≈0.21 | **0.83~0.86** | ✅ **해소** |
| best_epoch 1 비율 | 14~24% | **0%** | ✅ **해소** |
| r2_oos 분산 | ±0.57~0.68 | ±0.57~0.65 | 소폭 개선 |
| 위기 레짐 min R² | ≈ -4.1 | ≈ -3.1 | 소폭 개선 |
| IS-OOS 과적합 갭 | 큼 | 여전히 큼 | 유지 |
| previous baseline 대비 격차 | R²=0.05~0.07 vs 0.53~0.55 | R²=0.016~0.062 vs 0.53~0.55 | ⚠️ 미해소 |

Mean collapse와 early stopping 문제는 CEEMDAN 분해로 대부분 해소됐다. 위기 레짐 성능 붕괴와 IS-OOS 갭은 모델 구조 변경 없이 근본 해결이 어렵다. **previous baseline 대비 절대적 예측력 격차는 5d CEEMDAN에서도 해소되지 않았다** — 이는 horizon 불일치(5d vs 21d), CEEMDAN 전역 분해의 누수, 일별 수익률 자체의 예측 어려움이 복합적으로 작용한 결과다.
