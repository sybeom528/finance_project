## 1. 프로젝트 배경 및 목표

### 1-1. 전체 파이프라인에서 Phase 1의 위치

본 프로젝트는 7단계 파이프라인으로 구성되며, Phase 1은 Step 3(딥러닝 모델링)에 해당한다.

```
Step 1: 데이터 수집
Step 2: 전처리·EDA → long_panel.parquet
Step 3: [Phase 1] LSTM Walk-Forward → Q_dl, Omega_dl
Step 4: Black-Litterman → μ_BL, Σ_BL
Step 5: 위험성향별 MVO → w*
Step 6: 백테스팅
Step 7: Streamlit 대시보드
```

Phase 1의 최종 목표는 LSTM 모델로 **각 자산의 21일 누적 기대수익률(Q)과 불확실성(Omega)** 을 추출하여 Black-Litterman 모델에 입력하는 것이다.

### 1-2. Phase 1 세부 목표

- **예측 대상**: SPY, QQQ의 21일 누적 log-return (`target[t] = log(P[t+21]/P[t])`)
- **모델**: LSTM (univariate 또는 multivariate 탐색)
- **검증 방법**: Rolling Walk-Forward + Purge + Embargo (Lopez de Prado 2018)
- **성능 관문**: Hit Rate > 0.55 AND R²_OOS > 0

## 3. LSTM 설계 구조

### 3-1. Walk-Forward 검증 구조

```
Fold 1: ├── IS (231일) ──┤ purge(21) │ emb(21) │ OOS(21) ┤
Fold 2:   ├── IS (231일) ──┤ purge(21) │ emb(21) │ OOS(21) ┤
...                                           (step: 21일씩 우측 이동)
```

| 구성요소 | 길이 | 역할 |
|---|---|---|
| IS (In-Sample) | 231일 | 학습 + 검증 (train 80% / val 20%) |
| Purge | 21일 | 타깃 t+21 미래 참조 차단 |
| Embargo | 21일 | Fold 간 자기상관 누수 차단 |
| OOS | 21일 | 실제 성능 평가 |
| Step | 21일 | 폴드 슬라이드 간격 |

**최종 fold 수**: 105 (분석 기간 2016~2025, 유효 타깃 n=2,493 기준)

**Purge + Embargo를 함께 쓰는 이유**:
- Purge: IS 내 샘플의 타깃 생성 구간(t+1~t+21)이 OOS 첫 시점과 겹치는 것 차단. 타깃이 21일 누적이므로 purge=21 필수.
- Embargo: IS 직후 자기상관 잔존 효과로 OOS 첫 구간이 부풀려지는 것 방지.

### 3-2. 데이터 구성

- **수집 기간**: 2009-01-01 ~ 2026-03-31 (yfinance, 워밍업 1년 포함)
- **분석 기간**: 2016-01-01 ~ 2025-12-31 (2,514 거래일)
- **자산**: SPY, QQQ (Adj Close → log-return)
- **타깃 정의**: `target[t] = sum(log_ret[t+1:t+22])` — 21일 **누적** log-return

  | | 단순 방식 (기각) | **누적 방식 (채택)** |
  |---|---|---|
  | 공식 | `log_ret[t+21]` (하루) | `sum(log_ret[t+1:t+22])` |
  | 채택 근거 | — | BL Q는 리밸런싱 기간 전체 수익률 필요 |

- **유효 타깃 수**: 2,493 (마지막 21행 NaN 제외)

### 3-3. 모델 구조 (`scripts/models.py`)

```python
class LSTMRegressor(nn.Module):
    """
    구조: LSTM → Dropout(head) → Linear(hidden → 1)
    - batch_first=True (DataLoader 출력 (B,T,F) 호환)
    - num_layers=1일 때: LSTM 내부 dropout=0, head 앞 nn.Dropout 별도 적용
    - forget_gate_bias_init=None (기본값 b_f=0, 일별 ACF 부재로 b_f=1 기각)
    """
```

### 3-4. 학습 파이프라인 (`scripts/train.py`)

| 구성요소 | 설정값 | 근거 |
|---|---|---|
| Loss | HuberLoss (δ=0.01) | Fat-tail 분포 (Jarque-Bera 비정규), 이상치 완화 |
| Optimizer | AdamW (lr=1e-3, wd=1e-3) | L2 정규화 포함 |
| Scheduler | ReduceLROnPlateau (patience=3, factor=0.5) | val loss 정체 시 lr 절반 |
| EarlyStopping | patience=5 | 과적합 전 조기 중단 |
| max_epochs | 30 | 학습 상한 |

### 3-5. 평가 지표 (`scripts/metrics.py`)

| 지표 | 정의 | 관문 |
|---|---|---|
| Hit Rate | 방향 적중률 (부호 일치 비율) | > 0.55 |
| R²_OOS | `1 - SSE/sum(y²)` (Campbell & Thompson 2008, 0 예측 대비) | > 0 |

**Baseline 4종**: zero baseline / previous baseline(직전 값 복사) / train_mean / LSTM

## 7. 실패 원인 종합 분석

### 7-1. 근본 원인: 훈련 샘플 절대 부족

| 구성 | 훈련 샘플/fold | 파라미터 수 | 비율 |
|---|---|---|---|
| 1차 (seq=126, hidden=128) | 84개 | 199,297개 | 2,372배 |
| 2~5차 (seq=63, hidden=32) | 134개 | 4,513개 | 34배 (개선됐으나 여전히 과적합) |

피처를 추가할수록 모델이 fold-specific 패턴을 암기(과적합)하여 OOS에서 역방향 예측 발생.

- fold 12 (4차): pred_mean = +21.8% vs true_mean = -0.1% (방향 반전)
- fold 14 (5차): SPY R²=-131.10, pred_mean=-11.0% vs true_mean=+2.3%

### 7-2. Val > Train > Test 패턴의 구조적 원인

```
SPY 5차 Run: train HR=0.6545, val HR=0.7564, test HR=0.5338
```

**Val > Train (2가지 원인)**:
1. **Dropout 비대칭**: 학습 시 30% 유닛 비활성 vs 평가 시 전체 활성 → 같은 가중치라도 평가 버전이 더 안정
2. **EarlyStopping 선택 편향**: val 손실이 최소인 체크포인트 저장 → 낙관적 편향

**Test << Val (핵심 문제)**:
```
[IS 231일: 특정 시장 레짐]
  train(80%) │ val(20%)     ← 같은 레짐 내부 분할

[purge 42일 gap]

[OOS 21일: 다른 레짐일 수 있음]  ← 레짐 분포 이동
```

Val은 IS 내부의 같은 레짐에서 분할된 것이므로 val R²=+0.33이 가능. OOS는 레짐이 달라지면 IS에서 배운 패턴이 역방향으로 작동 → R²=-131 붕괴.

**결론**: val R²=+0.33은 "모델이 IS 안에서 뭔가를 배운다"는 증거. 문제는 IS 레짐에 특화되어 OOS로 전이되지 않는다는 점.

### 7-3. VIX 피처가 성능을 악화시킨 이유

```
고VIX 상황에서 21일 뒤 수익률:
  2020년 3월 (COVID):  VIX=80 → 급반등 (+30%)   ← 양(+)
  2022년 (Fed 긴축):   VIX=30 → 계속 하락         ← 음(-)
  2008년 (금융위기):   VIX=60 → 추가 하락          ← 음(-)
```

VIX 절대 수준이 같아도 결과 방향이 반대 → **VIX level → 21일 수익률 방향 관계 자체가 레짐에 따라 달라짐**. 134 훈련 샘플로는 이 비선형·비정상 관계를 학습 불가.

**대안 (미실험)**: VIX_change (전일 대비 변화율) 또는 VIX_zscore (최근 60일 편차) — 절대 레짐이 아닌 "공포가 커지고 있는가" 는 IS 구간에 무관하게 해석 일관.

### 10-2. 향후 선택지

| 선택지 | 설명 | 장점 | 단점 |
|---|---|---|---|
| **A) Phase 1 결과 수용** | 3차 Run을 최종 출력으로 확정 후 Phase 2(GRU 등) 진행 | 즉시 진행 가능. BL에서 Q 작게 + Omega 크게 설정하면 약한 신호도 활용 가능 | R²_OOS < 0 — 예측 파워 증명 불가 |
| **B) Walk-Forward IS 확대** | IS=231 → 504로 늘려 n_train=134 → 441로 증가 | 훈련 샘플 부족 근본 해결 | 폴드 수 감소, IS-OOS 레짐 이동 문제는 여전히 존재 |
| **C) 모델 교체** | GRU (파라미터 25% 감소) 또는 Ridge 단순화 | 용량 감소로 과적합 완화 | 코드 구조 변경 필요 (Phase 2 영역) |
