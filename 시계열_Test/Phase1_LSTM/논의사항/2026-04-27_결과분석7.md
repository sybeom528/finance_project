### 2-2. ret_3m 피처 근거

**ret_3m이란**: t일까지의 과거 63거래일(≈3개월) 누적 로그수익률.

```python
ret_3m[t] = sum(log_return[t-62 : t+1])  # rolling(63).sum()
           = log(price[t] / price[t-63])
```

**누수 여부**: t일 기준 backward-looking (t+1 이후 미사용) → **누수 없음** ✅

**선택 근거**:
- **학술 근거**: Jegadeesh & Titman (1993) — 3~12개월 모멘텀이 횡단면 수익률을 예측함
- **미시도**: ret_1m(=Y_trailing)은 4차 Run에서 실패했으나 ret_3m은 한 번도 시도하지 않은 구간
- **차별점**: ret_1m(21일)은 단기 반전 노이즈에 취약, ret_3m(63일)은 중기 추세 반영
- **mean collapse 가능성**: 모멘텀 신호가 strong 폴드를 구분해 주면 예측 분산 확대 기대

**NaN 처리**: 분석 기간 첫 62일 NaN → `fillna(0)` (중립 모멘텀 신호로 처리)  
→ 영향받는 폴드: fold 0~2 (IS 윈도우 시작이 첫 63일 이전인 경우) — 전체 92폴드의 3%

## 4. 근본 원인 분석 — 왜 피처를 추가하면 성능이 떨어지는가

### 4-1. 모멘텀-수익률 관계의 레짐 의존성

Jegadeesh & Titman(1993)의 모멘텀 효과는 **횡단면(cross-sectional)** 예측이다 — 같은 시점에서 '어떤 자산이 다른 자산보다 더 좋은가'를 판별하는 것.

우리가 시도한 것은 **시계열(time-series)** 예측 — 'SPY가 다음 달 오를 것인가'. 이 두 가지는 다르다:

| 구분 | 모멘텀이 잘 작동하는 맥락 | 우리 설정 |
|---|---|---|
| 횡단면 | 상승 자산 군 vs 하락 자산 군 분리 | 해당 없음 |
| 시계열 | 추세 지속 레짐(bull) | IS와 OOS 레짐이 다를 수 있음 |

특히 **21일(1개월) 선행 수익률**은 너무 짧은 horizon이다. 3개월 모멘텀이 3개월 이상 지속된다는 근거는 있지만, 21일 내 반전(mean-reversion)이 더 지배적일 수 있다.

### 4-2. pred_std 증가의 역설

7차 Run에서 가장 주목할 변화는 pred_std/true_std가 0.252 → 0.460으로 크게 뛰었다는 점이다. 이는 mean collapse가 해소된 것처럼 보이지만, **R²_OOS는 오히려 더 나빠졌다**.

```
R² = 1 - Σ(y_true - y_pred)² / Σ(y_true)²

pred_std 증가 → 예측 크기가 커짐
                → 방향이 맞으면 R² 개선
                → 방향이 틀리면 R² 더 악화 (오차 제곱이 더 커짐)
```

즉, ret_3m이 '더 강한 신호'를 주었지만 그 신호가 **잘못된 방향을 더 강하게** 예측하게 만든 것이다.

### 4-3. IS-OOS 레짐 이동(Regime Shift) 지속

IS=504(2년) 구간에서 학습된 ret_3m-수익률 패턴이 OOS 21일 구간에서 반전되는 폴드가 다수 존재한다.

```
예시:
  IS 구간: 2019-2021 (저금리 bull market) → ret_3m 높음 → 수익률 양(+) 관계 학습
  OOS 구간: 2022 초 (연준 긴축 전환)     → ret_3m 높음 → 수익률 음(-) (레짐 반전)
  결과: 모델이 강한 양(+) 예측 → 실제 음(-) → 극단적 R² 악화
```

이 메커니즘이 반복될수록 R²_OOS 분산(std)이 커지고 평균이 낮아진다.

### 4-4. 다중공선성 — log_ret 시퀀스와의 중복

ret_3m은 log_ret의 63일 rolling sum이다. 입력에 이미 log_ret 시퀀스 21일이 포함되어 있으므로 ret_3m과 log_ret 시퀀스 사이에 높은 상관관계가 존재한다.

```
seq_len=21인 log_ret 시퀀스 중 21일이 ret_3m의 일부 (21/63 = 33%)
→ LSTM이 ret_3m에 과잉 반응하고 시퀀스 자체 패턴 학습을 방해할 수 있음
```

### 4-5. 왜 IS=504(483샘플)로도 해결이 안 되는가

| 근거 | 설명 |
|---|---|
| 레짐 수 vs 샘플 수 불균형 | 2016-2025에 bull(2016-21), 충격(2020 코로나), bear(2022), 회복(2023-25) 등 레짐 4~5개 존재. 483샘플이 절대 부족한 것이 아니라 각 레짐당 학습 데이터가 불충분 |
| 모멘텀-수익률 비선형성 | 각 레짐에서 모멘텀→수익률 부호가 바뀜. LSTM이 이 비선형성을 학습하려면 훨씬 더 많은 레짐 전환 사례가 필요 |
| OOS 21일의 통계적 불안정성 | 21일 수익률은 분산이 매우 커서 OOS 1개 값의 방향이 우연에 의해 결정되는 비중이 높음 |

### 4-6. 결론

**피처를 추가하면 성능이 떨어지는 패턴이 7번의 Run 전체를 통해 일관되게 관찰된다.**

| 피처 추가 시도 | R²_OOS 변화 | 방향 |
|---|---|---|
| 4차: Y_trailing + mean_21 + std_21 | -0.21 → -2.15 | 대폭 악화 |
| 5차: VIX | -0.21 → -1.13 | 대폭 악화 |
| 7차: ret_3m | -0.025 → -0.090 | 악화 |

단, 악화 폭은 훈련샘플이 늘어날수록(134 → 483) 줄어드는 추세. 근본 한계는 **21일 선행 수익률 자체의 예측 불가능성**이다. 주가 일별 수익률 ACF가 lag 2 이후 ≈ 0이라는 사실은 어떤 단기 피처도 21일 수익률을 일관되게 예측할 수 없음을 시사한다.

## 5. 다음 단계

### 5-1. 현 상태 평가

7차 Run으로 6차(R²=-0.025)보다 오히려 악화(-0.090)됐다. **6차 Run이 현재 Phase 1 best**이며, 이를 최종 출력으로 확정하는 것이 현실적이다.

- hit_rate=0.659 > 0.55 관문 통과 ✅ — 방향성 신호는 존재
- R²=-0.025 ≈ zero baseline — 예측 크기 신뢰도 낮음
- Black-Litterman 적용: Q(뷰) 크기를 작게, Omega(불확실성)를 높게 설정하면 활용 가능

### 5-2. Phase 2 진행 판단

| 선택 | 근거 |
|---|---|
| **6차 Run 결과 수용 후 Phase 2 진행** | 추가 피처 시도는 일관되게 실패. 예측 한계는 데이터 구조에서 기인 (ACF ≈ 0). Black-Litterman에서 불확실성 높은 뷰로 처리 가능 |
