## 3. 원인 분석 — 가설 재검토

### 3-1. 핵심 오해: Y_trailing ≠ previous baseline이 사용하는 신호

3차 Run 계획에서의 가정:
> "Y_trailing이 핵심이다. previous baseline이 0.92를 찍는 자기상관을 LSTM이 비로소 활용할 수 있게 된다."

**이 가정이 틀렸다.** 정확한 계산:

```
타깃:            Y[t]   = sum(log_ret[t+1 : t+22])
previous 예측:   Y[t-1] = sum(log_ret[t   : t+21])  ← Y[t]와 20일 겹침
Y_trailing[t]:           = sum(log_ret[t-20: t+1])   ← Y[t]와 0일 겹침

Y_trailing[t] = Y[t-21]  (타깃 21일 이전 값 — 겹침 없음)
```

| 신호 | 타깃 Y[t]와 겹침 | 예측 가능? | 합법? |
|---|---|---|---|
| Y[t-1] (previous baseline 사용) | 20/21일 겹침 | 구조적 자기상관 ✅ | ❌ **미래 데이터 포함** |
| Y_trailing[t] (우리가 추가한 피처) | 0일 겹침 | 약한 모멘텀만 | ✅ 합법 |

**결론**: previous baseline의 hit_rate 0.92는 `log_ret[t+1:t+21]` (미래 20일)을 암묵적으로 포함하는 Y[t-1]을 사용하기 때문에 발생한 구조적 아티팩트다. **합법적인 입력 피처로는 이 신호를 재현할 수 없다.**

Y_trailing이 실제로 포착하는 것은 21일 모멘텀(trailing 21-day return → next 21-day return)이며, 이는 대형 미국 주식 ETF에서 실증적으로 매우 약한 신호다.

### 3-2. 피처 추가가 성능을 낮춘 이유

| 원인 | 설명 |
|---|---|
| **신호 대비 노이즈 증가** | 약한 모멘텀 신호(Y_trailing) + 방향성 없는 변동성 신호(std_21) → 타깃 예측에 직접적 기여가 낮음 |
| **소규모 훈련 세트 과적합** | 134 훈련 샘플 × 4 입력 채널. 채널 증가로 모델이 fold-specific 패턴 암기 |
| **초기 NaN→0 대체의 영향** | Y_trailing/std_21 초기 20행 = 0 대체. 해당 구간의 fold에서 인위적 신호 삽입 가능 |
| **모멘텀-평균회귀 혼재** | 시장 체제에 따라 Y_trailing이 모멘텀(연속) 또는 평균회귀(역전) 신호로 작동 → fold 12처럼 pred_mean이 true_mean과 반대 방향으로 큰 편향 발생 |

### 3-3. pred_std/true_std 해석

SPY pred_std/true_std = 1.19 → mean-collapse는 아니지만, **과분산**:
- 모델이 대담한 방향성 예측을 하고 있으나 방향이 틀린 fold에서 R²_OOS가 급격히 나빠짐
- 3차 Run과 비교 시, 피처가 오히려 모델을 더 확신하게 만들어 극단적 예측을 유발한 것으로 보임

## 7. 다음 피처 설계 — 체제(regime) 신호 기반

### 7-1. 4차 Run의 교훈: 피처 선택 원칙 수정

4차 실패로부터 얻은 원칙:

| 기각 원칙 | 이유 |
|---|---|
| **자산 자체의 과거 수익률 계열** (Y_trailing, Y_trailing_5) | 대형 ETF에서 21일 모멘텀은 실증적으로 약하며, 시장 체제에 따라 방향이 바뀌어 오히려 편향 유발 |
| **수익률에서 직접 파생된 피처** (std_21) | 방향 예측 기여 없이 입력 복잡도만 늘림 |
| **모멘텀 계열 일반** | 소규모 훈련 세트(134샘플)에서 체제 혼재 시 fold-specific 패턴 암기 |

유지할 원칙:
- **방향이 아닌 체제(regime)를 나타내는 신호** → 어떤 시장 환경인지를 알려줘 조건부 예측력 제공
- **자산 외부의 독립 정보** → log_return과 낮은 상관, 진짜 새로운 정보
- **이미 전처리된 데이터 우선** → `long_panel.parquet` 내 매크로 피처 활용

### 7-2. 5차 Run 후보 피처 (우선순위 순)

| 우선순위 | 피처 | 출처 | 이유 | 누수 |
