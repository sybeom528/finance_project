## 1. 데이터 누수 방지

### 1.1 Look-ahead Bias의 3가지 유형
- **출처**: `week2_시퀀스데이터/03_look_ahead_bias.md` §Look-ahead Bias의 3가지 유형
- **요지**: 미래 정보를 훈련 데이터에 포함시키는 모든 형태. **Label Leakage**, **Normalization Leakage**, **Window Leakage** 세 유형 존재.

### 1.2 Label Leakage
- **출처**: `week2_시퀀스데이터/03_look_ahead_bias.md` §Label Leakage
- **요지**: 훈련 시점 t에서 미래 수익률 `y[t+h:t+h+h]`를 알고 있는 상황. 현실에서 절대 불가능한 정보를 모델에 제공.

### 1.3 Normalization Leakage
- **출처**: `week2_시퀀스데이터/03_look_ahead_bias.md` §Normalization Leakage
- **요지**: 전체 데이터로 `scaler.fit` 후 split하면 test 정보가 훈련 정규화에 포함되어 누수. **train만 fit, val/test는 transform** 강제.

### 1.4 Window Leakage
- **출처**: `week2_시퀀스데이터/03_look_ahead_bias.md` §Window Leakage
- **요지**: sliding window로 변환할 때 동일 샘플에서 파생된 창들이 train/test에 분산되면 누수. **시간 간격 기반 분리** 필수.

### 1.5 Walk-Forward의 purge·embargo 메커니즘
- **출처**: `week2_시퀀스데이터/03_look_ahead_bias.md` §`make_wf_windows` 함수 해부
- **요지**: 예측 대상 기간(`t+h:t+2h`) 이후부터만 다음 훈련 윈도우 시작. **purge + embargo로 정보 누수 차단**.

### 1.6 SequenceDataset의 scaler 외부 주입
- **출처**: `week2_시퀀스데이터/04_pytorch_dataset.md` §SequenceDataset 설계
- **요지**: scaler는 **훈련 윈도우 데이터만으로 fit**. DataLoader 생성 시 각 윈도우별 독립 정규화로 누수 방지.

## 2. 시계열 데이터 처리

### 2.1 i.i.d 가정 위반
- **출처**: `week1_시계열기초/01_시계열데이터의본질.md` §i.i.d 가정 위반
- **요지**: 금융 시계열은 i.i.d 가정을 심각하게 위반. 연속적 의존성·이분산성으로 고전 통계 방법 부적용.

### 2.2 가격이 아닌 수익률을 써야 하는 이유
- **출처**: `week1_시계열기초/01_시계열데이터의본질.md` §수익률 변환
- **요지**: 원가 `P[t]` 직접 사용 → 비정상 시계열. **로그 수익률** `r[t] = log(P[t]/P[t-1])` 로 변환해 정상성 확보.

### 2.3 약정상성(weak stationarity) 3조건
- **출처**: `week1_시계열기초/02_정상성.md` §약정상성 조건
- **요지**: 평균 불변 / 자기상관이 lag에만 의존 / 분산 유한. 셋 중 하나라도 위배 시 모델 신뢰도 급감.

### 2.4 정상성 검정의 함정
- **출처**: `week1_시계열기초/02_정상성.md` §정상성 검정의 함정
- **요지**: ADF/KPSS p-value 작다 → 정상성 보장 X. 표본 크기·드리프트·구조적 단절에 민감, 해석 주의.

### 2.5 금융 시계열 양식화된 사실 (stylized facts)
- **출처**: `week1_시계열기초/03_자기상관.md` §금융 시계열 양식화된 사실
- **요지**: 수익률 = (a) 약한 자기상관, (b) 변동성 클러스터링(이분산성), (c) 두꺼운 꼬리. 간과 시 시장 변동성 저평가.

### 2.6 시계열 분해(가법 vs 승법) 선택 위험
- **출처**: `week1_시계열기초/04_시계열분해.md` §분해 선택의 위험
- **요지**: 가법 vs 승법 분해 오선택 → 추세·계절·잔차 해석 왜곡. 데이터 범위·분산 구조 먼저 검토.

### 2.7 Sliding Window 변환의 표본 수 트레이드오프
- **출처**: `week2_시퀀스데이터/01_tabular_vs_sequential.md` §Sliding Window 변환의 트레이드오프
- **요지**: `(2310, 17) → (N', T, 17)` 변환 시 샘플 수 급감. **충분한 훈련 데이터 vs 윈도우 크기 = 상충 관계**.

### 2.8 stride 선택 원칙
- **출처**: `week2_시퀀스데이터/02_sliding_window.md` §stride 선택 원칙
- **요지**: stride=1 → 높은 상관, stride=T → 독립이지만 표본 부족. 현실 권장은 **stride = T/2~T**, 문제 특성에 맞게 조정.

### 2.9 윈도우 크기 T 선정 기준
- **출처**: `week2_시퀀스데이터/02_sliding_window.md` §윈도우 크기 T 선정 기준
- **요지**: T<20 → 의존성 포착 불충분 / T>200 → 표본 급감. 평균적으로 **T=60 근처**에서 탐색.

## 3. 학습·PyTorch

### 3.1 Dataset vs DataLoader 역할 분담
- **출처**: `week2_시퀀스데이터/04_pytorch_dataset.md` §Dataset vs DataLoader 역할 분담
- **요지**: Dataset = `(X, y)` 쌍 제공 / DataLoader = 배치·셔플·병렬화. **두 모듈 경계 명확화 못 하면 누수 또는 중복 처리**.

### 3.2 `__len__` 메서드 정의의 중요성
- **출처**: `week2_시퀀스데이터/04_pytorch_dataset.md` §__len__ 메서드 정의의 중요성
- **요지**: SequenceDataset의 `__len__`은 가용 윈도우 개수 반환. 오류 시 epoch 미완료 또는 IndexError.

### 3.3 (B, T, F) 축 계약 준수
- **출처**: `week2_시퀀스데이터/05_tensor_dimensions.md` §(B,T,F) 축 계약
- **요지**: `batch_first=False` → `(T,B,F)`, `batch_first=True` → `(B,T,F)`. 모델 입력 형식과 DataLoader 출력 일치 필수.

### 3.4 permute vs reshape의 차이
- **출처**: `week2_시퀀스데이터/05_tensor_dimensions.md` §permute vs reshape
- **요지**: `permute`는 축 순서 변경(메모리 재배열) / `reshape`은 크기만 변경. 잘못 선택 시 데이터 순서 뒤섞임 또는 성능 저하.

### 3.5 Broadcasting 규칙 오류
- **출처**: `week2_시퀀스데이터/05_tensor_dimensions.md` §Broadcasting 규칙
- **요지**: `(B,T,F)`와 `(F,)` 연산은 자동 확대 가능, 그러나 `(T,F)`와 `(B,1)`은 불가. **명시적 reshape 필요**.

### 3.6 [추가 조사 필요] week3 PyTorch 학습 루프 흔한 실수 10가지
- **출처**: `week3_시퀀스모델/04_PyTorch_학습루프.md` (학습계획에 §16 경량 학습 패턴까지 명시되어 있으나, 본 정리에 세부 항목 누락)
- **요지**: 학습계획에 따르면 zero_grad 누락, eval 모드 미전환, `.item()`/`.detach()` 그래프 처리, gradient clipping 누락 등 10가지 함정이 정리되어 있음. **Step 2~3 학습 루프 작성 직전에 직접 정독 권장**.

### 3.7 [추가 조사 필요] week3 RNN/LSTM/GRU 게이트 함정
- **출처**: `week3_시퀀스모델/01_RNN_기본구조_한계.md`, `02_LSTM_게이트메커니즘.md`, `03_GRU.md`
- **요지**: 학습계획에 따르면 RNN gradient vanishing 18자리 분기, LSTM `b_f=1` 트릭, GRU `b_z=-2~-4` 초기화 효과 등 실측 결과가 정리됨. **모델 정의 시 초기화 규약 직접 확인 권장**.
